### 🔄 Five-Step Request Flow

1. Client → Server request
2. Server → Anthropic API
3. Claude model processing
4. API → Server response
5. Server → Client response [\[Accessing the API\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287726)

***

### 🔐 Why You Need a Server

* API key is **secret → must not expose in client-side code**
* Direct client calls = **security vulnerability**
* Correct pattern:
  * Client → Your server → Anthropic API [\[Accessing the API\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287726)

***

### 📡 API Request Essentials

Every request must include:

* `API Key`
* `Model` (e.g., claude-3-sonnet)
* `Messages` (user input)
* `Max Tokens` (output limit) [\[Accessing the API\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287726)

📦 Supported SDKs:

* Python, TypeScript, JavaScript, Go, Ruby [\[Accessing the API\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287726)

***

# ❓ 2. Q\&A

### Q1: Why shouldn’t you call the API directly from the client?

➡️ Because it exposes your **API key**, creating a major security risk.

***

### Q2: What are the 5 steps in the request lifecycle?

➡️ Client → Server → API → Server → Client

***

### Q3: What fields are required in an API request?

➡️ API Key, Model, Messages, Max Tokens

***

### Q4: What is tokenization?

➡️ Breaking text into smaller units (tokens)

***

### Q5: What is embedding?

➡️ Converting tokens into numerical representations of meaning

***

### Q6: What determines when Claude stops generating?

➡️ Max tokens, stop sequence, or natural ending

***

### Q7: What is included in the API response?

➡️ Generated text, token usage, and stop reason
