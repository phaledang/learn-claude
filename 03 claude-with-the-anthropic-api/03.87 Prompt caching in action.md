## 1. Summary

**Page:** *Prompt caching in action*  
**URL:** <https://anthropic.skilljar.com/claude-with-the-anthropic-api/287774>

**Key concepts:**

* Prompt caching is an **optimization feature** that reduces **latency and cost** by reusing previously processed input. [\[learn-clau...arn-claude\]](https://github.com/phaledang/learn-claude/edit/main/03%20claude-with-the-anthropic-api/03.86%20Rules%20of%20Prompt%20Caching.md)
* First request → writes content to cache (valid for **1 hour**). [\[learn-clau...arn-claude\]](https://github.com/phaledang/learn-claude/edit/main/03%20claude-with-the-anthropic-api/03.86%20Rules%20of%20Prompt%20Caching.md)
* Subsequent requests → reuse cached content instead of reprocessing. [\[learn-clau...arn-claude\]](https://github.com/phaledang/learn-claude/edit/main/03%20claude-with-the-anthropic-api/03.86%20Rules%20of%20Prompt%20Caching.md)

**Where it is useful:**

* Large system prompts (\~6K tokens)
* Complex tool schemas (\~1.7K tokens)
* Repeated message content [\[learn-clau...arn-claude\]](https://github.com/phaledang/learn-claude/edit/main/03%20claude-with-the-anthropic-api/03.86%20Rules%20of%20Prompt%20Caching.md)

**Key behaviors:**

* Cache is **strictly identical-match based** (even 1 character change invalidates cache). [\[learn-clau...arn-claude\]](https://github.com/phaledang/learn-claude/edit/main/03%20claude-with-the-anthropic-api/03.86%20Rules%20of%20Prompt%20Caching.md)
* Supports **granular caching** (tools, system prompt, messages separately). [\[learn-clau...arn-claude\]](https://github.com/phaledang/learn-claude/edit/main/03%20claude-with-the-anthropic-api/03.86%20Rules%20of%20Prompt%20Caching.md)
* Usage indicators:
  * `cache_creation_input_tokens` → cache written
  * `cache_read_input_tokens` → cache reused [\[learn-clau...arn-claude\]](https://github.com/phaledang/learn-claude/edit/main/03%20claude-with-the-anthropic-api/03.86%20Rules%20of%20Prompt%20Caching.md)

**Implementation highlights:**

* Add `cache_control={"type":"ephemeral"}` to:
  * tool schema (last tool)
  * system prompt block [\[learn-clau...arn-claude\]](https://github.com/phaledang/learn-claude/edit/main/03%20claude-with-the-anthropic-api/03.86%20Rules%20of%20Prompt%20Caching.md)

## 2. Q\&A

**Q1. What problem does prompt caching solve?**  
Reduces repeated computation by reusing previously processed prompts, improving performance and cost efficiency. [\[learn-clau...arn-claude\]](https://github.com/phaledang/learn-claude/edit/main/03%20claude-with-the-anthropic-api/03.86%20Rules%20of%20Prompt%20Caching.md)

**Q2. How long does the cache last?**  
The cache remains valid for **one hour**. [\[learn-clau...arn-claude\]](https://github.com/phaledang/learn-claude/edit/main/03%20claude-with-the-anthropic-api/03.86%20Rules%20of%20Prompt%20Caching.md)

**Q3. When is caching most effective?**  
When identical content (tools/system/messages) is repeatedly sent. [\[learn-clau...arn-claude\]](https://github.com/phaledang/learn-claude/edit/main/03%20claude-with-the-anthropic-api/03.86%20Rules%20of%20Prompt%20Caching.md)

**Q4. What invalidates the cache?**  
Any change in content, even a single character. [\[learn-clau...arn-claude\]](https://github.com/phaledang/learn-claude/edit/main/03%20claude-with-the-anthropic-api/03.86%20Rules%20of%20Prompt%20Caching.md)

**Q5. Can partial caching happen?**  
Yes. If only part changes (e.g., system prompt), other parts (e.g., tools) can still be reused. [\[learn-clau...arn-claude\]](https://github.com/phaledang/learn-claude/edit/main/03%20claude-with-the-anthropic-api/03.86%20Rules%20of%20Prompt%20Caching.md)

**Q6. What order does caching follow?**

1. Tools
2. System prompt
3. Messages [\[learn-clau...arn-claude\]](https://github.com/phaledang/learn-claude/edit/main/03%20claude-with-the-anthropic-api/03.86%20Rules%20of%20Prompt%20Caching.md)

## 4. Key Takeaways

* Prompt caching = **performance + cost optimization** mechanism
* Works only with **identical repeated inputs**
* Supports **component-level caching (tools, system, messages)**
* Extremely sensitive → **small changes = full cache invalidation**
* Best suited for:
  * Agent systems with stable prompts
  * Tool-based workflows
  * RAG pipelines with repeated context
* Cache duration is **short-lived (1 hour)** → optimized for active workloads
