### Using Multiple Tools — LangChain Equivalent

Adding multiple tools in LangChain follows the same pattern: define tools, register them, and let the agent orchestrate calls.

***

### The Tools We're Adding

We need three capabilities:

* Get current date time
* Add duration to date time
* Set a reminder

These are implemented as independent tools.

***

### Defining Tools

Each tool is a function wrapped with a schema.

```python
from langchain.tools import tool
from datetime import datetime, timedelta

@tool
def get_current_datetime() -> str:
    return datetime.now().isoformat()


@tool
def add_duration_to_datetime(base_datetime: str, days: int) -> str:
    dt = datetime.fromisoformat(base_datetime)
    new_dt = dt + timedelta(days=days)
    return new_dt.isoformat()


@tool
def set_reminder(reminder_time: str, message: str) -> str:
    return f"Reminder set for {reminder_time}: {message}"
```

***

### Adding Tools to the Agent

Register all tools in the agent configuration.

```python
from langchain.agents import initialize_agent
from langchain.chat_models import ChatOpenAI

llm = ChatOpenAI(temperature=0)

tools = [
    get_current_datetime,
    add_duration_to_datetime,
    set_reminder
]

agent = initialize_agent(
    tools=tools,
    llm=llm,
    agent="openai-functions",  # structured tool calling
    verbose=True
)
```

***

### Running a Multi-Tool Request

A single user request can trigger multiple tools.

```python
agent.run(
    "Set a reminder for my doctor's appointment 177 days after Jan 1st, 2050."
)
```

***

### How It Works

The agent performs:

1. Understand the task
2. Decide tools required
3. Call tools sequentially
4. Combine results into final output

Example flow:

* Call `add_duration_to_datetime` → compute target date
* Call `set_reminder` → create reminder

***

### Message Flow (Conceptual)

* User input
* Agent reasoning
* Tool call (structured)
* Tool result
* Next tool call (if needed)
* Final response

LangChain handles this loop automatically.

***

### The Simple Pattern for Adding Tools

1. Implement tool function
2. Wrap with `@tool` (schema included)
3. Add to tools list
4. Register in agent

No changes to core logic required.

***

### Notes (Architecture-Level)

* Tool selection is model-driven (function calling / tool calling)
* Execution is deterministic in Python layer
* Agent = orchestration layer (similar to Claude tool loop)
* Tools remain modular and reusable
* Supports chaining, branching, and more complex workflows
