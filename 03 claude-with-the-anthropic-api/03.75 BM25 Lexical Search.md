### Key ideas

* **Problem:** Semantic search alone is not enough
  * It retrieves conceptually similar content but may miss **exact terms (e.g., IDs like `INC-2023-Q4-011`)** [\[BM25 lexical search\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287767)
* **Solution:** Hybrid search
  * Combine:
    * **Semantic search (embeddings)** → captures meaning
    * **Lexical search (BM25 - Best Match 25)** → captures exact term matches [\[BM25 lexical search\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287767)

### BM25 algorithm flow

1. **Tokenize query** → split into terms
2. **Calculate term frequency** → count occurrences in documents
3. **Weight terms**
   * Rare terms → higher importance
   * Common terms → lower importance
4. **Rank documents**
   * Prefer documents containing high-weight (rare) terms [\[BM25 lexical search\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287767)

### Why BM25 matters in RAG

* Handles:
  * IDs
  * Technical keywords
  * Exact phrases
* Complements semantic search
* Produces more accurate hybrid results [\[BM25 lexical search\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287767)

## ❓ 2. Q\&A from the Page

### Q1. Why is semantic search alone insufficient?

Because it focuses on **conceptual similarity** and may return results that don’t contain the exact term being searched. [\[BM25 lexical search\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287767)

### Q2. What is BM25 used for?

BM25 is a **lexical search algorithm** used to find documents containing exact terms. [\[BM25 lexical search\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287767)

### Q3. How does BM25 prioritize results?

By giving **higher weight to rare terms** and ranking documents that contain those terms more frequently. [\[BM25 lexical search\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287767)

### Q4. What is hybrid search?

Running **semantic search + lexical search in parallel**, then merging results. [\[BM25 lexical search\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287767)

### Q5. When is BM25 especially useful?

For:

* Incident IDs
* Product codes
* Specific technical phrases [\[BM25 lexical search\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287767)

## 🔑 4. Key Takeaways

* Semantic search ≠ exact search → needs complement
* BM25:
  * Strong for **exact keyword matching**
  * Weak for understanding meaning
* Hybrid search = **best practice for RAG systems**
* Critical for enterprise scenarios:
  * Logs
  * Incident tracking
  * Financial/technical documents
