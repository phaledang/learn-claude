# 📘 1. Summary

**Page:** [Providing examples (Anthropic course)](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287746)

### ✅ Key points

* **Providing examples = powerful prompt engineering technique**
  * Also called **one-shot / multi-shot prompting**
  * You give **sample input → expected output** to guide Claude

* **Why it works**
  * Helps Claude understand:
    * Edge cases (e.g., sarcasm)
    * Desired output format
    * Tone/style expectations

* **Example use case: Sentiment analysis**
  * Basic prompt fails with sarcasm
  * Adding examples:
    * `"Great game tonight!" → Positive`
    * `"Oh yeah, I needed a flight delay!" → Negative`
  * Improves accuracy significantly [\[Providing examples\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287746)

* **Use XML tags for structure**
  * `<sample_input>`
  * `<ideal_output>`  
    → Makes mapping explicit for the model [\[Providing examples\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287746)

* **One-shot vs Multi-shot**
  * **One-shot** → 1 example (simple pattern)
  * **Multi-shot** → multiple examples (handle edge cases) [\[Providing examples\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287746)

* **When to use examples**
  * Complex formatting (e.g., JSON)
  * Ambiguous inputs
  * Edge/corner cases
  * Tone/style guidance [\[Providing examples\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287746)

* **Best practice from evaluations**
  * Use **highest scoring outputs from evals as examples**
  * Shows model what “perfect output” looks like [\[Providing examples\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287746)

* **Enhance with reasoning**
  * Don’t just show output → explain *why it is good* [\[Providing examples\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287746)

# ❓ 2. Q\&A

### Q1. What is one-shot prompting?

* Providing a **single example input-output pair** to define expected behavior.

### Q2. What is multi-shot prompting?

* Providing **multiple examples** to cover different scenarios and edge cases.

### Q3. Why are examples better than instructions alone?

* They **demonstrate expectations directly**, reducing ambiguity.

### Q4. Why are XML tags recommended?

* They clearly separate:
  * Input
  * Output  
    → improves model understanding.

### Q5. When should you use multi-shot instead of one-shot?

* When:
  * Inputs are ambiguous
  * Edge cases exist
  * Output variability is high

### Q6. Where should good examples come from?

* From **high-scoring outputs in prompt evaluations**.

### Q7. Why should you explain the example output?

* Helps Claude understand:
  * **Reasoning**
  * Not just format

# 🔑 4. Key Takeaways

* ✅ **Show, don’t tell** → examples outperform instructions
* ✅ Use **multi-shot** for reliability and edge cases
* ✅ Structure examples with **XML tags**
* ✅ Use **real evaluated outputs** as examples
* ✅ Add **reasoning context** to improve quality
* ✅ Critical for:
  * Formatting
  * Tone control
  * Ambiguity handling
