### Tool example (Document system)

* Documents stored in memory (Python dictionary)
* Two tools provided:

#### 1. Read document

```python
@mcp.tool(name="read_doc_contents")
def read_document(doc_id: str = Field(...)):
    return docs[doc_id]
```

 [\[Defining t...s with MCP\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287797)

#### 2. Edit document (find & replace)

```python
@mcp.tool(name="edit_document")
def edit_document(doc_id: str, old_str: str, new_str: str):
    docs[doc_id] = docs[doc_id].replace(old_str, new_str)
```

## 4. Key Takeaways

* MCP introduces a **standardized tool interface layer (protocol-based)**
* Python SDK simplifies MCP via:
  * Decorators → **define tools**
  * Type hints → **define schema**
* Eliminates:
  * Manual JSON schema complexity
  * Boilerplate code
* Promotes:
  * Clean tool design
  * Strong validation
  * Faster development
* Pattern aligns with:
  * **Tool registry → Agent execution → External capability integration**
