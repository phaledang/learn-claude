## What is Prompt Engineering?

* Process of **improving prompts iteratively** to get:
  * More **reliable**
  * Higher-quality outputs
* Not a one-time task → **continuous refinement cycle** [\[Prompt engineering\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287745)

## Iterative Improvement Cycle

1. **Set a goal** – define desired outcome
2. **Write initial prompt** – basic version
3. **Evaluate** – test against criteria
4. **Apply techniques** – improve prompt
5. **Re-evaluate** – validate improvement

👉 Repeat until performance is satisfactory [\[Prompt engineering\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287745)

***

## Evaluation Pipeline (Key Concept)

* Use a **PromptEvaluator** to:
  * Generate test datasets
  * Evaluate outputs using model grading
* Example control:
  * `max_concurrent_tasks` → manage API rate limits [\[Prompt engineering\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287745)

## Test Data Generation

* Define required inputs (e.g. height, weight, goal, restrictions)
* Auto-generate datasets for testing
* Use **small datasets (2–3 cases)** during development for faster iteration [\[Prompt engineering\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287745)

***

## Baseline Prompt Strategy

* Start with a **simple/naive prompt**
* Purpose:
  * Establish baseline
  * Measure improvement over iterations [\[Prompt engineering\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287745)

***

## Evaluation Criteria

* Define expected outputs explicitly:
  * Calories
  * Macronutrients
  * Meal details
* Helps model grading align with your requirements [\[Prompt engineering\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287745)

***

## Result Analysis

* Outputs include:
  * Numeric score
  * Detailed report with reasoning
* Low scores (e.g. \~2/10) are normal at start
* Use feedback to guide improvements [\[Prompt engineering\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287745)

***

## Key Principle

* Improve **one change at a time**
* Measure impact → build systematically [\[Prompt engineering\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287745)

# ❓ 2. Q\&A

### Q1: What is the main goal of prompt engineering?

**A:** To iteratively refine prompts to produce more reliable and higher-quality outputs.

### Q2: Why start with a simple prompt?

**A:** To create a baseline for measuring improvement.

### Q3: What is the purpose of evaluation?

**A:** To objectively measure prompt performance against defined criteria.

### Q4: Why use small datasets in early stages?

**A:** To speed up iteration cycles.

### Q5: What does the evaluation report provide?

**A:** Scores + reasoning explaining why outputs performed well or poorly.

### Q6: How should improvements be applied?

**A:** One change at a time to understand impact clearly.

# 🔑 4. Key Takeaways

* Prompt engineering = **iterative optimization**, not one-time writing
* Always **measure performance**, don’t rely on intuition
* Start simple → refine progressively
* Evaluation pipeline is critical for production-quality prompts
* Controlled experiments (one change at a time) give the best results
* Low initial scores are normal → improvement trend matters more
