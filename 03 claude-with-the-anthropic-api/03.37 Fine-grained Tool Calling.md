## 1. ✅ Summary

### Core concept

* Fine-grained tool calling is an **advanced mode of tool streaming in Claude API**.
* It allows **real-time streaming of tool arguments without waiting for validation**.

### Default behavior (without fine-grained)

* Claude streams tool arguments **in chunks**.
* However, the API:
  * Buffers chunks
  * Validates JSON
  * Sends only when a **complete top-level key-value pair** is ready
* This causes:
  * **Delays**
  * Burst-style updates instead of continuous streaming

### Fine-grained tool calling behavior

* Enables **immediate streaming of tool argument chunks**
* Disables:
  * ✅ JSON validation at API level
* Provides:
  * ✔ Faster updates
  * ✔ More granular streaming
  * ✔ Better responsiveness

### Trade-off

* ✅ Faster streaming
* ❌ Must handle **invalid JSON manually**

## 2. ❓ Q\&A

### Q1. What is `InputJsonEvent`?

* A streaming event type used when tools are invoked
* Contains:
  * `partial_json`: incremental chunk
  * `snapshot`: full accumulated JSON so far

***

### Q2. Why does default streaming feel delayed?

* Because the API:
  * Buffers chunks
  * Waits until **valid top-level key-value pair is complete**
  * Then sends all at once

***

### Q3. What does fine-grained tool calling change?

* Disables JSON validation
* Sends chunks **immediately as generated**

***

### Q4. How to enable fine-grained tool calling?

```python
run_conversation(
  messages,
  tools=[save_article_schema],
  fine_grained=True
)
```

***

### Q5. What risk comes with fine-grained mode?

* Claude may generate **invalid JSON**
* Example:

```json
"word_count": undefined
```

***

### Q6. How to handle invalid JSON?

```python
try:
    parsed_args = json.loads(chunk.snapshot)
except json.JSONDecodeError:
    print("Received invalid JSON, continuing...")
```

***

### Q7. When should you use fine-grained tool calling?

* When you need:
  * Real-time UI updates
  * Early partial results
  * High responsiveness

## 4. 🔑 Key Takeaways

* Tool streaming introduces **event-based JSON generation**

* Default behavior:
  * ✅ Safe (validated JSON)
  * ❌ Slower (buffering delays)

* Fine-grained tool calling:
  * ✅ Fast and responsive
  * ❌ Requires custom error handling

* Key design decision:
  * **Safe + simple → default**
  * **Fast + complex → fine-grained**

* Important implementation concern:
  * Always implement **robust JSON parsing logic**
