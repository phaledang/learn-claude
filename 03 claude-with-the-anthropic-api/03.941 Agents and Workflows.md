# 📘 1. Summary

**Page:** [Agents and Workflows](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287796)

### Core Concepts

* **Workflows vs Agents**
  * Both are strategies for solving tasks that **cannot be completed in a single LLM request**. [\[Agents and workflows\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287796)
  * **Workflow**
    * Predefined sequence of steps (deterministic)
    * Multiple Claude calls orchestrated by code
  * **Agent**
    * Goal-driven with tools
    * Claude decides how to achieve the objective dynamically [\[Agents and workflows\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287796)

* **Decision Rule**
  * Use **workflow** → when steps are clearly defined
  * Use **agent** → when tasks are unpredictable or open-ended [\[Agents and workflows\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287796)

***

### Example: Image → CAD Workflow

A structured pipeline:

1. Input image → Claude describes object
2. Generate CAD model (CadQuery)
3. Render output
4. Evaluate rendering vs original
5. Fix if needed (loop)

✅ This is a **workflow pattern**, not an agent  
✅ Because steps are known in advance [\[Agents and workflows\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287796)

***

### Evaluator–Optimizer Pattern

A key reusable workflow design:

* **Producer**
  * Creates output (e.g., CAD model)
* **Grader**
  * Evaluates output quality
* **Feedback Loop**
  * Sends improvements back
* **Iteration**
  * Repeat until acceptable result [\[Agents and workflows\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287796)

***

### Key Insight

* Workflows = **controlled orchestration**
* Agents = **autonomous reasoning**
* Patterns (like evaluator-optimizer) = **reusable design templates**

# ❓ 2. Q\&A

### Q1. What problem do workflows and agents solve?

They handle tasks that require **multiple steps or iterations**, rather than a single LLM call.

***

### Q2. What is the main difference between a workflow and an agent?

* Workflow → predefined steps
* Agent → goal + tools, decides steps dynamically

***

### Q3. When should workflows be used?

When the task is **well-understood and predictable**.

***

### Q4. When should agents be used?

When tasks are **uncertain, variable, or open-ended**.

***

### Q5. What is the evaluator–optimizer pattern?

A loop where:

* Output is generated
* Evaluated
* Improved iteratively until it meets criteria

***

### Q6. Why are workflow patterns important?

They provide **repeatable implementation recipes** for building reliable AI systems.

# 🔑 4. Key Takeaways

* Workflows and agents solve **multi-step AI problems**
* **Workflow**
  * Deterministic
  * Easier to control
  * Better for production pipelines
* **Agent**
  * Flexible and adaptive
  * Better for exploration and unknown tasks
* **Evaluator–Optimizer**
  * Powerful iterative improvement pattern
* Use **patterns** to standardize architecture design
