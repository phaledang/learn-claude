# 📘 1. Azure RAG Architecture (End-to-End)

## 1.1 Logical Flow

```
[User / App UI]
        ↓
[API Layer / Orchestrator]
        ↓
[Azure AI Search] ←→ [Embedding Model]
        ↓
[LLM (Claude / Azure OpenAI)]
        ↓
[Response]
```

### Mapping to your page flow

| Page Step         | Azure Component                          |
| ----------------- | ---------------------------------------- |
| Chunking          | Data pipeline (Functions / Data Factory) |
| Embeddings        | Azure OpenAI / Voyage                    |
| Vector DB         | Azure AI Search                          |
| Similarity search | Azure AI Search (vector + hybrid)        |
| Final prompt      | Orchestrator                             |
| Generation        | Claude / Azure OpenAI                    |

***

# 🧠 2. Two Core Pipelines

## 2.1 Data Pipeline (Offline)

This corresponds exactly to **Step 1–3 in your page**

### Flow

```
Data Sources → Ingestion → Chunk → Embedding → Index
```

### Azure Implementation

| Step           | Azure Service                         |
| -------------- | ------------------------------------- |
| Data ingestion | Blob Storage / SharePoint / Cosmos DB |
| Processing     | Azure Functions / Data Factory        |
| Chunking       | Custom logic                          |
| Embedding      | Azure OpenAI / Voyage                 |
| Storage        | Azure AI Search index                 |

✅ Microsoft explicitly defines:

* chunk → enrich → embed → store [\[learn.microsoft.com\]](https://learn.microsoft.com/en-us/azure/architecture/ai-ml/guide/rag/rag-solution-design-and-evaluation-guide)

***

## 2.2 Query Pipeline (Online)

This corresponds to **Step 4–6 in your page**

### Flow

```
User Query → Orchestrator → Search → Prompt → LLM → Response
```

### Azure Implementation

| Step         | Azure Service               |
| ------------ | --------------------------- |
| API layer    | App Service / AKS           |
| Orchestrator | Semantic Kernel / LangChain |
| Retrieval    | Azure AI Search             |
| LLM          | Claude or Azure OpenAI      |
| Response     | API → UI                    |

✅ Azure defines:

* orchestrator decides search
* top results injected into prompt
* sent to LLM [\[learn.microsoft.com\]](https://learn.microsoft.com/en-us/azure/architecture/ai-ml/guide/rag/rag-solution-design-and-evaluation-guide)

# 🔍 4. Retrieval Layer (Important Detail)

Azure AI Search is the **core of RAG**

### Capabilities

* Vector search (semantic similarity)
* Keyword search (BM25)
* Hybrid search (best practice)
* Semantic reranking [\[learn.microsoft.com\]](https://learn.microsoft.com/en-us/azure/search/retrieval-augmented-generation-overview)

### Why hybrid search matters

* Vector handles meaning
* Keyword handles exact terms
* Combined → better recall + precision [\[signisys.com\]](https://www.signisys.com/blog/azure-ai-search-the-complete-guide-to-enterprise-retrieval-and-rag-on-azure/)

***

# 🧩 5. Vector + Embedding Flow (Deep Alignment with Page)

## Equivalent to your page:

```
Text → Embedding → Similarity → Retrieval
```

### Azure implementation

```
Documents → embedding model → vector index
Query → embedding model → vector search
→ top-K chunks
```

* Query + documents are both converted to vectors
* Results retrieved via similarity search [\[learn.microsoft.com\]](https://learn.microsoft.com/en-us/azure/search/vector-search-overview)

# 🔄 7. Advanced Patterns (Architect-level)

## 7.1 Classic RAG (matches your page)

```
query → retrieve → generate
```

✅ Simple  
✅ Deterministic  
❌ Limited reasoning

***

## 7.2 Agentic RAG

```
query → plan → retrieve → refine → retrieve → generate
```

* Requires:
  * Semantic Kernel or LangGraph
* Solves:
  * multi-step queries
  * multi-source retrieval

***

## 7.3 Azure “On Your Data” Pattern

* Azure OpenAI directly integrates with AI Search
* Handles:
  * retrieval
  * prompt construction
  * citations

✅ Simplified architecture  
✅ Less control

***

# ⚖️ 8. Claude vs Azure OpenAI in Azure RAG

| Dimension      | Claude        | Azure OpenAI            |
| -------------- | ------------- | ----------------------- |
| Integration    | External      | Native                  |
| Tooling        | Manual        | Built-in (On Your Data) |
