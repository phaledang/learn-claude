
# 📘 1. Summary

**Source:** [Sending tool results – Anthropic Skilljar](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287752)

This page explains the **final step of the tool use workflow in Claude API**—how to execute a tool function after Claude requests it and then send the results back so Claude can produce a final response.

### 🔑 Key Concepts

* **Tool execution flow**
  1. Claude requests a tool using a `ToolUse` block
  2. Developer extracts parameters
  3. Executes the function
  4. Sends results back in a `ToolResult` block
  5. Claude generates the final response

* **Accessing tool inputs**
  * Parameters come from:
    ```python
    response.content[1].input
    ```
  * Must unpack dictionary into function:
    ```python
    get_current_datetime(**response.content[1].input)
    ```

* **Tool result block structure**
  * `tool_use_id`: links to the original tool call
  * `content`: output of function (string format)
  * `is_error`: indicates success or failure

* **Handling multiple tool calls**
  * Claude may request multiple tools in one response
  * Each tool has a unique ID
  * Responses must match IDs correctly

* **Follow-up request**
  * Add tool result into conversation history:
    ```python
    messages.append({
      "role": "user",
      "content": [{
        "type": "tool_result",
        "tool_use_id": response.content[1].id,
        "content": "15:04:22",
        "is_error": False
      }]
    })
    ```

* **Important requirement**
  * Must include **tool schema again** in the next API call
  * Claude uses it to understand previous tool references

* **Outcome**
  * Claude produces a **natural response using tool output**
  * Completes the tool-use loop ✅ [\[Sending tool results\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287752)

# ❓ 2. Q\&A

### Q1. What happens after Claude requests a tool?

After Claude sends a ToolUse block, you execute the function and return results in a ToolResult block. [\[Sending tool results\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287752)

### Q2. How do you get the tool parameters?

From `response.content[1].input`, which returns a dictionary of arguments. [\[Sending tool results\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287752)

### Q3. Why use `**` when calling the function?

Because the function expects keyword arguments instead of a dictionary. [\[Sending tool results\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287752)

### Q4. What is required in a ToolResult block?

* tool\_use\_id
* content
* is\_error [\[Sending tool results\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287752)

### Q5. How do you handle multiple tool calls?

Each tool call has a unique ID, and you must return results matching each ID. [\[Sending tool results\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287752)

### Q6. Where do you send the tool result?

Inside a **new user message** in the conversation history. [\[Sending tool results\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287752)

### Q7. Why must tool schema be included again?

Claude needs it to interpret tool references in the conversation history. [\[Sending tool results\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287752)

### Q8. What is the final step?

Send updated messages to Claude so it generates a natural response using tool output. [\[Sending tool results\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287752)

# 📌 4. Key Takeaways

* Tool use is a **closed loop workflow**: request → execute → return → final answer
* Always:
  * Extract parameters correctly
  * Execute function
  * Return results with correct `tool_use_id`
* Tool results must be:
  * Structured
  * Serialized as string
  * Linked to the correct tool call
* Multi-tool scenarios require **strict ID matching**
* Tool schema must be **included in every follow-up request**
* Final output = **Claude + real-time tool data** ✅
