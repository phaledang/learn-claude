
# 📘 1. Summary
Structure with XML tags
**Source:** [Structure with XML tags](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287741)

### 🔍 Key idea

XML tags help **structure prompts** so Claude can clearly understand different sections of content.

### 💡 Why it matters

* Without structure → Claude may **confuse instructions vs data**
* With structure → Claude can **parse intent correctly**

### ✅ Core concepts

* Use tags like:
  * `<sales_records>...</sales_records>`
  * `<my_code>...</my_code>`
  * `<docs>...</docs>`
* Tags act as **clear boundaries (delimiters)** between:
  * Instructions
  * Data
  * Context

### 🧠 Key principles

* Be **clear and explicit**
* Use **descriptive tag names**
  * ✅ `<athlete_information>`
  * ❌ `<data>`
* Helps especially when:
  * Large datasets
  * Mixed content (code + docs)
  * Complex prompts
  * Multiple variables

### 🧪 Example (from page)

```xml
<athlete_information>
- Height: 6'2"
- Weight: 180 lbs
- Goal: Build muscle
- Dietary restrictions: Vegetarian
</athlete_information>

Generate a meal plan based on the athlete information above.
```

👉 Result: Claude clearly understands that all items belong to one structured context.

# ❓ 2. Q\&A

### Q1. Why are XML tags useful in prompts?

**A:** They create clear boundaries so Claude can distinguish between instructions and data.

### Q2. What problem do XML tags solve?

**A:** They prevent confusion when prompts contain large or mixed content.

### Q3. Do XML tags need to follow strict standards?

**A:** No, you can create custom, descriptive tags.

### Q4. What makes a good tag name?

**A:** Specific and meaningful (e.g., `<sales_records>` instead of `<data>`).

### Q5. When should you use XML tags?

**A:**

* Large context inputs
* Mixed content types
* Complex prompts
