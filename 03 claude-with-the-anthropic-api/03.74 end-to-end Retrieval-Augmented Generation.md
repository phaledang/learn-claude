# 📘 1. Summary of the Page

**Source:** [The full RAG flow](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287764)

This page explains the **end-to-end Retrieval-Augmented Generation (RAG) pipeline**, showing how text data is processed and used to generate accurate responses with Claude.

### 🔑 Full RAG Flow (Step-by-step)

1. **Chunk Source Text**
   * Break large documents into smaller chunks.
   * Example:
     * Medical research section
     * Software engineering section [\[The full RAG flow\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287764)

2. **Generate Embeddings**
   * Convert each chunk into numerical vectors (embeddings).
   * Each dimension represents semantic meaning (e.g., medical vs software). [\[The full RAG flow\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287764)

3. **Normalization**
   * Embeddings are normalized (vector magnitude = 1).
   * Ensures consistent similarity comparison. [\[The full RAG flow\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287764)

4. **Store in Vector Database**
   * Store embeddings for fast similarity search.
   * This is a **preprocessing step (offline)**. [\[The full RAG flow\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287764)

5. **Process User Query**
   * Convert user query into embedding using the same model. [\[The full RAG flow\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287764)

6. **Find Similar Embeddings**
   * Use **cosine similarity** to retrieve closest chunks.
   * Higher score → more relevant. [\[The full RAG flow\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287764)

7. **Create Final Prompt**
   * Combine:
     * User question
     * Retrieved chunk(s)
   * Send to Claude to generate final answer. [\[The full RAG flow\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287764)

# ❓ 2. Q\&A from the Page

### Q1. What is the purpose of chunking in RAG?

**A:** To split large documents into smaller pieces so embeddings and retrieval are more efficient and accurate.

### Q2. What are embeddings?

**A:** Numerical representations of text capturing semantic meaning.

### Q3. Why normalize embeddings?

**A:** To ensure vectors have consistent scale for similarity comparison.

### Q4. What is a vector database used for?

**A:** To store embeddings and perform similarity search efficiently.

### Q5. How does RAG find relevant information?

**A:** By comparing query embeddings with stored embeddings using cosine similarity.

### Q6. What is cosine similarity?

**A:** A metric measuring how close two vectors are (range: -1 to 1).

### Q7. What is cosine distance?

**A:** $$1 - cosine\ similarity$$, used for easier interpretation.

### Q8. What is the final step in RAG?

**A:** Combine retrieved context with user query and send to Claude.

# 🧩 4. Key Takeaways

* RAG combines **retrieval + generation** for better accuracy.
* Pipeline has **2 phases**:
  * Offline: chunking, embeddings, storage
  * Online: query → retrieval → response
* **Embeddings are central** to semantic understanding.
* **Vector DB + cosine similarity** enable efficient search.
* Final answer quality depends on **retrieved context**.
