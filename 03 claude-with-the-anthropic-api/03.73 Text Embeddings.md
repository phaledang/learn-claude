## 1️⃣ Summary of the page

* In a **RAG (Retrieval-Augmented Generation) pipeline**, after chunking documents, the next step is to **find relevant chunks** for a user query. [\[Text embeddings\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287759)
* This is solved as a **search problem**, typically using **semantic search** instead of keyword matching. [\[Text embeddings\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287759)

### 🔹 Semantic Search

* Compares **meaning/context**, not exact words. [\[Text embeddings\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287759)
* Uses **text embeddings** for both:
  * User query
  * Document chunks [\[Text embeddings\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287759)

### 🔹 Text Embeddings

* A **numerical representation of text meaning**. [\[Text embeddings\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287759)

* Generated by:
  1. Inputting text into an embedding model
  2. Outputting a vector (list of numbers)
  3. Each number ranges roughly**-1 to +1** [\[Text embeddings\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287759)

* Key property:
  * Each dimension represents some **learned feature of text**
  * **Not interpretable by humans directly** [\[Text embeddings\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287759)

### 🔹 Tooling (Important)

* Anthropic **does NOT provide embeddings**
* Recommended provider: **VoyageAI** [\[Text embeddings\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287759)

### 🔹 Implementation Flow

* Install SDK
* Generate embedding via API
* Output = vector for similarity comparison [\[Text embeddings\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287759)

```python
def generate_embedding(text, model="voyage-3-large", input_type="query"):
    result = client.embed([text], model=model, input_type=input_type)
    return result.embeddings[0]
```

👉 Core idea: embeddings → compare vectors → retrieve relevant chunks.

## 2️⃣ Q\&A from the page

**Q1: What problem do text embeddings solve in RAG?**

* Finding the most relevant text chunks for a user query. [\[Text embeddings\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287759)

**Q2: What is semantic search?**

* A method that compares meaning instead of exact keywords. [\[Text embeddings\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287759)

**Q3: What is a text embedding?**

* A numerical vector representing the meaning of text. [\[Text embeddings\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287759)

**Q4: Are embedding dimensions interpretable?**

* No, their meaning is learned and not directly human-readable. [\[Text embeddings\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287759)

**Q5: Does Anthropic provide embedding models?**

* No, external providers like VoyageAI are used. [\[Text embeddings\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287759)

**Q6: What is the output of an embedding model?**

* A list of floating-point numbers. [\[Text embeddings\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287759)

## 4️⃣ Key Takeaways

* Embeddings = **foundation of semantic search + RAG**
* Transform text → **vector space**
* Retrieval = **vector similarity comparison**
* Dimensions:
  * meaningful for model
  * not interpretable for humans
* Separation of concerns:
  * **Claude → generation**
  * **VoyageAI → embedding**
* Critical pipeline:
  ```text
  Chunking → Embeddings → Similarity Search → Relevant Context → LLM
  ```
