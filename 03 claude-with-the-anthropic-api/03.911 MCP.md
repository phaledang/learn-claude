## 1. Summary

**Page:** *[Introducing MCP](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287780)*

* **Model Context Protocol (MCP)** is a communication layer that allows Claude to access **tools, prompts, and external resources** without requiring custom integration logic on the client side. [\[Introducing MCP\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287780)
  <img width="1280" height="720" alt="image" src="https://github.com/user-attachments/assets/fdbb7300-4288-4562-8626-62e10398152b" />

* The architecture consists of:
  * **MCP Client** (application/server)
  * **MCP Servers** (external services exposing tools/resources) [\[Introducing MCP\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287780)
* MCP shifts responsibility for:
  * Tool definition
  * Tool execution
  * Integration maintenance  
    → from the application to **specialized MCP servers** [\[Introducing MCP\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287780)
* Example: GitHub integration
  * Without MCP → must implement multiple API tools manually
  * With MCP → reuse prebuilt tools from an MCP server [\[Introducing MCP\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287780)
* MCP servers act as **wrappers for external services**, exposing reusable capabilities. [\[Introducing MCP\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287780)
* MCP is **not the same as tool use**:
  * Tool use = model calling functions
  * MCP = who provides/maintains those functions [\[Introducing MCP\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287780)

## 2. Q\&A

**Q1. What is MCP?**

* A protocol that provides Claude with structured access to tools and context without manual integration code. [\[Introducing MCP\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287780)

**Q2. What problem does MCP solve?**

* Eliminates the need to build, test, and maintain many tool integrations manually. [\[Introducing MCP\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287780)

**Q3. What are MCP Servers?**

* External services that package tools, prompts, and resources into reusable components. [\[Introducing MCP\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287780)

**Q4. How is MCP different from direct API calls?**

* Direct API → tool schemas + logic must be implemented manually
* MCP → tools already defined and ready to use [\[Introducing MCP\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287780)

**Q5. Is MCP the same as tool use?**

* No. MCP focuses on **tool provisioning**, not tool invocation. [\[Introducing MCP\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287780)

## 4. Key Takeaways

* MCP introduces a **standardized integration layer** for AI applications
* Enables **plug-and-play external capabilities** (GitHub, AWS, etc.)
* Reduces **engineering effort and maintenance cost**
* Encourages **reusable, shared tool ecosystems**
* Clear separation of concerns:
  * Client → orchestration
  * MCP server → capabilities
