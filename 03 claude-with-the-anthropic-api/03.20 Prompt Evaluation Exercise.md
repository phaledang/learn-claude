# ❓ 2. Q\&A from the Content

### Q1. What is the goal of prompt evaluation?

**A:** To systematically measure how well prompts perform and ensure consistent, high-quality model outputs.

### Q2. What are the key steps in a typical eval workflow?

**A:**

* Generate test datasets
* Run evaluations
* Apply grading (model-based or code-based)
* Analyze results

### Q3. Why are exercises important in prompt evaluation?

**A:** They provide hands-on experience in testing and improving prompts, which is essential for real-world applications.

### Q4. What is the role of test datasets?

**A:** They provide standardized inputs to evaluate prompt consistency and performance.

### Q5. What tools are used in this exercise?

**A:** A Jupyter notebook (`.ipynb`) for running evaluations and experiments.

# 🔑 4. Key Takeaways

* Prompt evaluation is **critical for production-grade AI systems**
* A structured eval workflow includes:
  * Dataset creation
  * Execution
  * Grading
  * Iteration
* Two main grading approaches:
  * **Model-based** → flexible, semantic
  * **Code-based** → deterministic, precise
* Hands-on exercises help:
  * Identify weaknesses in prompts
  * Improve reliability
* Evaluation is not optional → it’s **a core engineering discipline**
