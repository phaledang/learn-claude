## 🧠 Core Idea: Multi-Turn Tool Pattern

* Example scenario:
  1. User asks:*“What day is 103 days from today?”*
  2. Claude calls `get_current_datetime`
  3. App executes tool → returns result
  4. Claude calls `add_duration_to_datetime`
  5. App executes tool
  6. Claude provides final answer [\[Multi-turn...with tools\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287750)

👉 This creates a **loop of interaction between Claude and your backend**.

## 🔁 Conversation Loop Pattern

* Your application must:
  * Keep running until Claude stops requesting tools
  * Handle multiple tool calls automatically

### Example logic (pseudo):

```python
def run_conversation(messages):
    while True:
        response = chat(messages)
        add_assistant_message(messages, response)

        if response isn't asking for a tool:
            break

        tool_result_blocks = run_tools(response)
        add_user_message(messages, tool_result_blocks)

    return messages
```

### Q1: What is a multi-turn conversation with tools?

A pattern where Claude calls multiple tools sequentially before generating a final answer.

### Q2: Why is a conversation loop required?

To keep processing until Claude stops requesting additional tools.

### Q3: What happens after Claude requests a tool?

* Your server executes the tool
* Returns the result to Claude
* Claude may request another tool

### Q4: Why return full message objects instead of text?

To preserve structured content like tool calls and blocks.

### Q5: What is the role of message handlers?

They store and manage conversation history across turns.
