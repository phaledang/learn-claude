## 1. Summary

**Source:** [Building with extended thinking - Claude API Docs](https://platform.claude.com/docs/en/build-with-claude/extended-thinking#feature-compatibility)

* **Extended thinking** is a capability in Claude models that enables **deeper reasoning for complex tasks**, by generating internal “thinking” steps before producing the final answer. [\[Building w...e API Docs\]](https://platform.claude.com/docs/en/build-with-claude/extended-thinking#feature-compatibility)
* The API response includes **two types of content blocks**:
  * `thinking` → internal reasoning (summarized or omitted)
  * `text` → final response [\[Building w...e API Docs\]](https://platform.claude.com/docs/en/build-with-claude/extended-thinking#feature-compatibility)
* Modern models primarily use **adaptive thinking**, where the model **automatically decides when and how much to think**, controlled by an `effort` parameter instead of manual token budgets. [\[Building w...e API Docs\]](https://platform.claude.com/docs/en/build-with-claude/extended-thinking#feature-compatibility)
* Thinking visibility is configurable:
  * `summarized` → shows condensed reasoning
  * `omitted` → hides reasoning but preserves it internally (via encrypted signature) [\[Building w...e API Docs\]](https://platform.claude.com/docs/en/build-with-claude/extended-thinking#feature-compatibility)
* Supports **streaming**, **tool use**, and **multi-turn conversations**, but requires:
  * preserving thinking blocks during tool loops
  * consistent thinking mode per assistant turn [\[Building w...e API Docs\]](https://platform.claude.com/docs/en/build-with-claude/extended-thinking#feature-compatibility)
* Advanced capability: **interleaved thinking**
  * reasoning occurs between tool calls
  * enables complex multi-step agent workflows [\[Building w...e API Docs\]](https://platform.claude.com/docs/en/build-with-claude/extended-thinking#feature-compatibility)

```python
client = anthropic.Anthropic()

with client.messages.stream(
    model="claude-sonnet-4-6",
    max_tokens=16000,
    thinking={"type": "enabled", "budget_tokens": 10000},
    messages=[
        {
            "role": "user",
            "content": "What is the greatest common divisor of 1071 and 462?",
        }
    ],
) as stream:
    thinking_started = False
    response_started = False

    for event in stream:
        if event.type == "content_block_start":
            print(f"\nStarting {event.content_block.type} block...")
            # Reset flags for each new block
            thinking_started = False
            response_started = False
        elif event.type == "content_block_delta":
            if event.delta.type == "thinking_delta":
                if not thinking_started:
                    print("Thinking: ", end="", flush=True)
                    thinking_started = True
                print(event.delta.thinking, end="", flush=True)
            elif event.delta.type == "text_delta":
                if not response_started:
                    print("Response: ", end="", flush=True)
                    response_started = True
                print(event.delta.text, end="", flush=True)
        elif event.type == "content_block_stop":
            print("\nBlock complete.")
```
## 2. Q\&A

### Q1. What is extended thinking?

A mechanism that allows Claude to perform step-by-step reasoning before generating the final response. [\[Building w...e API Docs\]](https://platform.claude.com/docs/en/build-with-claude/extended-thinking#feature-compatibility)

### Q2. What is adaptive thinking?

A default mode where the model automatically determines when and how deeply to reason based on the request. [\[Building w...e API Docs\]](https://platform.claude.com/docs/en/build-with-claude/extended-thinking#feature-compatibility)

### Q3. What is the structure of a response?

A sequence of blocks:

```json
[
  { "type": "thinking", "thinking": "..." },
  { "type": "text", "text": "final answer" }
]
```

 [\[Building w...e API Docs\]](https://platform.claude.com/docs/en/build-with-claude/extended-thinking#feature-compatibility)

### Q4. What is the purpose of the `display` field?

Controls whether thinking is:

* shown as summarized text
* hidden but retained internally [\[Building w...e API Docs\]](https://platform.claude.com/docs/en/build-with-claude/extended-thinking#feature-compatibility)

### Q5. What is interleaved thinking?

A capability that allows reasoning between tool calls, enabling multi-step decision-making workflows. [\[Building w...e API Docs\]](https://platform.claude.com/docs/en/build-with-claude/extended-thinking#feature-compatibility)

### Q6. Why must thinking blocks be preserved during tool use?

To maintain reasoning continuity and allow the model to continue from previous steps. [\[Building w...e API Docs\]](https://platform.claude.com/docs/en/build-with-claude/extended-thinking#feature-compatibility)

### Q7. Does hiding thinking reduce cost?

No. Billing is based on full internal reasoning tokens, not visible output. [\[Building w...e API Docs\]](https://platform.claude.com/docs/en/build-with-claude/extended-thinking#feature-compatibility)

## 4. Key Takeaways

* Extended thinking improves **accuracy for complex tasks** (math, coding, reasoning).
* **Adaptive thinking is the new default**, replacing manual configuration.
* Thinking output is **separated from final response**, structured as blocks.
* Visibility options (`summarized` vs `omitted`) control UX, not cost.
* **Tool + thinking integration requires strict context handling** (preserve blocks).
* Interleaved thinking enables **agent-like workflows with reasoning loops**.
* Token usage includes **hidden reasoning**, important for cost optimization.
