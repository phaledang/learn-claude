# 📘 1. Summary

**Page:** *Response streaming*  
**URL:** <https://anthropic.skilljar.com/claude-with-the-anthropic-api/287734>

### 🔑 Key points

* **Problem**
  * Standard API responses may take **10–30 seconds**, causing poor UX (loading spinner, no feedback). [\[Response streaming\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287734)

* **Solution: Response Streaming**
  * Deliver output **incrementally (chunk-by-chunk)** as Claude generates it. [\[Response streaming\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287734)
  * Improves perceived speed and user experience.

* **How it works**
  * Claude sends:
    * Initial acknowledgment
    * Then a **stream of events containing partial results** [\[Response streaming\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287734)
  * Your backend forwards chunks to the UI → users see text **word-by-word**.

* **Stream Event Types**
  * `MessageStart` → Message begins
  * `ContentBlockStart` → New content block
  * `ContentBlockDelta` → **Actual text chunks**
  * `ContentBlockStop` → Block ends
  * `MessageDelta` → Message completion progress
  * `MessageStop` → Final end [\[Response streaming\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287734)

* **Implementation**
  * Enable streaming:
    ```python
    stream = client.messages.create(
        model=model,
        max_tokens=1000,
        messages=messages,
        stream=True
    )
    ```
  * Iterate events:
    ```python
    for event in stream:
        print(event)
    ```

* **Simplified Streaming (Recommended)**
  * SDK provides:
    ```python
    with client.messages.stream(...) as stream:
        for text in stream.text_stream:
            print(text, end="")
    ```
  * Only returns **clean text chunks** (no event parsing needed). [\[Response streaming\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287734)

* **Final Message Retrieval**
  * After streaming:
    ```python
    final_message = stream.get_final_message()
    ```
  * Use for:
    * DB storage
    * Post-processing [\[Response streaming\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287734)

# ❓ 2. Q\&A

### Q1. What problem does response streaming solve?

* It removes long waiting times (10–30s) and improves user experience by showing output progressively.

### Q2. What is the key benefit of streaming?

* Users see responses **in real time**, improving perceived performance.

### Q3. Which event contains actual generated text?

* `ContentBlockDelta`

### Q4. What does `stream=True` do?

* Enables chunked streaming instead of waiting for the full response.

### Q5. Why use `text_stream` instead of raw events?

* It simplifies handling by returning only text (no need to parse events).

### Q6. How do you get the full response after streaming?

* Use `stream.get_final_message()`.

# 📌 4. Key Takeaways

* Streaming = **better UX + perceived speed**
* Core concept:
  * **events → chunks → UI rendering**
* `ContentBlockDelta` = most important event
* Prefer:
  * ✅ `text_stream` (simplified)
  * ✅ `get_final_message()` (post-processing)
* Best practice:
  * Combine **real-time streaming + final message storage**
