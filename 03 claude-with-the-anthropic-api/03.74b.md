# 📘 1. Mapping “Full RAG Flow” → Implementations

## 1.1 Step Mapping

| RAG Step            | Claude API                         | LangChain               | LangGraph         |
| ------------------- | ---------------------------------- | ----------------------- | ----------------- |
| Chunking            | Manual preprocessing               | `TextSplitter`          | Same as LangChain |
| Embeddings          | Claude / external (Voyage, OpenAI) | `Embeddings` interface  | Same              |
| Storage             | External vector DB                 | `VectorStore`           | Same              |
| Query embedding     | Same embedding API                 | handled via retriever   | Same              |
| Retrieval           | manual similarity search           | `Retriever` abstraction | Node in graph     |
| Prompt construction | manual                             | `prompt template`       | Node              |
| Generation          | Claude messages API                | `LLMChain` / `Runnable` | Final node        |

# 🧠 2. Claude API – Full RAG Pipeline

## 2.1 Flow (explicit implementation)

```python
from anthropic import Anthropic
import voyageai

# Init
client = Anthropic(api_key="...")
voyage = voyageai.Client(api_key="...")

# Step 1-2: Chunk + embeddings (offline)
documents = [
    "Medical: This year saw significant strides in XDR-47...",
    "Software: This division studied infection vectors..."
]

doc_embeddings = voyage.embed(
    documents,
    model="voyage-2"
).embeddings

# Step 3: Store (simulate)
vector_store = list(zip(documents, doc_embeddings))


# Step 4: Query embedding
query = "What did the software engineering dept do?"
query_embedding = voyage.embed(
    [query],
    model="voyage-2"
).embeddings[0]


# Step 5: Cosine similarity
import numpy as np

def cosine_sim(a, b):
    return np.dot(a, b)

top_doc = max(
    vector_store,
    key=lambda x: cosine_sim(x[1], query_embedding)
)[0]


# Step 6: Final prompt
response = client.messages.create(
    model="claude-3",
    max_tokens=300,
    messages=[
        {
            "role": "user",
            "content": f"""
<question>{query}</question>
<context>{top_doc}</context>
"""
        }
    ]
)

print(response.content)
```

### Characteristics

* Full control of:
  * embeddings
  * retrieval
  * prompt construction
* Matches page logic directly:
  * chunk → embed → similarity → prompt → generation [\[The full RAG flow\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287764)

# 🔗 3. LangChain Equivalent

## 3.1 Pipeline abstraction

```python
from langchain.vectorstores import FAISS
from langchain.embeddings import VoyageEmbeddings
from langchain.llms import Anthropic
from langchain.chains import RetrievalQA

# Embedding
embedding = VoyageEmbeddings()

# Vector store
db = FAISS.from_texts(
    documents,
    embedding
)

# Retriever
retriever = db.as_retriever()

# LLM
llm = Anthropic(model="claude-3")

# RAG chain
qa = RetrievalQA.from_chain_type(
    llm=llm,
    retriever=retriever
)

result = qa.run("What did the software engineering dept do?")
```

### Abstraction mapping

| Concept  | LangChain abstraction |
| -------- | --------------------- |
| Step 1–3 | `FAISS.from_texts()`  |
| Step 4–5 | `retriever`           |
| Step 6   | `RetrievalQA`         |

### Characteristics

* Removes manual similarity logic
* Encapsulates:
  * embedding
  * retrieval
  * prompt composition
* Reduced control vs Claude native pipeline

# 🔄 4. LangGraph (Agent-level RAG)

## 4.1 Graph-based pipeline

```python
from langgraph.graph import StateGraph

def retrieve(state):
    query = state["query"]
    docs = retriever.get_relevant_documents(query)
    return {"docs": docs}

def generate(state):
    context = state["docs"]
    response = llm.invoke({
        "question": state["query"],
        "context": context
    })
    return {"answer": response}

graph = StateGraph()

graph.add_node("retrieve", retrieve)
graph.add_node("generate", generate)

graph.set_entry_point("retrieve")
graph.add_edge("retrieve", "generate")

app = graph.compile()

result = app.invoke({"query": "What did the software team do?"})
```

***

## 4.2 Extension over LangChain

| Capability               | LangChain | LangGraph |
| ------------------------ | --------- | --------- |
| Linear pipeline          | ✅         | ✅         |
| Branching                | ❌         | ✅         |
| Multi-step reasoning     | Limited   | ✅         |
| Retry logic              | ❌         | ✅         |
| Stateful workflows       | ❌         | ✅         |
| Multi-retriever strategy | Hard      | Native    |

# 🚀 6. Extension Patterns (Advanced RAG)

## 6.1 Multi-Index RAG

* Problem:
  * multiple domains (finance, engineering, medical)
* Implementation:
  * Claude → manual routing logic
  * LangChain → router chain
  * LangGraph → routing node

***

## 6.2 Iterative Retrieval (Agentic RAG)

Flow:

```
query → retrieve → evaluate → refine query → retrieve → generate
```

* Claude: manual loop
* LangChain: not native
* LangGraph: native workflow pattern

***

## 6.3 Hybrid Search

Combine:

* BM25 (keyword)
* embeddings (semantic)

Claude:

```python
final_score = 0.7 * cosine + 0.3 * bm25
```

LangGraph:

* parallel nodes:
  * vector search
  * keyword search
* merge results
