 **LangChain / LangGraph equivalent implementation** of the *Claude Web Search Tool*

# 🧩 1. Mapping Concept

| Claude Native Tool         | LangChain Equivalent                           | LangGraph Equivalent                |
| -------------------------- | ---------------------------------------------- | ----------------------------------- |
| Built-in web\_search tool  | External tool wrapper (Tavily / Serper / Bing) | Tool node in graph                  |
| Automatic orchestration    | Agent decides tool usage                       | Graph controls execution flow       |
| Structured response blocks | Tool + output parser                           | Node outputs + state                |
| Citations included         | Must be implemented manually                   | Custom node for citation formatting |

***

# ⚙️ 2. LangChain Implementation

## Tool definition (Web Search)

```python
from langchain.tools import Tool
from langchain_community.tools import TavilySearchResults

search = TavilySearchResults(max_results=5)

web_search_tool = Tool(
    name="web_search",
    func=search.run,
    description="Search the web for current or factual information"
)
```

***

## Agent setup (Claude model)

```python
from langchain.chat_models import ChatAnthropic
from langchain.agents import initialize_agent, AgentType

llm = ChatAnthropic(
    model="claude-3-opus-20240229",
    temperature=0
)

agent = initialize_agent(
    tools=[web_search_tool],
    llm=llm,
    agent=AgentType.OPENAI_FUNCTIONS,   # tool-calling style
    verbose=True
)
```

***

## Execution

```python
result = agent.run("Latest developments in quantum computing")
print(result)
```

***

## Notes (vs Claude native)

* Tool execution:
  * Claude → automatic
  * LangChain → agent reasoning loop
* Citations:
  * Must parse and format manually
* Domain restriction:
  * Must configure in tool provider (not built-in schema)

***

# 🧭 3. LangGraph Implementation (Agent-level control)

## State definition

```python
from typing import TypedDict, List

class State(TypedDict):
    question: str
    search_results: List[str]
    final_answer: str
```

***

## Tool node

```python
def web_search_node(state: State):
    results = search.run(state["question"])
    return {"search_results": results}
```

***

## LLM node

```python
def generate_answer_node(state: State):
    prompt = f"""
    Question: {state['question']}
    Search Results: {state['search_results']}
    Generate answer with citations.
    """
    response = llm.invoke(prompt)
    return {"final_answer": response}
```

***

## Graph definition

```python
from langgraph.graph import StateGraph

builder = StateGraph(State)

builder.add_node("search", web_search_node)
builder.add_node("generate", generate_answer_node)

builder.set_entry_point("search")
builder.add_edge("search", "generate")

graph = builder.compile()
```

***

## Execution

```python
result = graph.invoke({
    "question": "Latest developments in quantum computing"
})

print(result["final_answer"])
```

***

# 🔒 4. Domain Restriction (Equivalent to `allowed_domains`)

## Not native → implement manually

```python
def filter_domains(results, allowed_domains):
    return [
        r for r in results
        if any(domain in r for domain in allowed_domains)
    ]
```

Apply inside tool node:

```python
filtered = filter_domains(results, ["nih.gov"])
```

# ✅ 6. Comparison Summary

| Capability        | Claude Native                | LangChain        | LangGraph            |
| ----------------- | ---------------------------- | ---------------- | -------------------- |
| Setup effort      | Minimal (schema)             | Medium           | High                 |
| Search execution  | Automatic                    | Agent-driven     | Explicit node        |
| Multi-step search | Built-in                     | Needs agent loop | Needs graph design   |
| Citations         | Built-in                     | Manual           | Manual               |
| Domain filtering  | Built-in (`allowed_domains`) | External config  | Custom logic         |
| Transparency      | High (structured blocks)     | Medium (logs)    | High (state + nodes) |
| Control level     | Low                          | Medium           | High                 |

***

# 🎯 7. Key Takeaways (Architect View)

* Claude web search tool = **fully managed tool orchestration**
* LangChain:
  * Suitable for **quick agent integration**
  * Less control over execution flow
* LangGraph:
  * Required for:
    * Deterministic workflows
    * Multi-step reasoning pipelines
    * Controlled search → refine → answer loops
* Feature gap vs Claude:
  * No native citations
  * No built-in domain restriction
  * No automatic multi-search behavior

***

# 🔗 8. Extension Pattern (Recommended)

### Hybrid architecture (best practice)

* Use:
  * Claude native tool → when simplicity + speed needed
  * LangGraph → when:
    * multi-step reasoning
    * RAG + search hybrid
    * deterministic agent workflows
