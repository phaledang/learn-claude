### Key idea

* When Claude needs **multiple tools to answer one query**, it creates a **multi-turn conversation flow**.
* Your application must **loop through tool calls automatically** until Claude produces a final answer.

### Example scenario

* User asks:*“What day is 103 days from today?”*
* Claude:
  1. Calls `get_current_datetime`
  2. Then calls `add_duration_to_datetime`
  3. Finally returns the answer  
     👉 This requires **multiple sequential tool calls**, not a single step. [\[Multi-turn...with tools\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287750)

***

### Core Pattern: Multi-Turn Tool Workflow

1. User sends request
2. Claude requests a tool
3. App executes tool
4. App sends result back
5. Claude may request another tool
6. Repeat until final answer

👉 This is called a **conversation loop**. [\[Multi-turn...with tools\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287750)

***

### Conversation Loop (Pseudo Code)

```python
def run_conversation(messages):
    while True:
        response = chat(messages)
        add_assistant_message(messages, response)

        if response isn't asking for a tool:
            break

        tool_result_blocks = run_tools(response)
        add_user_message(messages, tool_result_blocks)

    return messages
```

👉 Loop continues until no tool is requested. [\[Multi-turn...with tools\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287750)

### Key Implementation Changes

#### 1. Flexible Message Handling

* Support:
  * raw text
  * structured blocks
  * full message objects

```python
def add_user_message(messages, message):
    user_message = {
        "role": "user",
        "content": message.content if isinstance(message, Message) else message
    }
    messages.append(user_message)
```

***

#### 2. Chat Function Must Return Full Message

* Not just text → return full structured object
* Include tools + system prompt support

```python
def chat(messages, system=None, temperature=1.0, tools=None):
    params = {...}
    if tools:
        params["tools"] = tools
    if system:
        params["system"] = system

    return client.messages.create(**params)
```

***

#### 3. Extract Text from Message

* Messages contain multiple blocks → need helper

```python
def text_from_message(message):
    return "\n".join(
        [block.text for block in message.content if block.type == "text"]
    )
```

# ❓ 2. Q\&A

### Q1. What is a multi-turn conversation with tools?

**A:** A flow where Claude makes multiple tool calls sequentially before producing a final answer.

***

### Q2. Why is a conversation loop required?

**A:** Because Claude may request more than one tool, and the app must continue execution until no more tools are needed.

***

### Q3. What triggers the loop to stop?

**A:** When Claude returns a response that does **not include a tool request**.

***

### Q4. Why return full message objects instead of text?

**A:** To preserve:

* tool calls
* structured blocks
* metadata

***

### Q5. What is the role of `text_from_message`?

**A:** Extract readable text from structured response blocks.
