**Multi‑Index RAG pipeline**
## 1. Summary 

**Source:** [Open this page](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287766)

[005_hybrid.ipynb](005_hybrid.ipynb)

* Combines **semantic search (vector embeddings)** and **lexical search (BM25)** into a **single hybrid retrieval pipeline**
* Introduces a **Retriever layer** that:
  * Sends queries to multiple indexes
  * Collects results
  * Merges them using **Reciprocal Rank Fusion (RRF)**
* Key idea: unify different retrieval methods via **common interface (`add_document`, `search`)** → easy extensibility
* **RRF formula** used to rank results fairly across different scoring systems:
  ```
  RRF_score(d) = Σ(1 / (k + rank_i(d)))
  ```
* Hybrid approach improves relevance:
  * Fixes issues where vector search alone returns semantically similar but incorrect results
  * Combines **exact keyword match (BM25)** + **semantic similarity (embeddings)**
* Architecture is **modular**:
  * New indexes can be added (keyword, graph, domain-specific) without changing the Retriever logic [\[A Multi-In...G pipeline\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287766)

## 2. Q\&A from the Page

**Q1. Why use a Multi-Index RAG pipeline?**

* To combine strengths of different retrieval methods (semantic + lexical) and overcome their individual limitations [\[A Multi-In...G pipeline\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287766)

**Q2. What is the role of the Retriever?**

* Acts as a coordinator that queries multiple indexes and merges results [\[A Multi-In...G pipeline\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287766)

**Q3. Why is merging results non-trivial?**

* Different search methods use different scoring systems → need normalization (RRF) [\[A Multi-In...G pipeline\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287766)

**Q4. What does Reciprocal Rank Fusion do?**

* Combines rankings from multiple sources into a single score using rank positions [\[A Multi-In...G pipeline\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287766)

**Q5. Why maintain a consistent API across indexes?**

* Enables loose coupling and easy integration of new retrieval methods [\[A Multi-In...G pipeline\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287766)

**Q6. What problem does hybrid search solve?**

* Improves ranking accuracy when semantic search alone gives misleading results [\[A Multi-In...G pipeline\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287766)

## 4. Key Takeaways

* Hybrid RAG = **Semantic + Lexical = Better retrieval quality**
* **Retriever abstraction** is the core design pattern
* **RRF is critical** to combine rankings fairly
* **Consistent interface design** enables plug-and-play indexes
* Architecture is:
  * Modular
  * Extensible
  * Production-friendly
