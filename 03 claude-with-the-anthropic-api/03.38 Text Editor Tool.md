
# 📘 1. Summary

**Source:** [The text edit tool](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287760)

The **Text Editor Tool** is a **built-in tool in Claude API** that enables the model to interact with files and directories similarly to a standard code editor.

### Key points:

* Claude **already knows the tool schema**, so you **don’t need to define it fully**
* However, you **must implement the backend functions** that execute file operations
* This tool significantly extends Claude’s capability → acts like a **software engineer**

### Supported capabilities:

* View file or directory contents
* Read specific line ranges
* Replace text in files
* Create new files
* Insert text at specific positions
* Undo recent changes

### Architecture model:

```
Claude → Requests file operation (via tool schema)
Your App → Executes the operation (actual code)
```

### Important nuance:

* Text editor tool = **half-built tool**
  * ✅ Schema knowledge → provided by Claude
  * ❌ Execution logic → must be implemented by developer

### Schema handling:

* Only a **small stub schema** is required
* Claude expands it internally based on model version

Example:

```python
def get_text_edit_schema(model):
    if model.startswith("claude-3-7-sonnet"):
        return {
            "type": "text_editor_20250124",
            "name": "str_replace_editor",
        }
```

### Typical workflow:

Example request:

```
Open ./main.py and summarize it
```

Execution:

1. Claude calls text editor tool → read file
2. Your backend returns file content
3. Claude summarizes

Extended example:

```
Modify main.py and create test.py
```

Claude:

* Reads file
* Rewrites content
* Creates new file

### Why this matters:

* Enables **programmatic file editing**
* Useful for:
  * AI coding assistants in custom apps
  * Automated refactoring systems
  * Environments without IDEs
  * Agent workflows (multi-step file manipulation)

# ❓ 2. Q\&A

### Q1. Is the text editor tool fully implemented by Claude?

No. Claude provides the schema, but developers must implement file operations.

### Q2. What is the key difference from normal tools?

You don’t define full schema—only a stub.

### Q3. What does the tool enable Claude to do?

Read, create, edit, insert, and undo file operations.

### Q4. How does Claude interact with files?

Through tool calls that your backend executes.

### Q5. Why is this tool powerful?

It lets Claude behave like a software engineer.
