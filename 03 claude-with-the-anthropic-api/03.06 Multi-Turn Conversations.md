# 📘 1. Summary

**Source:** [Multi-Turn Conversations (Anthropic)](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287735)

### 🔑 Core Concept

* Claude API is **stateless** → it **does NOT store conversation history**.
* Each request is **independent**.
* To build conversations, **you must manage state manually**.

### ⚠️ Problem

* If you send:
  * Q1: “What is quantum computing?” ✅
  * Q2: “Write another sentence” ❌
* Claude **won’t understand context** → produces unrelated output.

### ✅ Solution: Multi-Turn Conversation Pattern

You must:

* Maintain a **message list (conversation history)**
* Send **full history on every request**

### 🔄 Conversation Flow

1. Add user message
2. Call Claude API
3. Store Claude response
4. Add follow-up user message
5. Send **entire history again**

### 🧩 Helper Functions Pattern (Python-style)

```python
def add_user_message(messages, text):
    messages.append({"role": "user", "content": text})

def add_assistant_message(messages, text):
    messages.append({"role": "assistant", "content": text})

def chat(messages):
    message = client.messages.create(
        model=model,
        max_tokens=1000,
        messages=messages,
    )
    return message.content[0].text
```

### 🚀 Example Usage

```python
messages = []

add_user_message(messages, "Define quantum computing in one sentence")

answer = chat(messages)
add_assistant_message(messages, answer)

add_user_message(messages, "Write another sentence")

final_answer = chat(messages)
```

👉 Result: Claude now understands context because **full history is provided** [\[Multi-Turn...versations\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287735)

# ❓ 2. Q\&A

### Q1: Does Claude remember previous messages automatically?

* ❌ No — it is **stateless** [\[Multi-Turn...versations\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287735)

### Q2: Why do follow-up questions fail sometimes?

* Because **context is not included in the request** [\[Multi-Turn...versations\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287735)

### Q3: How to enable multi-turn conversations?

* Maintain and resend **complete message history** [\[Multi-Turn...versations\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287735)

### Q4: What roles exist in messages?

* `user`
* `assistant`

### Q5: What must be sent in every API call?

* ✅ The **entire conversation history**

# 📌 4. Key Takeaways

* Claude API is **stateless by design**
* Multi-turn conversations require **manual state management**
* Always:
  * Store messages
  * Append new ones
  * Send full history
* Helper functions simplify implementation
* This pattern is **critical for building chat apps / agents**
