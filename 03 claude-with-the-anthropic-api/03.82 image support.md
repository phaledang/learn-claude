### Core concept

* Claude supports **vision capabilities** → images can be included in prompts and analyzed.
* Supports tasks such as:
  * Object description
  * Counting
  * Image comparison
  * Complex visual reasoning

### Input structure

* Messages contain **multiple blocks**:
  * `image` block (base64 or URL)
  * `text` block (instruction/prompt)
* Example flow:

```
User message → (image + text)
Claude response → text analysis
```

### Image constraints

* Max **100 images / request**
* Max **5MB per image**
* Size limits:
  * Single image: up to 8000px
  * Multiple images: up to 2000px
* Token cost:
  ```
  tokens = (width × height) / 750
  ```
```python
with open("image.png", "rb") as f:
    image_bytes = base64.standard_b64encode(f.read()).decode("utf-8")

add_user_message(messages, [
    # Image Block
    {
        "type": "image",
        "source": {
            "type": "base64",
            "media_type": "image/png",
            "data": image_bytes,
        }
    },
    # Text Block
    {
        "type": "text",
        "text": "What do you see in this image?"
    }
])
```
### Prompting for images (critical insight)

* Same principles as text prompting:
  * Simple prompt → low accuracy
  * Structured prompt → high accuracy

### Techniques to improve results

* Provide **step-by-step reasoning**
* Use **methodology**
* Add **one-shot examples**
* Break tasks into smaller steps

### Example pattern (recommended)

* Instead of:
  * “How many marbles?”
* Use:
  * Step-by-step counting
  * Verification method

### Real-world use case

* **Fire risk assessment (insurance)**
  * Analyze satellite images
  * Detect:
    * Tree overhang
    * Vegetation density
    * Emergency access
  * Output structured risk score
