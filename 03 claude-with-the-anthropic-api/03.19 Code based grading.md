

# 🔎 1. Summary

**Page:** *Code based grading* (Anthropic – Claude API course)

### Key concept

* **Code-based grading** is used to evaluate AI-generated code beyond just readability or correctness of explanation.
* It ensures:
  * ✅ Output format is correct (Python / JSON / Regex only)
  * ✅ Syntax is valid and parsable
  * ✅ Task is correctly addressed (combined with model grading)

### How grading works

* Two evaluation components:
  * **Code grader**
    * Validates format
    * Checks syntax
  * **Model grader**
    * Checks task correctness & relevance
* Final score = combination of both

### Syntax validation approach

* Use parsing functions for each format:
  * JSON → `json.loads`
  * Python → `ast.parse`
  * Regex → `re.compile`
* Scoring:
  * Valid → score 10
  * Invalid → score 0 [\[Code based grading\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287737)

### Dataset requirement

* Each test case must include expected format:

```json
{
  "task": "Create a Python function to validate an AWS IAM username",
  "format": "python"
}
```

 [\[Code based grading\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287737)

### Prompt improvement techniques

* Be explicit:
  * “Respond only with Python/JSON/Regex”
  * “No explanations or comments”
* Use prefilled assistant message:

````python
add_assistant_message(messages, "```code")
````

→ encourages raw code output [\[Code based grading\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287737)

### Final scoring

```python
score = (model_score + syntax_score) / 2
```

* Combines quality + correctness equally
* Can adjust weighting based on use case [\[Code based grading\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287737)

### Purpose

* Enables **objective measurement** of prompt performance
* Supports iterative prompt engineering improvement [\[Code based grading\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287737)

# ❓ 2. Q\&A

### Q1. Why do we need code-based grading?

* To verify that AI-generated code is not only correct logically but also valid in format and syntax.

### Q2. What are the two main components of evaluation?

* Code grader (format + syntax)
* Model grader (task correctness)

### Q3. How is syntax validated?

* By parsing output using language-specific libraries:
  * `json.loads`, `ast.parse`, `re.compile`

### Q4. What happens if syntax fails?

* It returns a score of **0**

### Q5. Why include “format” in dataset?

* To tell the grader which validator to use

### Q6. How can prompts improve code output quality?

* Specify strict format rules
* Avoid explanations
* Pre-fill assistant code block

### Q7. How is the final score calculated?

* Average of model score and syntax score

# 📌 4. Key Takeaways

* Code evaluation must go beyond “looks correct”
* Always validate:
  * Format
  * Syntax
  * Task completion
* Use automated parsing for reliable scoring
* Prompt quality directly impacts evaluation scores
* Combine **subjective (model)** + **objective (syntax)** grading
* Enables:
  * Measurable prompt improvements
  * Scalable evaluation pipelines
