**Core concept:**

* *Extended Thinking* is an advanced reasoning feature in Claude that allows the model to **explicitly generate intermediate reasoning (“thinking”) before producing the final answer**. [\[Extended thinking\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287773)

**Key characteristics:**

* Output becomes **structured**:
  * `thinking` → intermediate reasoning steps
  * `final answer` → the actual response [\[Extended thinking\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287773)
* Acts like “scratch paper”:
  * Improves reasoning quality
  * Provides transparency into how the answer was derived [\[Extended thinking\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287773)

**Benefits:**

* Better handling of **complex reasoning tasks**
* Increased **accuracy on difficult problems**
* Improved **transparency/debugging** [\[Extended thinking\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287773)

**Trade-offs:**

* Higher cost (extra tokens for reasoning)
* Increased latency
* More complex response handling in applications [\[Extended thinking\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287773)

**When to use:**

* Apply only after:
  1. Prompt optimization is already done
  2. Evaluation shows insufficient accuracy
* Decision should be driven by **prompt evaluation results** [\[Extended thinking\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287773)

**Implementation concept:**

* Enable via API parameter:

```python
params["thinking"] = {
    "type": "enabled",
    "budget": thinking_budget
}
```
```python
client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=16000,
    thinking={"type": "enabled", "budget_tokens": 10000},
    messages=[
        {
            "role": "user",
            "content": "Are there an infinite number of prime numbers such that n mod 4 == 3?",
        }
    ],
)

# The response contains summarized thinking blocks and text blocks
for block in response.content:
    if block.type == "thinking":
        print(f"\nThinking summary: {block.thinking}")
    elif block.type == "text":
        print(f"\nResponse: {block.text}")
```
```python
client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=16000,
    thinking={
        "type": "enabled",
        "budget_tokens": 10000,
        "display": "omitted",
    },
    messages=[
        {"role": "user", "content": "What is 27 * 453?"},
    ],
)

for block in response.content:
    if block.type == "thinking":
        if block.thinking:
            print(f"Thinking: {block.thinking}")
        else:
            print("Thinking: [omitted]")
    elif block.type == "text":
        print(f"Response: {block.text}")
```

* Requires:
  * `thinking_budget >= 1024`
  * `max_tokens > thinking_budget` [\[Extended thinking\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287773)

**Security model:**

* Includes **cryptographic signature**
  * Ensures reasoning is not modified
* Supports **redacted thinking**
  * Encrypted reasoning retained for context reuse
  * Triggered by safety systems [\[Extended thinking\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287773)

## ❓ 2. Q\&A from the page

**Q1: What is Extended Thinking?**  
A: A feature that enables Claude to generate intermediate reasoning before producing the final answer. [\[Extended thinking\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287773)

**Q2: How does the response structure change?**  
A: It returns both the reasoning (“thinking”) and the final answer instead of a single text output. [\[Extended thinking\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287773)

**Q3: What are the main benefits?**  
A: Better reasoning, higher accuracy, and transparency. [\[Extended thinking\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287773)

**Q4: What are the trade-offs?**  
A: Increased cost, latency, and implementation complexity. [\[Extended thinking\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287773)

**Q5: When should Extended Thinking be enabled?**  
A: Only after prompt optimization and evaluation show the need for better reasoning. [\[Extended thinking\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287773)

**Q6: What is the thinking budget?**  
A: The maximum number of tokens allocated for reasoning (minimum 1024). [\[Extended thinking\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287773)

**Q7: What is redacted thinking?**  
A: Encrypted reasoning returned when safety filters are triggered. [\[Extended thinking\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287773)

## 🔑 4. Key Takeaways

* Extended Thinking = **explicit reasoning layer before final output**
* Improves **accuracy for complex tasks**, not necessary for simple ones
* Must be **evaluation-driven**, not default-enabled
* Introduces **cost + latency trade-offs**
* Requires **structured response handling in applications**
* Includes **security (signature) + redacted reasoning support**
* Best practice:
  ```
  Prompt Optimization → Evaluation → Enable Thinking (if needed)
  ```
