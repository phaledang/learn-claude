# 📘 1. Summary

**Page:** *A typical eval workflow*  
(From Anthropic Claude API course)

This page explains a **standard prompt evaluation workflow** used to systematically improve prompts using measurable performance.

### 🔑 Core Idea

Instead of guessing whether a prompt is good, you:

* Test it with datasets
* Score the results
* Iteratively improve it

***

## 🧩 5-Step Eval Workflow

### 1. Draft a Prompt

* Start with a **basic prompt (baseline)**
* Example:
  ```
  Please answer the user's question: {question}
  ```

***

### 2. Create an Eval Dataset

* Prepare **sample inputs** representing real use cases
* Example dataset:
  * “What's 2+2?”
  * “How do I make oatmeal?”
  * “How far away is the Moon?”
* Can scale from **few → thousands of records**
* Can be created manually or generated using Claude

***

### 3. Feed Through Claude

* Combine prompt + dataset → send to Claude
* Collect responses
* Example:
  * Input: “What’s 2+2?”
  * Output: “2 + 2 = 4”

***

### 4. Feed Through a Grader

* Evaluate outputs with a **scoring mechanism**
* Usually numeric (e.g., **1–10 scale**)
* Example scores:
  * Math: 10 ✅
  * Oatmeal: 4 ❌
  * Moon distance: 9 ✅

👉 Compute **average score**:

```
(10 + 4 + 9) / 3 = 7.66
```

***

### 5. Change Prompt & Repeat

* Improve prompt (e.g., add instruction):
  ```
  Please answer the user's question: {question}
  Answer the question with ample detail
  ```
* Re-run evaluation
* Compare scores (e.g., 7.66 → 8.7 ✅)

***

## ✅ Key Benefits

* Objective measurement of prompt quality
* Compare prompt versions numerically
* Iterative improvement loop
* Reduces guesswork in prompt engineering [\[A typical...l workflow\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287736)

# ❓ 2. Q\&A

### Q1. What is the goal of an eval workflow?

**A:** To systematically improve prompts using measurable evaluation rather than guesswork.

### Q2. Why create an eval dataset?

**A:** To simulate real-world inputs your prompt will handle in production.

### Q3. What does the grader do?

**A:** Scores Claude’s outputs to quantify performance (e.g., 1–10 scale).

### Q4. Why calculate an average score?

**A:** To objectively compare different prompt versions.

### Q5. What is the purpose of iteration?

**A:** To refine prompts and achieve better performance over time.

# 🎯 4. Key Takeaways

* Prompt engineering should be **measured, not guessed**
* Always create a **realistic evaluation dataset**
* Use **scoring (grader)** to quantify performance
* Focus on **average score** for comparison
* Improvement = **iterate + re-evaluate**
* This is essentially:
  ```
  Prompt → Test → Score → Improve → Repeat
  ```
