### Core concept

* **Model-based grading** is part of a **prompt evaluation workflow** used to measure AI output quality.
* A **grader** takes model output and returns a **score (typically 1–10)** representing quality. [\[Model based grading\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287742)

### Types of graders

1. **Code graders**
   * Programmatic checks (rules/logic)
   * Examples:
     * Output length
     * Keyword presence
     * JSON/Python syntax validation [\[Model based grading\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287742)

2. **Model graders (focus of this page)**
   * Use another AI model to evaluate output quality
   * Can assess:
     * Quality
     * Instruction following
     * Completeness
     * Helpfulness
     * Safety [\[Model based grading\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287742)

3. **Human graders**
   * Manual evaluation
   * Most flexible but slow and costly [\[Model based grading\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287742)

***

### Evaluation criteria (critical step)

Before grading, define **clear criteria**, for example:

* **Format** → correct output structure (e.g., JSON only)
* **Valid syntax** → code correctness
* **Task following** → aligns with user request [\[Model based grading\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287742)

👉 Insight:

* Code graders = best for format/syntax
* Model graders = best for reasoning, completeness, intent

***

### How Model Graders work

* Send model output into another AI evaluation prompt
* Ask for structured feedback:
  * strengths
  * weaknesses
  * reasoning
  * score (1–10) [\[Model based grading\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287742)

👉 Key insight:

* If you only ask for a score → results tend to cluster around \~6
* Adding reasoning improves score accuracy and usefulness [\[Model based grading\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287742)

***

### Evaluation workflow

1. Run prompt → get output
2. Send output to grader
3. Extract:
   * score
   * reasoning
4. Aggregate results across dataset
5. Compute **average score** to track improvement [\[Model based grading\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287742)

    # Create evaluation prompt

    You are an expert code reviewer. Evaluate this AI-generated solution.

#### Integrating Grading into Your Workflow
Update your test case runner to call the grader:
```
def run_test_case(test_case):
    output = run_prompt(test_case)
    
    # Grade the output
    model_grade = grade_by_model(test_case, output)
    score = model_grade["score"]
    reasoning = model_grade["reasoning"]
    
    return {
        "output": output, 
        "test_case": test_case, 
        "score": score,
        "reasoning": reasoning
    }
```
#### Calculate an average score across all test cases:
```
from statistics import mean

def run_eval(dataset):
    results = []
    
    for test_case in dataset:
        result = run_test_case(test_case)
        results.append(result)
    
    average_score = mean([result["score"] for result in results])
    print(f"Average score: {average_score}")
    
    return results
```    
### Key takeaway concept

* Model graders provide a **consistent baseline metric**
* Even if imperfect, they are useful for:
  * comparing prompt versions
  * tracking improvements over time

# ❓ 2. Q\&A

### Q1. What is a grader in prompt evaluation?

A system that evaluates model output and returns a measurable score (typically 1–10). [\[Model based grading\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287742)

### Q2. What are the three types of graders?

* Code graders
* Model graders
* Human graders [\[Model based grading\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287742)

### Q3. When should you use a model grader?

When evaluating subjective qualities like:

* helpfulness
* completeness
* instruction following [\[Model based grading\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287742)

### Q4. Why not rely only on human graders?

They are time-consuming and inefficient at scale. [\[Model based grading\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287742)

### Q5. Why include reasoning in grading output?

To avoid generic scoring and improve evaluation quality. [\[Model based grading\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287742)

### Q6. What metric tracks overall prompt performance?

Average score across test cases. [\[Model based grading\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287742)

# 🔑 4. Key Takeaways

* Grading is essential for **objective prompt evaluation**
* Three approaches:
  * Code → deterministic checks
  * Model → flexible evaluation
  * Human → highest quality but slow
* Model graders enable:
  * scalable evaluation
  * consistent benchmarking
* Always define **evaluation criteria first**
* Include **reasoning + strengths + weaknesses** to improve grading quality
* Use **average score** as a performance metric
