# 1. Summary

**Source:** [Defining resources (Claude MCP course)](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287782)

## Overview

* **Resources in MCP (Model Context Protocol)** expose **data to clients**, similar to **GET endpoints in HTTP APIs**. [\[Defining resources\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287782)
* They are used for **retrieval-only scenarios** (no side effects), unlike tools which perform actions. [\[Defining resources\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287782)

## Core Concept

* Resource interaction follows a **request–response pattern**:
  * Client sends `ReadResourceRequest` with a **URI**
  * Server returns data mapped to that URI [\[Defining resources\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287782)

## Example Use Case

* **Document mention system (`@document`)**
  * List all documents → autocomplete
  * Fetch specific document → inject content into prompt [\[Defining resources\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287782)

## Resource Types

* **Direct resources (static)**
  * Example: `docs://documents`
* **Templated resources (dynamic)**
  * Example: `docs://documents/{doc_id}` [\[Defining resources\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287782)

## Implementation

* Defined using decorator:

```python
@mcp.resource("docs://documents", mime_type="application/json")
def list_docs() -> listreturn list(docs.keys())
```

```python
@mcp.resource("docs://documents/{doc_id}", mime_type="text/plain")
def fetch_doc(doc_id: str) -> str:
    if doc_id not in docs:
        raise ValueError(f"Doc with id {doc_id} not found")
    return docs[doc_id]
```

 [\[Defining resources\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287782)

## Key Features

* Automatic parameter parsing from URI → function arguments [\[Defining resources\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287782)
* Automatic serialization (no manual JSON handling) [\[Defining resources\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287782)
* Supports multiple data types via MIME types [\[Defining resources\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287782)

## Testing

* Use **MCP Inspector**

```bash
uv run mcp dev mcp_server.py
```

* View:
  * Resources
  * Resource templates
  * Actual response output [\[Defining resources\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287782)

## Key Distinction

* **Resources → read-only data access**
* **Tools → actions / operations** [\[Defining resources\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287782)

# 2. Q\&A

### Q1. What are MCP resources?

* Components that expose **data endpoints** for clients, similar to HTTP GET APIs.

### Q2. How are resources accessed?

* Through a **URI-based request (`ReadResourceRequest`)**.

### Q3. What is the difference between tools and resources?

* Resources: fetch data
* Tools: perform actions

### Q4. What are the two resource types?

* Direct (static URI)
* Templated (parameterized URI)

### Q5. How are parameters passed in templated resources?

* Extracted from URI and automatically mapped to function arguments.

### Q6. Why define MIME types?

* To inform clients about the **format of returned data**.

### Q7. Do developers need to serialize responses manually?

* No, the SDK handles serialization automatically.

### Q8. How to test MCP resources?

* Using **MCP Inspector (dev mode)**.

# 4. Key Takeaways

* MCP **resources = data layer abstraction** (read-only API-like endpoints)
* Use:
  * **Direct resources** → static datasets
  * **Templated resources** → dynamic queries
* URI acts as **contract/interface**
* SDK simplifies:
  * parameter mapping
  * serialization
* Essential for:
  * RAG pipelines
  * document retrieval
  * context injection for LLM prompts
* Clear separation:
  * **Resource = READ**
  * **Tool = EXECUTE**
