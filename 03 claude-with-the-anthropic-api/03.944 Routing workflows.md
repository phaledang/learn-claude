## 1. Summary (page: [Routing workflows](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287801))

**Core concept**

* Routing workflows address **heterogeneous request handling** in AI systems.
* Replace **single generic prompt** with **category-based specialized pipelines**.

**Problem**

* Generic prompt cannot handle different intents effectively  
  → Example:
  * *Programming* → needs **educational explanation**
  * *Surfing* → needs **entertainment style content**

**Solution**

* Introduce **routing layer**:
  1. **Categorize input**
  2. **Route to specialized prompt/pipeline**

**Architecture**

```
User Input 
   → Router (Claude classification)
   → Category
   → Specific Pipeline (prompt/tool/workflow)
   → Final Response
```

**Categories example**

* Entertainment
* Educational
* Comedy
* Personal vlog
* Reviews
* Storytelling

**Execution flow**

1. Send input → classification prompt
2. Claude returns category
3. Select corresponding prompt template
4. Generate final output with optimized prompt

**Key design principle**

* Input is routed to **ONE pipeline only**
* Each pipeline can be:
  * different prompt
  * different tools
  * different workflow

**When to use**

* Multiple request types
* Clear categorization possible
* Need higher quality vs generic output
* Typical use cases:
  * Customer service bots
  * Content generation
  * Multi-domain assistants [\[Routing workflows\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287801)

## 2. Q\&A

**Q1. What problem does routing solve?**  
→ Handling different request types that require different processing strategies.

**Q2. Why not use one prompt?**  
→ Generic prompts cannot optimize for diverse intents (e.g., educational vs entertainment).

**Q3. What are the 2 main steps in routing?**

* Categorization
* Specialized processing

**Q4. What does the router do?**  
→ Classifies input into predefined categories using Claude.

**Q5. Does input go to multiple pipelines?**  
→ No, only **one selected pipeline**.

**Q6. What defines a pipeline?**

* Prompt template
* Tools (optional)
* Workflow logic

**Q7. What makes routing effective?**

* Clearly defined categories
* Reliable classification
* Strong specialized prompts

## 4. Key Takeaways

* Routing = **classification + dispatch pattern**
* Improves **quality and relevance** of output
* Enables **modular pipeline design**
* Works best when:
  * categories are **well-defined**
  * classification is **accurate**
* Important design rule:
  * **One input → one pipeline**
* Acts as **foundation for agent orchestration**
  * (can be extended with tools, RAG, multi-step workflows)
