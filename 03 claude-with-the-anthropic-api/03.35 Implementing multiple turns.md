This page explains how to build **multi-turn conversations with Claude when using tools**.

### Key concepts:

* **Conversation loop**
  * You must repeatedly call Claude until it stops requesting tools.
  * The loop ends when `stop_reason != "tool_use"`.

* **Detecting tool usage**
  * Claude signals tool usage via:
    ```
    response.stop_reason == "tool_use"
    ```
  * If true → execute tools and continue the loop.

* **Core workflow**
  ```
  while True:
      call Claude
      if tool requested:
          run tools
          send results back
      else:
          break
  ```

* **Handling multiple tool calls**
  * Claude can request **multiple tools in one response**
  * Each tool request is a **block** in `message.content`

* **Tool result structure**
  * Each tool request must have a matching result:
    ```
    {
      "type": "tool_result",
      "tool_use_id": "...",
      "content": "...",
      "is_error": False
    }
    ```

* **Error handling**
  * Always return a tool result, even if it fails
  * Use:
    ```
    is_error = True
    ```

* **Tool routing**
  * Use a dispatcher function:
    ```
    if tool_name == "...":
        call function
    ```

* **End-to-end flow**
  1. Send user message + tools
  2. Claude responds (text + tool requests)
  3. Execute tools
  4. Send results back
  5. Repeat until final answer

✅ Outcome:  
Enables **agent-like behavior**, where Claude chains multiple tools across turns to solve complex problems.

# ❓ 2. Q\&A

### Q1. How do you know Claude wants to call a tool?

**A:** Check `response.stop_reason == "tool_use"`.

### Q2. When does the conversation loop stop?

**A:** When `stop_reason` is NOT `"tool_use"`.

### Q3. Can Claude call multiple tools at once?

**A:** Yes, tool calls appear as multiple blocks in `message.content`.

### Q4. What must you return after executing a tool?

**A:** A `tool_result` block linked to the tool request ID.

### Q5. What happens if a tool fails?

**A:** You must still return a result with `is_error = True`.

### Q6. What is the purpose of tool routing?

**A:** To map tool names to their implementation functions.

### Q7. Why is conversation history important?

**A:** It preserves context across turns so Claude can build on prior results.

# 🔑 4. Key Takeaways

* Multi-turn tool usage requires a **loop-based architecture**
* `stop_reason` is the **control signal** for flow
* Claude can:
  * Call **multiple tools per turn**
  * Chain tools across **multiple turns**
* Always:
  * Return tool results (even on failure)
  * Maintain conversation history
* Use:
  * **Routing layer** → scalable tool integration
* This pattern is foundational for:
  * AI agents
  * automation workflows
