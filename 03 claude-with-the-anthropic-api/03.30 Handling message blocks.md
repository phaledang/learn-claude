
# 📘 1. Summary

**Page:** *Handling message blocks*  
**URL:** <https://anthropic.skilljar.com/claude-with-the-anthropic-api/287757>

### Key concepts

* Claude tool usage introduces **multi-block responses** instead of simple text.
* A response can contain:
  * **Text Block** → Human-readable explanation
  * **ToolUse Block** → Instruction for calling a tool (function name + parameters)
* To enable tools:
  * Include a `tools` parameter (JSON schemas) in API calls.
* You must **manually manage conversation history**:
  * Claude does NOT store context
  * Always preserve the **full multi-block message structure**
* Tool workflow pattern:
  1. Send user message + tool schema
  2. Claude returns multi-block message (text + tool call)
  3. Your code executes the tool
  4. Send tool result back with full history
  5. Claude returns final answer
* Existing helper functions must be updated to support:
  * Multi-block content (not just plain text)

# ❓ 2. Q\&A

### Q1: What is a multi-block message in Claude?

**A:** A response containing multiple content blocks, including text and tool usage instructions. [\[Handling m...age blocks\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287757)

### Q2: Why is the `tools` parameter required?

**A:** It defines the available functions Claude can call using JSON schemas. [\[Handling m...age blocks\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287757)

### Q3: What information is inside a ToolUse block?

**A:**

* Tool call ID
* Function name
* Input parameters
* Type = `"tool_use"` [\[Handling m...age blocks\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287757)

### Q4: Does Claude store conversation history automatically?

**A:** No, you must manage and resend the full history yourself. [\[Handling m...age blocks\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287757)

### Q5: Why must you preserve the full response content?

**A:** Because both text and tool blocks are required to maintain correct context across turns. [\[Handling m...age blocks\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287757)

### Q6: What happens after Claude requests a tool?

**A:** Your system executes the tool and sends the result back for final response generation. [\[Handling m...age blocks\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287757)

# 🔑 4. Key Takeaways

* 🔹 Claude tool usage = **multi-block response architecture**
* 🔹 Always include **tools schema** in API calls
* 🔹 Preserve **entire conversation history** (critical!)
* 🔹 Tool flow = **request → tool call → execution → final response**
* 🔹 Update helper functions to support **complex content structures**
* 🔹 Mis-handling blocks = broken context or incorrect outputs
