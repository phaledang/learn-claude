## 1. 📄 Summary of the Page

**Page URL:** [Using multiple tools](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287749)

This page explains how to extend a Claude-powered application by integrating **multiple tools** into a single conversation workflow.

### Key concepts:

* Once the **core tool-handling infrastructure** is built, adding more tools becomes straightforward.
* Example use case: a **reminder system** that requires:
  * Getting current date/time
  * Adding duration to a date
  * Setting a reminder
* Tools are integrated by:
  * Adding **tool schemas** into the API request (`tools=[...]`)
  * Updating the **tool router** to map tool names → functions
* Claude can:
  * Call **multiple tools sequentially**
  * Combine **text explanation + tool calls in a single response**
* The conversation flow includes:
  * User message
  * Assistant message (with tool calls)
  * Tool results
  * Follow-up assistant response
* Adding new tools follows a **simple modular pattern**:
  1. Implement function
  2. Define schema
  3. Add to tools list
  4. Handle in router

👉 Overall: This demonstrates **agent-like orchestration**, where Claude dynamically selects and chains tools to complete complex tasks. [\[Using multiple tools\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287749)

## 2. ❓ Q\&A from the Page

**Q1: What is required before adding multiple tools?**  
A: A working tool-handling infrastructure must already exist.

**Q2: How do you expose tools to Claude?**  
A: By passing tool schemas in the `tools` parameter of the API request.

**Q3: What is the role of the tool router (`run_tool`)?**  
A: It maps tool names to their corresponding function implementations.

**Q4: Can Claude use more than one tool in a single request?**  
A: Yes, it can call multiple tools sequentially in a conversation.

**Q5: What kind of response structure does Claude produce?**  
A: A mix of text explanation + tool use blocks + tool results.

**Q6: What pattern is used to add new tools?**  
A: Implement function → define schema → register tool → update router.
