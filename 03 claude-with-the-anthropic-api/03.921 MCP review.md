# 1. Summary (Page: MCP review)

Source: [MCP review](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287790)

This page is part of the *Model Context Protocol (MCP)* section in Anthropic’s Claude API course.

**Key focus of the page:**

* It acts as a **review checkpoint** for MCP concepts
* Reinforces how MCP enables:
  * **Standardized tool integration**
  * **Client–server architecture**
  * **Structured interaction between LLMs and external capabilities**

**Core MCP concepts covered across this section:**

* MCP defines a **protocol layer** between:
  * Claude (model)
  * External tools/resources
* Key components:
  * **MCP Client**
  * **MCP Server**
  * **Tools (functions)**
  * **Resources (data endpoints)**
  * **Prompts (reusable templates)**
* Enables:
  * Tool discovery
  * Tool invocation
  * Multi-step workflows
* Supports:
  * Inspecting server capabilities (server inspector)
  * Structured tool definitions
  * Reusable and scalable integrations

# 2. Q\&A

### Q1. What is MCP mainly used for?

* MCP is used to **standardize how Claude interacts with external tools and resources**

### Q2. What are the core components of MCP?

* Client
* Server
* Tools
* Resources
* Prompts

### Q3. What does an MCP server do?

* Hosts tools, resources, and prompts
* Exposes capabilities to the client

### Q4. What does an MCP client do?

* Connects to MCP servers
* Sends requests and receives structured responses

### Q5. Why is MCP important compared to raw tool calling?

* Provides:
  * Standardization
  * Reusability
  * Separation of concerns
  * Scalable architecture

# 4. Key Takeaways

* MCP (Model Context Protocol) = **standard protocol for tool + data integration**
* Enables **clean separation**:
  * Model logic vs tool execution
* Promotes:
  * Reusability
  * Scalability
  * Maintainability
* Supports:
  * Multi-tool workflows
  * Agent-based architectures
* Foundation for:
  * **Agent systems**
  * **Enterprise integrations**
  * **Multi-service orchestration**

# 5.Extension Mapping

Since previously discussed topics include LangChain / LangGraph:

### MCP vs LangChain / LangGraph

| Concept         | MCP               | LangChain          | LangGraph              |
| --------------- | ----------------- | ------------------ | ---------------------- |
| Tool definition | Standard protocol | Python abstraction | Graph node             |
| Execution       | Client-server     | Chain/Agent        | State machine workflow |
| Reusability     | High (cross-app)  | Medium             | High                   |
| Architecture    | Decoupled         | In-process         | Orchestrated graph     |

* MCP = **interoperability layer**
