# 🧠 1. Summary

**Page:** *Overview of Claude models*  
👉 [Open page](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287722)

### Key Points

* Claude provides **3 model families**:
  * **Opus**
    * Highest intelligence
    * Best for complex reasoning & planning
    * Trade‑off: higher cost + slower latency
  * **Sonnet**
    * Balanced performance (intelligence + speed + cost)
    * Strong for coding and real-world applications
    * ✅ *Default choice for most use cases*
  * **Haiku**
    * Fastest and lowest cost
    * Optimized for speed & high-volume tasks
    * Limited reasoning depth
<img width="1294" height="537" alt="image" src="https://github.com/user-attachments/assets/ca36cc4c-d3d4-44a2-8498-cba3f7304991" />

### Selection Framework

* Need **deep reasoning → Opus**
* Need **speed & cost efficiency → Haiku**
* Need **balance → Sonnet**

### Important Insight

* ✅ Best practice: **Use multiple models in one system**
  * Example:
    * Haiku → real-time UI
    * Sonnet → logic layer
    * Opus → heavy reasoning

### Shared Capabilities

All models can:

* Generate text
* Write and edit code
* Analyze images

👉 Difference = **optimization focus (not core capability)**

# ❓ 2. Q\&A

### Q1: What is the main difference between Opus, Sonnet, and Haiku?

**A:** Their priorities: intelligence (Opus), balance (Sonnet), or speed/cost (Haiku).

### Q2: Which model should I use for most applications?

**A:** Sonnet — it provides the best balance of performance, cost, and speed.

### Q3: Which model is best for complex reasoning tasks?

**A:** Opus.

### Q4: Which model is best for real-time user interactions?

**A:** Haiku.

### Q5: Do different Claude models have different capabilities?

**A:** No — they share the same capabilities; only optimization differs.

### Q6: Should I use only one model in my system?

**A:** No — it’s recommended to combine models for different tasks.

# 🔑 4. Key Takeaways

* Claude models differ by **optimization, not capability**
* **Sonnet = default choice** for most systems
