It also describes **size-based with overlap** as a common production fallback, **structure-based** as strong when document format is controlled, **sentence-based** as a practical middle ground, and **semantic-based** as the most sophisticated but more computationally expensive. # 2) Shared Claude API syntax

Anthropic’s Python SDK documentation shows the raw `Anthropic()` client and `client.messages.create(...)` pattern, where `messages` is a list of role/content blocks and the returned text is accessed from `message.content[0].text`. [\[platform.claude.com\]](https://platform.claude.com/docs/en/api/sdks/python), [\[platform.claude.com\]](https://platform.claude.com/docs/en/home)

```python
import os
from anthropic import Anthropic

client = Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

def summarize_chunks_with_claude(chunks):
    joined = "\n\n--- CHUNK ---\n\n".join(chunks)

    message = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=800,
        messages=[
            {
                "role": "user",
                "content": (
                    "Summarize the following chunks and keep topics separated:\n\n"
                    f"{joined}"
                ),
            }
        ],
    )

    return message.content[0].text
```

### Syntax notes

* `Anthropic(...)` creates the SDK client. [\[platform.claude.com\]](https://platform.claude.com/docs/en/api/sdks/python)
* `client.messages.create(...)` is the main messages call. [\[platform.claude.com\]](https://platform.claude.com/docs/en/api/sdks/python), [\[platform.claude.com\]](https://platform.claude.com/docs/en/home)
* `messages=[{...}]` is the structured conversation payload. [\[platform.claude.com\]](https://platform.claude.com/docs/en/api/sdks/python), [\[platform.claude.com\]](https://platform.claude.com/docs/en/home)
* `message.content[0].text` retrieves the first text block in the response. [\[platform.claude.com\]](https://platform.claude.com/docs/en/home)

# 3) Shared LangGraph syntax

LangGraph documentation defines `StateGraph` as a graph where nodes read/write shared state, and notes that the builder must be **compiled** before calling methods such as `invoke()`. LangGraph is positioned as a **low-level orchestration framework**, and the docs explicitly say it commonly uses LangChain components but does not require LangChain. [\[docs.langchain.com\]](https://docs.langchain.com/oss/python/langgraph/overview), [\[reference....gchain.com\]](https://reference.langchain.com/python/langgraph/graph/state/StateGraph)

```python
from typing import TypedDict, List
from langgraph.graph import StateGraph, START, END

class ChunkState(TypedDict):
    text: str
    chunks: List[str]
    final_summary: str

def summarize_node(state: ChunkState):
    return {
        "final_summary": summarize_chunks_with_claude(state["chunks"])
    }

def build_graph(chunk_node_fn):
    graph = StateGraph(ChunkState)
    graph.add_node("chunk_text", chunk_node_fn)
    graph.add_node("summarize", summarize_node)
    graph.add_edge(START, "chunk_text")
    graph.add_edge("chunk_text", "summarize")
    graph.add_edge("summarize", END)
    return graph.compile()
```

### Syntax notes

* `StateGraph(StateSchema)` defines a graph with shared state. [\[reference....gchain.com\]](https://reference.langchain.com/python/langgraph/graph/state/StateGraph)
* Each node follows a `State -> Partial<State>` update pattern. [\[reference....gchain.com\]](https://reference.langchain.com/python/langgraph/graph/state/StateGraph)
* `compile()` is required before `invoke()`. [\[reference....gchain.com\]](https://reference.langchain.com/python/langgraph/graph/state/StateGraph)
* LangGraph focuses on orchestration/stateful workflow control rather than being a document-splitting library itself. [\[docs.langchain.com\]](https://docs.langchain.com/oss/python/langgraph/overview)

# 4) Strategy 1 — Size-Based Chunking

The page describes size-based chunking as dividing text into equal-length strings, with **overlap** added to reduce context loss. [\[Text chunk...strategies\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287776)

## Claude (raw API)

```python
def chunk_by_size(text: str, chunk_size: int = 300, overlap: int = 50):
    chunks = []
    start = 0

    while start < len(text):
        end = min(start + chunk_size, len(text))
        chunks.append(text[start:end])

        if end == len(text):
            break

        start = end - overlap

    return chunks

with open("sample_input.txt", "r", encoding="utf-8") as f:
    text = f.read()

chunks = chunk_by_size(text, chunk_size=300, overlap=50)
final_summary = summarize_chunks_with_claude(chunks)
print(final_summary)
```

### Syntax explanation

* `chunk_size`: maximum characters per chunk.
* `overlap`: repeats trailing context from the previous chunk.
* `start = end - overlap`: sliding-window behavior.
* This mirrors the page’s “size-based + overlap” idea. [\[Text chunk...strategies\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287776)

## LangChain

LangChain’s recommended generic splitter is `RecursiveCharacterTextSplitter`, which recursively tries separators such as paragraph, newline, space, and finally character. The docs describe `chunk_size`, `chunk_overlap`, and `split_text(...)` directly. [\[reference....gchain.com\]](https://reference.langchain.com/python/langchain-text-splitters/character/RecursiveCharacterTextSplitter), [\[docs.langchain.com\]](https://docs.langchain.com/oss/python/integrations/splitters/recursive_text_splitter), [\[docs.langchain.com\]](https://docs.langchain.com/oss/python/integrations/splitters)

```python
from langchain_text_splitters import RecursiveCharacterTextSplitter

with open("sample_input.txt", "r", encoding="utf-8") as f:
    text = f.read()

splitter = RecursiveCharacterTextSplitter(
    chunk_size=300,
    chunk_overlap=50,
    length_function=len,
    is_separator_regex=False,
)

chunks = splitter.split_text(text)
print(chunks[:2])
```

### Syntax explanation

* `RecursiveCharacterTextSplitter(...)`: LangChain generic recursive splitter. [\[reference....gchain.com\]](https://reference.langchain.com/python/langchain-text-splitters/character/RecursiveCharacterTextSplitter), [\[docs.langchain.com\]](https://docs.langchain.com/oss/python/integrations/splitters/recursive_text_splitter)
* `chunk_size`: max chunk length. [\[docs.langchain.com\]](https://docs.langchain.com/oss/python/integrations/splitters/recursive_text_splitter)
* `chunk_overlap`: preserves continuity between chunks. [\[docs.langchain.com\]](https://docs.langchain.com/oss/python/integrations/splitters/recursive_text_splitter)
* `split_text(text)`: returns plain string chunks. [\[reference....gchain.com\]](https://reference.langchain.com/python/langchain-text-splitters/character/RecursiveCharacterTextSplitter), [\[docs.langchain.com\]](https://docs.langchain.com/oss/python/integrations/splitters/recursive_text_splitter)

## LangGraph

```python
def chunk_by_size_node(state: ChunkState):
    text = state["text"]
    chunks = []
    start = 0
    chunk_size = 300
    overlap = 50

    while start < len(text):
        end = min(start + chunk_size, len(text))

        if end == len(text):

        start = end - overlap

    text = f.read()

print(result["final_summary"])
```

### Syntax explanation

* `chunk_by_size_node(...)` is a graph node that updates only `chunks`.
* `app.invoke(...)` runs the compiled graph. [\[reference....gchain.com\]](https://reference.langchain.com/python/langgraph/graph/state/StateGraph)
* LangGraph adds orchestration and shared-state control; the actual chunking logic is still custom code. [\[docs.langchain.com\]](https://docs.langchain.com/oss/python/langgraph/overview), [\[reference....gchain.com\]](https://reference.langchain.com/python/langgraph/graph/state/StateGraph)

# 5) Strategy 2 — Structure-Based Chunking

The course page defines structure-based chunking as splitting on natural document structure such as headers/sections and notes it works especially well when formatting is controlled. [\[Text chunk...strategies\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287776)

## Claude (raw API)

```python
import re

def chunk_by_markdown_headers(text: str):
    parts = re.split(r"(?m)^## ", text)
    chunks = [p.strip() for p in parts if p.strip()]
    return chunks

with open("sample_input.txt", "r", encoding="utf-8") as f:
    text = f.read()

chunks = chunk_by_markdown_headers(text)
final_summary = summarize_chunks_with_claude(chunks)
print(final_summary)
```

### Syntax explanation

* `(?m)^## ` matches second-level Markdown headers at line starts.
* `re.split(...)` cuts the file into section-level chunks.
* This is suitable because the sample input intentionally uses Markdown-style sections.

## LangChain

LangChain documents `MarkdownHeaderTextSplitter` for splitting Markdown documents by specified headers, grouping text under common header metadata. [\[docs.langchain.com\]](https://docs.langchain.com/oss/python/integrations/splitters/markdown_header_metadata_splitter), [\[reference....gchain.com\]](https://reference.langchain.com/python/langchain-text-splitters/markdown/MarkdownHeaderTextSplitter)

```python
from langchain_text_splitters import MarkdownHeaderTextSplitter

with open("sample_input.txt", "r", encoding="utf-8") as f:
    markdown_text = f.read()

headers_to_split_on = [
    ("#", "Header 1"),
    ("##", "Header 2"),
]

splitter = MarkdownHeaderTextSplitter(
    headers_to_split_on=headers_to_split_on,
    strip_headers=True,
    return_each_line=False,
)

docs = splitter.split_text(markdown_text)

for doc in docs:
    print("CONTENT:", doc.page_content)
    print("METADATA:", doc.metadata)
    print("---")
```

### Syntax explanation

* `headers_to_split_on`: tuples of header marker and metadata key. [\[docs.langchain.com\]](https://docs.langchain.com/oss/python/integrations/splitters/markdown_header_metadata_splitter), [\[reference....gchain.com\]](https://reference.langchain.com/python/langchain-text-splitters/markdown/MarkdownHeaderTextSplitter)
* `strip_headers=True`: removes the header text from chunk content. [\[docs.langchain.com\]](https://docs.langchain.com/oss/python/integrations/splitters/markdown_header_metadata_splitter), [\[reference....gchain.com\]](https://reference.langchain.com/python/langchain-text-splitters/markdown/MarkdownHeaderTextSplitter)
* `split_text(...)` returns `Document` objects with content + metadata. [\[docs.langchain.com\]](https://docs.langchain.com/oss/python/integrations/splitters/markdown_header_metadata_splitter), [\[reference....gchain.com\]](https://reference.langchain.com/python/langchain-text-splitters/markdown/MarkdownHeaderTextSplitter)

# 6) Strategy 3 — Semantic-Based Chunking

The course page describes semantic chunking as the most sophisticated option because it groups semantically related sentences rather than only using text length or syntax; it also notes the higher computational cost. [\[Text chunk...strategies\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287776)

> **Transparency note:** The following Claude raw implementation is an **illustrative design suggestion**, not a built-in Anthropic chunking feature. Anthropic documents the Messages API call shape, but semantic chunking itself is application logic. [\[platform.claude.com\]](https://platform.claude.com/docs/en/api/sdks/python), [\[platform.claude.com\]](https://platform.claude.com/docs/en/home)

## Claude (raw API — illustrative app-side semantic chunking)

```python
import re
import math

def cosine_similarity(a, b):
    dot = sum(x * y for x, y in zip(a, b))
    norm_a = math.sqrt(sum(x * x for x in a))
    norm_b = math.sqrt(sum(x * x for x in b))
    return dot / (norm_a * norm_b + 1e-9)

def embed_sentence(sentence: str):
    """
    Placeholder.
    Replace with an embedding provider used in the application.
    """
    raise NotImplementedError("Plug in your embedding model here.")

def semantic_chunk(text: str, similarity_threshold: float = 0.80):
    sentences = re.split(r"(?<=[.!?])\s+", text.strip())
    if not sentences:
        return []

    chunks = [[sentences[0]]]
    prev_embedding = embed_sentence(sentences[0])

    for sentence in sentences[1:]:
        current_embedding = embed_sentence(sentence)
        similarity = cosine_similarity(prev_embedding, current_embedding)

        if similarity >= similarity_threshold:
            chunks[-1].append(sentence)
        else:
            chunks.append([sentence])

        prev_embedding = current_embedding

    return [" ".join(group) for group in chunks]
```

### Syntax explanation

* `embed_sentence(...)` is the extension point for any embedding service.
* `cosine_similarity(...)` compares adjacent sentence vectors.
* `similarity_threshold`: determines whether a sentence stays in the current chunk or starts a new one.
* This matches the course idea of grouping **related sentences** rather than using fixed windows. [\[Text chunk...strategies\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287776)

## LangChain

DeepWiki documentation for `langchain_experimental` describes `SemanticChunker` as an embedding-based text splitter that analyzes sentence similarity and creates semantically coherent chunks. [\[deepwiki.com\]](https://deepwiki.com/langchain-ai/langchain-experimental/3.2-semanticchunker)

```python
from langchain_experimental.text_splitter import SemanticChunker
from langchain_openai import OpenAIEmbeddings

with open("sample_input.txt", "r", encoding="utf-8") as f:
    text = f.read()

embeddings = OpenAIEmbeddings()

splitter = SemanticChunker(
    embeddings=embeddings,
    breakpoint_threshold_type="percentile",
)

chunks = splitter.split_text(text)
print(chunks[:2])
```

### Syntax explanation

* `SemanticChunker(...)` uses embeddings to decide breakpoints. [\[deepwiki.com\]](https://deepwiki.com/langchain-ai/langchain-experimental/3.2-semanticchunker)
* `embeddings=...` supplies the vector model used to compare semantic distance. [\[deepwiki.com\]](https://deepwiki.com/langchain-ai/langchain-experimental/3.2-semanticchunker)
* `breakpoint_threshold_type="percentile"` is one documented thresholding option. [\[deepwiki.com\]](https://deepwiki.com/langchain-ai/langchain-experimental/3.2-semanticchunker)

# 7) Strategy 4 — Sentence-Based Chunking

The course page presents sentence-based chunking as a practical middle ground: split into sentences, then group several sentences into a chunk with optional overlap. [\[Text chunk...strategies\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287776)

## Claude (raw API)

```python
import re

def chunk_by_sentence(text: str, max_sentences: int = 3, overlap_sentences: int = 1):
    sentences = re.split(r"(?<=[.!?])\s+", text.strip())
    chunks = []

    start = 0
    step = max(1, max_sentences - overlap_sentences)

    while start < len(sentences):
        end = min(start + max_sentences, len(sentences))
        chunks.append(" ".join(sentences[start:end]))

        if end == len(sentences):
            break

        start += step

    return chunks

with open("sample_input.txt", "r", encoding="utf-8") as f:
    text = f.read()

chunks = chunk_by_sentence(text, max_sentences=3, overlap_sentences=1)
final_summary = summarize_chunks_with_claude(chunks)
print(final_summary)
```

### Syntax explanation

* `re.split(r"(?<=[.!?])\s+", ...)` splits on sentence boundaries.
* `max_sentences`: caps sentence count per chunk.
* `overlap_sentences`: retains some prior sentence context.
* This closely follows the sentence-grouping behavior described on the page. [\[Text chunk...strategies\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287776)

## LangChain

LangChain’s text splitter references list sentence-oriented splitters such as `NLTKTextSplitter` and `SpacyTextSplitter`. [\[langchain-...thedocs.io\]](https://langchain-doc.readthedocs.io/en/latest/reference/modules/text_splitter.html)

### Syntax explanation

* `NLTKTextSplitter()` is a sentence-aware splitter listed in LangChain’s text splitter reference. [\[langchain-...thedocs.io\]](https://langchain-doc.readthedocs.io/en/latest/reference/modules/text_splitter.html)
* `split_text(...)` returns chunk strings. [\[langchain-...thedocs.io\]](https://langchain-doc.readthedocs.io/en/latest/reference/modules/text_splitter.html)
* If stronger sentence parsing is needed, the same reference also lists `SpacyTextSplitter`. [\[langchain-...thedocs.io\]](https://langchain-doc.readthedocs.io/en/latest/reference/modules/text_splitter.html)

# 8) Quick comparison

| Strategy        | Claude raw API          | LangChain                                        | LangGraph                            |
| --------------- | ----------------------- | ------------------------------------------------ | ------------------------------------ |
| Size-Based      | Manual loop + overlap   | `RecursiveCharacterTextSplitter`                 | Custom node with size window         |
| Structure-Based | Regex/header parsing    | `MarkdownHeaderTextSplitter`                     | Custom node using document structure |
| Semantic-Based  | Custom embedding logic  | `SemanticChunker` (experimental ecosystem docs)  | Custom embedding node inside graph   |
| Sentence-Based  | Regex sentence grouping | `NLTKTextSplitter` / sentence-oriented splitters | Custom node with sentence windows    |

The course page’s practical guidance still holds across all three approaches: **structure-based** is best when document structure is reliable, **sentence-based** is a strong middle ground, and **size-based with overlap** is often the most dependable fallback for production pipelines. [\[Text chunk...strategies\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287776)

# 9) Recommended implementation pattern

For a clean enterprise-grade implementation:

1. Use [sample\_input.txt](blob:https://outlook.office.com/b72af9a2-6a4e-4c3a-8f75-54cfec745cb1) for local testing.
2. Start with **size-based + overlap** for broad document compatibility. This aligns with both the course guidance and LangChain’s default “recommended generic splitter” direction. [\[Text chunk...strategies\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287776), [\[docs.langchain.com\]](https://docs.langchain.com/oss/python/integrations/splitters/recursive_text_splitter), [\[docs.langchain.com\]](https://docs.langchain.com/oss/python/integrations/splitters)
3. Upgrade to **structure-based** when document format is known (Markdown, HTML, sectioned internal docs). [\[Text chunk...strategies\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287776), [\[docs.langchain.com\]](https://docs.langchain.com/oss/python/integrations/splitters/markdown_header_metadata_splitter)
4. Use **semantic-based** only when higher chunk quality justifies embedding cost and extra complexity. [\[Text chunk...strategies\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287776), [\[deepwiki.com\]](https://deepwiki.com/langchain-ai/langchain-experimental/3.2-semanticchunker)
5. Introduce **LangGraph** when the pipeline needs explicit workflow control, state, retries, branching, or agentic orchestration. [\[docs.langchain.com\]](https://docs.langchain.com/oss/python/langgraph/overview), [\[reference....gchain.com\]](https://reference.langchain.com/python/langgraph/graph/state/StateGraph)
