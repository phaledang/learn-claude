

# 📘 1. Summary

**Source:** [Introducing Tool Use](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287747)

### 🔹 What this page explains

* **Tool use** allows Claude to access **external data and systems** beyond its training knowledge. [\[Introducing tool use\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287747)
* By default, Claude:
  * Cannot access **real-time data**
  * Cannot query **external systems (APIs, databases)** [\[Introducing tool use\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287747)

### 🔹 The problem without tools

* Claude cannot answer **live or current queries** (e.g., weather, stock prices). [\[Introducing tool use\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287747)
* This creates a **poor user experience** when real-time data is needed. [\[Introducing tool use\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287747)

### 🔹 How tool use works (core flow)

1. **Initial Request:** User asks a question + instructions for tool usage
2. **Tool Request:** Claude decides what data it needs
3. **Data Retrieval:** Your backend fetches data (API/DB)
4. **Final Response:** Claude combines data + question into answer [\[Introducing tool use\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287747)

### 🔹 Example (Weather)

* User asks for weather
* Claude requests weather data
* Server calls API → returns data
* Claude produces **accurate real-time answer** [\[Introducing tool use\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287747)

### 🔹 Key benefits

* Real-time information
* Integration with APIs and systems
* Dynamic, context-aware responses
* Structured interaction between app and model [\[Introducing tool use\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287747)

# ❓ 2. Q\&A

### Q1: What is tool use in Claude?

**A:** A mechanism that allows Claude to request and use external data beyond its training knowledge.

### Q2: Why is tool use needed?

**A:** Because Claude cannot access real-time or external system data by default.

### Q3: What happens without tools?

**A:** Claude cannot answer questions requiring up-to-date information (e.g., weather, stock prices).

### Q4: Who retrieves the external data?

**A:** Your application/server fetches it from APIs or databases.

### Q5: What is the main interaction pattern?

**A:** Request → Tool call → Data retrieval → Final response.

# 🔑 4. Key Takeaways

* Tool use transforms Claude from:
  👉 **Static model → Dynamic assistant**
* It enables:
  * Real-time data access
  * Integration with external systems
  * More accurate and useful responses
* Important architectural pattern:
  * Claude **does NOT call APIs directly**
  * Your **backend orchestrates everything**
* Critical for building:
  * AI agents
  * Enterprise integrations
  * Real-time applications
