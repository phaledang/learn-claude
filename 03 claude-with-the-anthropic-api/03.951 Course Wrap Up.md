## 2. Q\&A (Derived from the page)

### Q1: What are the main building blocks of Claude API applications?

* Prompt engineering
* Tool use
* RAG pipelines
* MCP integration
* Workflow/agent orchestration

***

### Q2: Why is prompt engineering still critical even with tools and RAG?

* It controls:
  * reasoning clarity
  * output structure
  * instruction adherence
* Poor prompts → unreliable pipelines

***

### Q3: When should workflows be preferred over agents?

* When:
  * steps are predictable
  * reliability is critical
* Example:
  * document processing pipeline

***

### Q4: When should agents be used?

* When:
  * tasks are dynamic
  * unknown paths / decisions required
* Example:
  * research assistant

***

### Q5: What is the purpose of MCP?

* Standardizes integration of:
  * tools
  * prompts
  * resources
* Enables reusable AI system architecture

***

### Q6: What problem does prompt caching solve?

* Reduces:
  * latency
  * cost
* For repeated large prompts

***

### Q7: What completes a modern AI system?

Combination of:

* RAG + Tool use + Prompting + Execution + Orchestration

## 4. Key Takeaways

### Architecture mindset

* Move from:
  * **Feature-level thinking → system design**
* Combine:
  * Prompting
  * Tools
  * RAG
  * MCP
  * Workflows

***

### Design trade-offs

| Aspect      | Workflow | Agent    |
| ----------- | -------- | -------- |
| Reliability | High     | Medium   |
| Flexibility | Low      | High     |
| Control     | Explicit | Emergent |

***

### Production patterns

* Use:
  * **Workflow** → deterministic pipelines
  * **Agent** → exploratory tasks
* Hybrid pattern:
  * Workflow orchestrates agents

***

### Performance optimization

* Prompt caching → reduce cost
* RAG → reduce hallucination
* Tools → extend capability
* MCP → standardize integration

***

### Enterprise-ready direction (important for Solution Architect role)

* Build systems that:
  * are modular (MCP)
  * are scalable (RAG pipelines)
  * are testable (prompt eval)
  * balance reliability vs flexibility
