# LangChain Equivalent: Multi-turn Tool Loop

## Core Behavior

LangChain implements the same loop implicitly:

```
while not finished:
    call LLM
    if tool requested:
        execute tool(s)
        append results to messages
    else:
        return final response
```

This loop is handled by **AgentExecutor**. [\[sj-langcha...thedocs.io\]](https://sj-langchain.readthedocs.io/en/latest/agents/langchain.agents.agent.AgentExecutor.html)

***

# Minimal Implementation Structure

## 1. Tool Definition

```
@tool
def tool_name(input: type) -> type:
    """description"""
    ...
```

* Name, description, schema extracted automatically
* Used by LLM to decide tool usage [\[stackoverflow.com\]](https://stackoverflow.com/questions/77803857/create-tool-for-langchain-agent-that-accepts-multiple-inputs)

***

## 2. Model

```
llm = ChatAnthropic(model="claude-3-5-sonnet")
```

* Provides tool-calling capability
* Returns structured tool calls when needed

***

## 3. Agent Construction

```
agent = create_tool_calling_agent(
    llm=llm,
    tools=tools,
    prompt=prompt
)
```

* Combines:
  * model
  * tool schemas
  * prompt
* Enables reasoning + action selection

***

## 4. Execution Loop

```
executor = AgentExecutor(
    agent=agent,
    tools=tools
)
```

* Controls lifecycle:
  * iteration
  * tool invocation
  * stopping condition [\[sj-langcha...thedocs.io\]](https://sj-langchain.readthedocs.io/en/latest/agents/langchain.agents.agent.AgentExecutor.html)

***

## 5. Invocation

```
executor.invoke({"input": "..."} )
```

***

# Internal Flow (Equivalent Mapping)

## Raw Claude API

```
response = chat()
if stop_reason == "tool_use":
    run_tools()
    send_result()
repeat
```

***

## LangChain

```
executor.invoke()
    ↓
AgentExecutor loop:
    - LLM decides action
    - tool executed
    - observation appended
    - repeat
```

AgentExecutor performs:

* tool detection
* execution
* message update
* loop termination [\[wasilzafar.com\]](https://www.wasilzafar.com/pages/series/ai-app-dev-xtreme/ai-app-dev-sdk-langchain-part04-agents-tools.html)

***

# Multi-tool Handling

* Agent may request multiple steps:

```
step 1 → tool A
step 2 → tool B
step N → final answer
```

* Each step:
  * LLM reasoning
  * tool selection
  * observation
  * next decision

This follows **ReAct pattern (reason → act → observe)** [\[stackoverflow.com\]](https://stackoverflow.com/questions/77803857/create-tool-for-langchain-agent-that-accepts-multiple-inputs)

***

# Error Handling (Built-in)

AgentExecutor parameters:

```
max_iterations
max_execution_time
handle_parsing_errors
```

* Prevent infinite loops
* Control failure behavior [\[sj-langcha...thedocs.io\]](https://sj-langchain.readthedocs.io/en/latest/agents/langchain.agents.agent.AgentExecutor.html)

***

# Key Equivalence

| Concept (Claude API) | LangChain               |
| -------------------- | ----------------------- |
| stop\_reason         | internal decision logic |
| tool\_use block      | Tool abstraction        |
| tool\_result block   | observation             |
| while loop           | AgentExecutor           |
| run\_tools()         | automatic execution     |

***

# Summary

* LangChain abstracts:
  * loop
  * tool routing
  * state management
* Execution model remains:
  ```
  LLM → tool → result → LLM → ... → final
  ```
* AgentExecutor is the loop controller
* Tools are invoked based on model decisions, not explicit branching logic
