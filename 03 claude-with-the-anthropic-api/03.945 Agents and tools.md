### Core concepts

* **Agents vs Workflows**
  * **Workflows**: predefined sequence of steps (deterministic, reliable)
  * **Agents**: goal-driven + tool-based (dynamic, adaptive)
  * Agents decide **how to solve**, not just execute predefined steps [\[Agents and tools\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287803)

* **Agent paradigm**
  * Input: *goal + toolset*
  * Claude:
    * Plans actions
    * Selects tools
    * Chains tool calls
  * Output: achieves objective without fixed flow [\[Agents and tools\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287803)

* **Trade-offs**
  * ✅ Flexibility (handle unpredictable tasks)
  * ✅ Reusability (single agent supports many use cases)
  * ❌ Lower reliability vs workflows
  * ❌ Higher cost (more reasoning + tool calls) [\[Agents and tools\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287803)

***

### Tool orchestration (key mechanism)

* Example tool set:
  ```
  get_current_datetime
  add_duration_to_datetime
  set_reminder
  ```
* Behavior:
  * Simple query → single tool
  * Complex query → chain tools
  * Missing info → ask user before execution [\[Agents and tools\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287803)

### Execution patterns

| User request                  | Tool behavior                      |
| ----------------------------- | ---------------------------------- |
| "What's the time?"            | call`get_current_datetime`         |
| "Day in 11 days?"             | chain: current date → add duration |
| "Set reminder next Wednesday" | combine multiple tools             |
| Missing data                  | ask clarification                  |

### Tool Design Principles

#### 1. Abstract > Specialized

* Avoid:
  * `refactor_code`
  * `install_dependencies`
* Prefer:
  * `bash`, `read`, `write`, `edit`, `grep`, `glob` [\[Agents and tools\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287803)

➡ Benefit:

* Enables **generalization**
* Supports **unknown future tasks**

***

#### 2. Composability (critical)

* Tools must be:
  * Simple
  * Reusable
  * Chainable

Example (media agent):

```
bash (FFMPEG)
generate_image
text_to_speech
post_media
```

→ Agent can:

* Generate preview
* Ask for approval
* Compose video
* Publish content [\[Agents and tools\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287803)

***

### Mental Model (important)

```
Workflows:
  Step1 → Step2 → Step3 → Output

Agents:
  Goal + Tools →
       reasoning →
       dynamic tool selection →
       iterative execution →
  Output
```

## 2. Q\&A

**Q1. What defines an agent?**  
A system where Claude is given a goal and tools, and decides how to combine them to achieve the objective.

**Q2. Difference between workflow and agent?**

* Workflow: fixed sequence
* Agent: dynamic decision-making

**Q3. Why are tools critical?**  
They provide capabilities that Claude can orchestrate to solve tasks.

**Q4. Why prefer abstract tools?**  
They allow broader reuse and adaptation to unseen tasks.

**Q5. What happens when information is missing?**  
Claude asks clarifying questions before continuing.

**Q6. What is the key strength of agents?**  
Handling **unpredictable, multi-step tasks**.

**Q7. Main drawback of agents?**  
Lower reliability and higher cost compared to workflows.

## 4. Key Takeaways

* Agent = **Goal + Tools + Reasoning loop**
* Tools are the **execution layer**
* Design tools:
  * Abstract
  * Composable
* Agent strength:
  * Adaptive problem solving
* Trade-off:
  * Flexibility vs reliability/cost
* Use agents when:
  * Problem steps are **unknown or dynamic**
