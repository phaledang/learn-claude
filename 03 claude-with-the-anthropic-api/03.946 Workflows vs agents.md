## 1. Summary

**Page:** [Workflows vs agents](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287794)

### Core concept

* Two architectural patterns for AI apps using Claude:
  * **Workflows**
  * **Agents**

### Workflows

* Definition:
  * Predefined sequence of steps (Claude calls) to solve known problems
* Characteristics:
  * Deterministic flow
  * Each step = focused subtask
* Benefits:
  * Higher accuracy (focus per step)
  * Easier testing & evaluation
  * Predictable execution
  * Reliable for production
* Drawbacks:
  * Low flexibility
  * Requires known inputs + predefined flow
  * More upfront design effort

### Agents

* Definition:
  * Claude receives tools → dynamically plans how to solve tasks
* Characteristics:
  * Adaptive, non-deterministic
  * Plans + decides tool usage at runtime
* Benefits:
  * Flexible user experience
  * Can handle unknown / novel tasks
  * Combines tools dynamically
  * Can request more input when needed
* Drawbacks:
  * Lower success rate vs workflows
  * Hard to test & evaluate
  * Less predictable behavior

### Recommendation (important)

* **Prefer workflows by default**
* Use **agents only when necessary**
  * when requirements are unpredictable
  * when task cannot be predefined

## 2. Q\&A

**Q1: What is a workflow?**  
A predefined sequence of Claude calls to solve a known task.

**Q2: When should workflows be used?**  
When the task flow is known and well-defined.

**Q3: What is an agent?**  
A system where Claude plans and uses tools dynamically without predefined steps.

**Q4: What advantage do agents provide?**  
Flexibility and ability to handle novel/unexpected tasks.

**Q5: Why are workflows more reliable?**  
Because execution steps are fixed and predictable.

**Q6: Why are agents harder to evaluate?**  
Because the execution path is not known in advance.

**Q7: What is the main trade-off?**

| Dimension      | Workflows | Agents |
| -------------- | --------- | ------ |
| Predictability | High      | Low    |
| Flexibility    | Low       | High   |
| Reliability    | High      | Lower  |
| Evaluation     | Easy      | Hard   |

## 4. Key Takeaways

* **Workflows = deterministic pipelines**
  * best for production, stability, accuracy

* **Agents = dynamic planners**
  * best for exploration, unknown tasks

* **Trade-off = Reliability vs Flexibility**

* **Engineering principle:**
  * Start with workflow
  * Introduce agent **only when required**

* **Design heuristic:**
  ```text
  If you can define steps → Workflow
  If you cannot define steps → Agent
  ```
