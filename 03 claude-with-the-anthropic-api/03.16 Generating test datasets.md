* Part of a **prompt evaluation workflow**:
  1. Define a prompt
  2. Generate test dataset
  3. Run evaluations
  4. Analyze outputs

## 🔹 Key concepts

* **Evaluation dataset**
  * A collection of **input tasks** used to test a prompt.
  * Each dataset item is a **JSON object** with a `"task"` field.

* **Use case example**
  * Prompt generates:
    * Python code
    * JSON configs
    * Regex
  * Output must be:
    * Clean (no explanation/header/footer)

* **Dataset creation approaches**
  * ✅ Manual creation
  * ✅ Auto-generation using Claude (recommended)

* **Using Claude for dataset generation**
  * Use a **fast model (e.g., Haiku)** for efficiency.
  * Ask Claude to generate structured test cases.

* **Code workflow**
  * Build helper functions:
    * `add_user_message`
    * `add_assistant_message`
    * `chat`

# ❓ 2. Q\&A from the page

### Q1. What is an evaluation dataset?

**A:** A set of structured inputs (tasks) used to test how well a prompt performs.

### Q2. What format is used for datasets?

**A:** JSON array of objects, each containing a `"task"` field.

### Q3. Why generate datasets with Claude?

**A:** It automates test case creation and speeds up evaluation setup.

### Q4. Why use a smaller model like Haiku?

**A:** It is faster and cheaper for generating datasets.

### Q5. What is prefilling used for?

**A:** To guide Claude to output in the correct format (e.g., JSON).

### Q6. What are stop sequences used for?

**A:** To stop generation at the right point (e.g., end of JSON block).

### Q7. What is the goal of the example prompt?

**A:** Generate AWS-related Python, JSON, or regex outputs.
