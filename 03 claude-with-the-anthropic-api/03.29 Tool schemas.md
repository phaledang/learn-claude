
# 📘 **1. Summary (with URL)**

**Source:** [Tool schemas lesson](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287753)

### 🔑 Core concept

* After defining a tool function, you must create a **JSON schema** to describe it. [\[Tool schemas\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287753)
* This schema acts as **documentation for Claude**, helping it understand:
  * What the tool does
  * When to use it
  * What inputs it expects [\[Tool schemas\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287753)

### 🧱 Tool schema structure

A complete schema has **3 main components**:

* `name` → descriptive tool name
* `description` → explains usage and behavior
* `input_schema` → JSON schema defining parameters [\[Tool schemas\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287753)

### 🧠 Why JSON Schema?

* Not AI-specific → widely used for **data validation**
* Adopted in AI because it clearly defines inputs and expectations [\[Tool schemas\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287753)

### ✍️ Best practices for descriptions

* Write **3–4 clear sentences**
* Include:
  * What the tool does
  * When Claude should use it
  * What it returns
* Provide **detailed descriptions for each argument** [\[Tool schemas\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287753)

### ⚡ Faster way to create schemas

* Use **Claude itself**:
  * Paste your function
  * Ask Claude to generate a JSON schema
* Example prompt:
  > “Write a valid JSON schema spec for tool calling for this function…” [\[Tool schemas\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287753)

### 🧩 Implementation pattern

* Store schema alongside function:

```python
get_current_datetime_schema = {
  "name": "get_current_datetime",
  "description": "...",
  "input_schema": {...}
}
```

* Naming convention:
  ````
  function_name + "_schema"
  ``` 【1-d8604d】  
  ````

### 🛡️ Type safety

* Optional but recommended:

```python
from anthropic.types import ToolParam
```

* Helps prevent **type errors** and improves robustness [\[Tool schemas\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287753)

# ❓ **2. Q\&A**

### Q1: What is the purpose of a tool schema?

A: It documents the tool so Claude understands how and when to use it.

### Q2: What are the three main parts of a tool schema?

A: `name`, `description`, and `input_schema`.

### Q3: Why is JSON Schema used?

A: It is a standard format for describing and validating structured data.

### Q4: What makes a good tool description?

A: Clear explanation of:

* What the tool does
* When to use it
* What it returns

### Q5: How can you quickly generate a schema?

A: Ask Claude to create it from your function code.

### Q6: What naming convention is recommended?

A: `function_name_schema`.

### Q7: What is ToolParam used for?

A: Adding type safety to schemas.
