**Page:** [Accessing resources](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287783)

### Core concept

* **MCP Resources** = a way for the server to expose data that can be **directly injected into prompts**, instead of requiring tool calls. [\[Accessing resources\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287783)
* This enables **faster and more efficient context provisioning** for Claude. [\[Accessing resources\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287783)

### Flow (high-level)

* User references resource → e.g. `@report.pdf`
* MCP client:
  * Fetches resource from server
  * Parses content
  * Injects content into prompt sent to Claude  
    → No tool call required inside the model loop [\[Accessing resources\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287783)

### Key implementation

```python
async def read_resource(self, uri: str) -> Any:
    result = await self.session().read_resource(AnyUrl(uri))
    resource = result.contents[0]
```

* Response contains `contents[]`
* Typically use **first element only** [\[Accessing resources\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287783)

### Content handling

```python
if isinstance(resource, types.TextResourceContents):
    if resource.mimeType == "application/json":
        return json.loads(resource.text)
    return resource.text
```

* MIME type determines parsing strategy:
  * `application/json` → parse as object
  * others → return raw text [\[Accessing resources\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287783)

### Required imports

```python
import json
from pydantic import AnyUrl
```

### Key advantage

* **Direct prompt injection → no tool round-trip → lower latency & simpler flows** [\[Accessing resources\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287783)

## 2. ❓ Q\&A

**Q1. What is the purpose of MCP resources?**

* To expose data that can be directly included in prompts without tool calls. [\[Accessing resources\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287783)

**Q2. How does the client access a resource?**

* By calling `read_resource(uri)` via MCP session. [\[Accessing resources\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287783)

**Q3. What does the server response contain?**

* A `contents` list containing resource data + metadata. [\[Accessing resources\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287783)

**Q4. Why use MIME type?**

* To determine how to parse the resource content. [\[Accessing resources\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287783)

**Q5. What happens when user types `@file`?**

* System fetches the resource and injects it into the prompt automatically. [\[Accessing resources\]](https://anthropic.skilljar.com/claude-with-the-anthropic-api/287783)

## 4. 📌 Key Takeaways

* MCP Resources = **context injection layer (not tool layer)**
* Eliminates:
  * tool invocation latency
  * multi-step loops
* Requires:
  * client-side `read_resource` implementation
  * MIME-aware parsing
* Best suited for:
  * documents
  * static knowledge
  * large context inputs

👉 Conceptually:

* **Tools** → dynamic actions
* **Resources** → static data injection
