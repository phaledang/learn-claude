# Context Management - Claude Code 101

## Summary
Context is Claude's working memory, including prompts, files, and tool interactions.

### Context Window
- Limited memory capacity
- Everything added consumes space

### When Context Fills
- Auto compaction occurs
- Important info kept
- Less relevant info removed
- Possible data loss

### Commands
- /compact → summarize and free space
- /clear → reset all context
- /context → view usage breakdown

### Usage Guidelines
- Use /compact:
  - When continuing a feature
- Use /clear:
  - When starting new work

### Best Practices
- Be specific in prompts
- Disable unused MCP servers
- Use Skills instead of MCP servers when possible
- Use subagents for isolated tasks

### Optimization Insight
Vague prompts increase context usage due to additional exploration.

## Q&A

**Q: What is context?**  
A: Claude’s working memory.

**Q: What is the context window?**  
A: Limited memory space for context.

**Q: What happens when full?**  
A: Context is compacted.

**Q: What does /compact do?**  
A: Summarizes context.

**Q: What does /clear do?**  
A: Clears all context.

**Q: How to check context?**  
A: /context command.

**Q: Why avoid vague prompts?**  
A: They increase context usage.

**Q: What are subagents?**  
A: Agents with separate context windows.
