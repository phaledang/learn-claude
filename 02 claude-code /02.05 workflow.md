# The Explore → Plan → Code → Commit Workflow

Source: https://anthropic.skilljar.com/claude-code-101/469792

---

## Summary

The recommended workflow for Claude Code consists of four steps:

1. Explore
2. Plan
3. Code
4. Commit

This workflow helps reduce errors and avoids rework caused by jumping directly into coding.

---

## Workflow Details

### 1. Explore
- Claude reads files to understand the project context
- Can use plan mode or explore subagent

### 2. Plan
- Done in Plan Mode
- Claude cannot edit files
- Produces a structured implementation plan
- Best stage for making corrections

### 3. Code
- Execute after plan approval
- Best practices:
  - Define success criteria
  - Add tools
  - Provide test suites

### 4. Commit
- Run code review subagent
- Generate commit message
- Push code

---

## Key Concepts

- Plan Mode = analysis without execution
- Subagents = independent reviewers
- Tests = validation mechanism
- CLAUDE.md = memory for recurring fixes

## Q&A

**Q: What is the workflow?**  
A: Explore → Plan → Code → Commit  

**Q: Why plan first?**  
A: To reduce rework  

**Q: What does Plan Mode do?**  
A: Reads and plans without editing  

**Q: How to improve coding accuracy?**  
A: Use success criteria and test suites  

**Q: Why review with subagent?**  
A: To get unbiased feedback
