## Summary

Claude Code provides built-in features to streamline the Git workflow:
- Subagent-based code review
- Automated commit → push → PR flow
- Session linking to PRs for resuming work

---

## Key Concepts

### 1. Subagent for Code Review
- Runs in a separate context (unbiased review)
- Should be restricted to read-only tools
- Flags issues instead of modifying code
- Configuration should be committed to repo for team consistency

---

### 2. /commit-push-pr Skill
Automates:
- Commit
- Push
- Pull Request creation

Additional:
- Can post PR link to Slack if MCP Slack server is configured

---

### 3. Session Linking (`--from-pr`)

Resume working on a PR:
```
claude --from-pr <PR_NUMBER>
```
- Claude automatically links session to PR
- Enables continuation of work (fix comments, builds, etc.)

---

## Q&A

**Q: Why use a subagent for code review?**  
A: To ensure unbiased review using a fresh context.

**Q: What restriction should a review subagent have?**  
A: It should use read-only tools.

**Q: What does `/commit-push-pr` do?**  
A: Automates commit, push, and PR creation.

**Q: Can Claude notify a team about a PR?**  
A: Yes, via Slack MCP integration.

**Q: How do you resume work on a PR?**  
A: Use `claude --from-pr <PR_NUMBER>`.
