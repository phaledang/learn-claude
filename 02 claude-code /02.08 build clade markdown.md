# CLAUDE.md Training Guide

## 🔗 Source
- https://anthropic.skilljar.com/claude-code-101/469795

---

## 🧠 Summary
CLAUDE.md is a Markdown file used in Claude Code to provide persistent memory about a project. It ensures Claude starts with full context instead of relearning the codebase every session.

Think of it as an onboarding script for your codebase.

---

## ❗ Problem It Solves

### Without CLAUDE.md:
- Claude re-explores the codebase every session  
- Needs to rediscover dependencies  
- May make incorrect assumptions  

### With CLAUDE.md:
- Context is preloaded  
- Outputs are more accurate  
- Less repetition required  

---

## ⚙️ How It Works
- Located in the project root directory  
- Automatically appended to every prompt  
- Provides:
  - Project stack  
  - Commands  
  - Coding conventions  

---

## 📄 Example CLAUDE.md

# Project
Next.js 15 app using Tailwind + Drizzle ORM

# Commands
- pnpm dev
- pnpm test
- pnpm lint

# Code Style
- 2-space indentation
- Named exports
- API routes in app/api/

---

## 🏢 Memory Hierarchy

### Project-level CLAUDE.md
- Stored in project root  
- Shared with team  
- Should be committed to version control  

### User-level CLAUDE.md
- Stored in local config  
- Personal preferences  
- Applies across all projects  

---

## 💡 Best Practices
- Save repeated corrections into CLAUDE.md  
- Use `@file` references:
  - Example: @README.md  
- Start without CLAUDE.md → identify gaps → then create  
- Use `/init` to generate a starter file

## ❓ Q&A

**Q: What is CLAUDE.md?**  
A: A Markdown file providing persistent context for Claude Code.

**Q: Where is it located?**  
A: In the root directory of the project.

**Q: Why use it?**  
A: To prevent Claude from relearning the project every session.

**Q: What should it contain?**  
A: Stack, commands, and coding conventions.

**Q: How does Claude use it?**  
A: It is appended automatically to every prompt.

**Q: What is the difference between project and user-level?**  
A: Project-level is shared; user-level is personal.

**Q: How to create one quickly?**  
A: Use `/init`.
