# 2. Key Concepts

## Learning Objectives

After completing this lesson, learners should be able to:

* Open Claude Cowork in the Claude Desktop application.
* Connect Claude to a working folder.
* Connect external applications and data sources.
* Understand Claude's permission model.
* Delegate work confidently while maintaining control.

***

## What is Claude Cowork Setup?

Claude Cowork is a mode inside the Claude Desktop application that can:

* Read project files.
* Create new files.
* Modify existing files.
* Save outputs directly to the user's computer.
* Access connected business applications.

This makes Cowork different from traditional chat interactions.

### Chat vs Cowork

| Feature                     | Chat    | Cowork    |
| --------------------------- | ------- | --------- |
| Read uploaded files         | ✅       | ✅         |
| Create files                | ❌       | ✅         |
| Edit files                  | ❌       | ✅         |
| Save outputs back to folder | ❌       | ✅         |
| Work across project context | Limited | Extensive |

***

## The Most Important Setup Decision: Choose a Working Folder

### Why the Folder Matters

The working folder is:

* Cowork's workspace.
* The location where project files live.
* The location where outputs are saved.

Cowork can:

* Read files inside the folder.
* Modify files.
* Create new files.
* Organize project artifacts.

### Recommended Practice

Choose a folder that contains:

* Project documents
* Source materials
* Templates
* Related references

Avoid:

* Giving access to your entire Documents folder.
* Using folders that contain unrelated work.

### Key Principle

> Cowork is only as useful as the context available inside the working folder.

***

## Local Folder vs Cloud Files

### Local Folder

Recommended for:

* Drafting documents
* Iterative work
* Creating deliverables
* Editing project artifacts

Provides reliable read/write access.

### Cloud Connectors

Different connectors offer different capabilities:

Some are:

* Search-only
* Read-only

Others may support:

* File creation
* Updates
* Edits

Always verify connector capabilities before using them.

# Connectors

## What Are Connectors?

Connectors allow Cowork to access work systems directly.

Examples include:

* Email
* Calendar
* Team messaging
* CRM systems
* Cloud storage

Configuration is done once in the **Customize** area.

Connectors can then be enabled or disabled per task.

# Claude in Chrome

Some systems lack native connectors.

Examples:

* Internal dashboards
* Vendor portals
* Custom web applications

Claude in Chrome acts as a bridge by allowing Claude to:

* Read pages
* Interact with websites
* Access information behind browser sessions

Important:

* Availability may vary by enterprise plan.

# Permissions Model

A major design goal is:

> Delegate work safely without losing control.

***

## Permission Mode 1: Ask Before Acting

Default mode.

Claude pauses before actions such as:

* Sending emails
* Posting messages
* Sharing files

Best for:

* New workflows
* Unfamiliar tools
* Higher-risk operations

Benefits:

* Maximum visibility
* Human supervision

***

## Permission Mode 2: Act Without Asking

Claude performs actions without pausing.

Best for:

* Trusted workflows
* Trusted files
* Direct supervision

Benefits:

* Faster execution

Trade-off:

* Increased operational risk

***

## Permanent Deletion Protection

A critical safeguard:

> Claude always asks before permanently deleting files.

This protection applies in all modes.

There are no exceptions.

***

## Additional Controls

Users can control:

### Connectors

* Which connectors are available.
* Connector-specific permissions.

### MCPs

* Which MCP integrations can be used.
* Permission frequency.

### Web Access

Administrators can:

* Disable web search.
* Restrict allowed websites.

Users should extend access only to trusted systems.

# Delegating, Not Just Chatting

The core Cowork value proposition is:

### Traditional Chat

* Ask questions.
* Receive answers.

### Cowork

* Delegate substantial work.
* Give access to project context.
* Receive completed outputs.

This becomes practical because:

* Context is available.
* Files are editable.
* Permissions are controlled.

# First Task Walkthrough

## Step 1: Select a Real Project Folder

Use:

* Client work
* Active projects
* Current deliverables

Avoid toy examples.

Real context creates better outcomes.

***

## Step 2: Connect One Application

Recommended starting points:

* Email
* Cloud storage

These often contain the highest-value context.

***

## Step 3: Request a Folder Summary

Example prompt:

> Review everything in this folder and create a brief explaining:
>
> * What documents exist
> * Relationships between files
> * Important insights
> * Relevant observations

***

## Step 4: Validate the Results

Review:

* What Claude discovered
* What it missed
* What surprised you

Use the findings to calibrate trust and improve future delegation.

# 3. Q\&A

### Q1. What is the most important setup choice in Claude Cowork?

**Answer:** Selecting the correct working folder because it determines what files Claude can read, edit, create, and save.

***

### Q2. How is Cowork different from Chat?

**Answer:** Cowork can directly read, create, edit, and save files in a project folder, while Chat cannot save changes back to a computer.

***

### Q3. Why should folders be project-specific?

**Answer:** Project-specific folders provide relevant context while minimizing unnecessary access.

***

### Q4. What do connectors do?

**Answer:** Connectors allow Claude to access external systems such as email, calendars, messaging platforms, cloud storage, and CRM tools.

***

### Q5. Can all connectors edit data?

**Answer:** No. Some connectors only provide read/search access, while others allow creation or editing.

***

### Q6. What is Claude in Chrome used for?

**Answer:** It allows Claude to read and interact with websites and web applications that do not have native connectors.

***

### Q7. When should "Ask Before Acting" be used?

**Answer:** For new tools, unfamiliar workflows, or situations requiring close supervision.

***

### Q8. When should "Act Without Asking" be used?

**Answer:** For trusted workflows and environments where faster execution is desired.

***

### Q9. What action always requires approval?

**Answer:** Permanent file deletion.

***

### Q10. Why is real project data recommended for the first task?

**Answer:** Cowork performs best when provided with realistic context and meaningful project information.
