## Understanding Standing Context

Claude Cowork uses two primary layers of persistent context:

| Layer               | Purpose                           | Scope                    |
| ------------------- | --------------------------------- | ------------------------ |
| Global Instructions | Personal working preferences      | All Cowork sessions      |
| Projects            | Context for a specific workstream | Only within that project |

### Key Difference from Chat

* Chat memory can build automatically across conversations.
* Cowork relies primarily on:
  * Global Instructions
  * Project Memory

This gives more control over what Claude remembers and where.

***

# Global Instructions

## What Are Global Instructions?

A permanent briefing document that Claude consults during:

* Chats
* Scheduled Tasks
* Dispatches

Think of it as:

> A standing operating manual about how you work.

***

## How to Configure

1. Open Claude Desktop.

2. Navigate to:

   ```
   Settings → Cowork
   ```

3. Select:

   ```
   Edit → Global Instructions
   ```

4. Write instructions.

5. Save.

***

## Recommended Content

### About You

* Job role
* Responsibilities
* Primary work areas

Example:

* Solution Architect
* AI projects
* Customer solution design

***

### Common Terminology

Add:

* Acronyms
* Internal terms
* Frequently used shorthand

Examples:

* QBR
* PRD
* RAG
* MCP
* AI-SDLC

Benefit:

* Claude understands organization-specific language immediately.

***

### Output Preferences

Specify:

* Format
* Tone
* Length
* Structure

Examples:

* Use bullet points
* Executive summary first
* Markdown format
* Technical writing style
* Include risks and assumptions

***

## Example Product Manager Instructions

The lesson shows a PM example:

* Lead with user problem.
* Use decision-oriented writing.
* Cite metric sources.
* State assumptions explicitly.

Result:

* Claude automatically produces PRD-like outputs.
* Better consistency.
* Less prompting required.

***

## Evolving Global Instructions

Global instructions should improve over time.

Whenever repeatedly correcting Claude:

Examples:

* "Put recommendation first."
* "Don't use Oxford commas."
* "Use markdown tables."

Add those corrections into global instructions.

# Projects

## What Is a Project?

A workspace dedicated to a particular stream of work.

Examples:

* Customer account
* Product launch
* Weekly reporting
* Transformation initiative

Projects represent:

> What you're working on.

while Global Instructions represent:

> How you work.

***

## Four Components of a Project

### 1. Instructions

Project-specific guidance.

Example:

* Weekly transformation status reporting
* Create executive slide decks

***

### 2. Scheduled Tasks

Recurring automation.

Examples:

* Weekly status collection
* Monthly reporting
* Meeting preparation
* Renewal review preparation

***

### 3. Context

Resources available to every conversation.

Examples:

* Local folders
* Documents
* Links
* Knowledge repositories

***

### 4. Memory

Most important project capability.

Claude automatically learns:

* Client preferences
* Decisions
* Open items
* Historical context

This memory grows naturally from conversations.

***

## Why Projects Matter

Outside a Project:

* Sessions begin mostly fresh.
* Only Global Instructions carry forward.

Inside a Project:

* Conversations accumulate knowledge.
* Context improves over time.
* Less repetitive explanation needed.

Claude already understands:

* Previous discussions
* Decisions
* Customer history
* Pending work

# Ideal Project Candidates

## 1. Customer Accounts

Use when working repeatedly with one customer.

Benefits:

* Meeting preparation
* Follow-ups
* Reviews
* Account history

remain in one place.

***

## 2. Recurring Deliverables

Examples:

* Weekly leadership reports
* Monthly reviews
* Quarterly business reviews

Benefit:

Each cycle builds on previous cycles.

***

## 3. Launches & Initiatives

Examples:

* Platform migration
* Product redesign
* New program rollout

Benefit:

All planning, decisions, and updates stay together.

***

# Three Ways to Create a Project

## Option 1: Start From Scratch

Best when:

* Work is brand new.
* No existing materials.

***

## Option 2: Start From Existing Folder

Best when:

* Documents already exist.

The folder becomes:

> Project working directory.

***

## Option 3: Convert from Chat Project

Migrate:

* Instructions
* Existing knowledge

Important:

* Migration is one-way.
* Changes in Cowork do not sync back to Chat.

***

# Practical Setup Exercise

## Step 1: Create Global Instructions

Suggested template:

```text
I'm a [role] at [company].

I work primarily on:
- [workstream 1]
- [workstream 2]

Common acronyms:
- ...

Deliverables:
- documents
- presentations
- technical designs

Preferences:
- concise updates
- recommendation first
- markdown output
- include assumptions
```

***

## Step 2: Create a Project

Choose a recurring work area such as:

* Customer account
* AI initiative
* Architecture review process
* Monthly reporting stream

Add:

* Instructions
* Folder
* Links
* Context

Then work exclusively within that project.

# Key Takeaways

* **Global Instructions = Personal operating manual**
* **Projects = Workstream-specific workspace**
* **Memory is project-scoped**
* **Projects accumulate organizational knowledge**
* **Use projects for recurring work**
* **Use global instructions for personal preferences**
* **Continuously refine instructions based on repeated corrections**

# 3. Q\&A

### Q1. What is the purpose of Global Instructions?

To define personal working preferences that Claude applies across all Cowork sessions.

***

### Q2. What is the primary purpose of a Project?

To organize and persist context for a specific stream of work.

***

### Q3. What information should be placed in Global Instructions?

* Role
* Responsibilities
* Acronyms
* Output preferences
* Communication style

***

### Q4. What are the four components of a Project?

* Instructions
* Scheduled Tasks
* Context
* Memory

***

### Q5. How is Project Memory created?

Automatically from conversations inside the project.

***

### Q6. When should a Project be created?

When work is recurring:

* Customer account
* Monthly report
* Product launch
* Long-term initiative

***

### Q7. Can Chat projects synchronize with Cowork projects?

Only one-way from Chat to Cowork.

***

### Q8. Why are Projects more powerful than standalone chats?

They accumulate knowledge, decisions, and context over time.
