# How to Run Ollama as a Linux Service with systemd

**Source:** [How to Run Ollama as a Linux Service with systemd](https://mljourney.com/how-to-run-ollama-as-a-linux-service-with-systemd/) [\[How to Run...ML Journey\]](https://mljourney.com/how-to-run-ollama-as-a-linux-service-with-systemd/)

## Overview

Running Ollama as a systemd service provides:

* ✅ Automatic startup on boot
* ✅ Automatic restart on failure
* ✅ Background execution without a terminal
* ✅ Centralized logging with `journalctl`
* ✅ GPU support
* ✅ Production-ready deployment management

According to the article, this is the preferred approach for servers and long-running AI infrastructure. [\[How to Run...ML Journey\]](https://mljourney.com/how-to-run-ollama-as-a-linux-service-with-systemd/)

***

# 1. Check Existing Ollama Service

If Ollama was installed using the official installer:

```bash
systemctl status ollama
```

Expected status:

```text
active (running)
```

The official installer may already create the service automatically. [\[How to Run...ML Journey\]](https://mljourney.com/how-to-run-ollama-as-a-linux-service-with-systemd/)

***

# 2. Create a Systemd Service

Create the service definition:

```bash
sudo nano /etc/systemd/system/ollama.service
```

Add:

```ini
[Unit]
Description=Ollama LLM Service
After=network-online.target
Wants=network-online.target

[Service]
Type=exec
User=ollama
Group=ollama

ExecStart=/usr/local/bin/ollama serve

Restart=always
RestartSec=3

Environment="HOME=/usr/share/ollama"
Environment="OLLAMA_MODELS=/usr/share/ollama/.ollama/models"

[Install]
WantedBy=default.target
```

 [\[How to Run...ML Journey\]](https://mljourney.com/how-to-run-ollama-as-a-linux-service-with-systemd/)

***

# 3. Create a Dedicated Ollama User

Recommended for security:

```bash
sudo useradd -r \
  -s /bin/false \
  -U \
  -m \
  -d /usr/share/ollama \
  ollama
```

Benefits:

* Least-privilege execution
* No interactive login
* Separation from root/admin account

 [\[How to Run...ML Journey\]](https://mljourney.com/how-to-run-ollama-as-a-linux-service-with-systemd/)

# 4. Enable and Start Ollama

```bash
sudo systemctl daemon-reload
sudo systemctl enable ollama
sudo systemctl start ollama
```

Verify:

```bash
sudo systemctl status ollama
```

 [\[How to Run...ML Journey\]](https://mljourney.com/how-to-run-ollama-as-a-linux-service-with-systemd/)

***

# 5. Configure NVIDIA GPU Support

Add Ollama user to GPU groups:

```bash
sudo usermod -aG video ollama
sudo usermod -aG render ollama
```

Optional GPU settings:

```ini
Environment="LD_LIBRARY_PATH=/usr/local/lib:/usr/lib"
Environment="CUDA_VISIBLE_DEVICES=0"
```

Reload service:

```bash
sudo systemctl daemon-reload
sudo systemctl restart ollama
```

Verify CUDA detection:

```bash
sudo journalctl -u ollama -n 20 | grep -i cuda
```

# 6. Useful Environment Variables

## Remote Access

```ini
Environment="OLLAMA_HOST=0.0.0.0:11434"
```

Allows access from other machines.

## Custom Model Storage

```ini
Environment="OLLAMA_MODELS=/data/ollama/models"
```

## Keep Models Loaded

```ini
Environment="OLLAMA_KEEP_ALIVE=1h"
```

## Concurrent Model Loading

```ini
Environment="OLLAMA_MAX_LOADED_MODELS=2"
```

## Parallel Requests

```ini
Environment="OLLAMA_NUM_PARALLEL=2"
```

## Flash Attention

```ini
Environment="OLLAMA_FLASH_ATTENTION=1"
```

# 8. Auto-Pull Models at Startup

Create:

```bash
sudo nano /etc/systemd/system/ollama-setup.service
```

Example:

```ini
[Unit]
Description=Pull required Ollama models
After=ollama.service
Requires=ollama.service

[Service]
Type=oneshot
User=ollama

ExecStart=/bin/bash -c '
until curl -sf http://localhost:11434/ > /dev/null;
do sleep 1;
done;

ollama pull llama3.2
ollama pull nomic-embed-text
'

RemainAfterExit=yes

[Install]
WantedBy=multi-user.target
```

Enable:

```bash
sudo systemctl daemon-reload
sudo systemctl enable ollama-setup
```

# 11. Troubleshooting

## Service Running but API Not Responding

Check logs:

```bash
journalctl -u ollama -n 50
```

Potential causes:

* Port conflicts
* Permissions issues

## GPU Not Detected

Verify:

```bash
id ollama
nvidia-smi
```

## Models Missing After Restart

Check:

* `OLLAMA_MODELS`
* Directory permissions

## Service Fails to Start

Run manually:

```bash
sudo -u ollama /usr/local/bin/ollama serve
```

 [\[How to Run...ML Journey\]](https://mljourney.com/how-to-run-ollama-as-a-linux-service-with-systemd/)

# 12. Remote Access Configuration

Listen on all interfaces:

```ini
Environment="OLLAMA_HOST=0.0.0.0:11434"
```

Verify:

```bash
ss -tlnp | grep 11434
```

Test from another machine:

```bash
curl http://SERVER_IP:11434/
```

⚠️ The article notes that exposing Ollama publicly without authentication can be a security risk. Consider firewall rules or a reverse proxy. [\[How to Run...ML Journey\]](https://mljourney.com/how-to-run-ollama-as-a-linux-service-with-systemd/)

***

# 13. Upgrade Ollama

```bash
sudo systemctl stop ollama

curl -fsSL https://ollama.com/install.sh | sudo sh

sudo systemctl start ollama

sudo systemctl status ollama
```

Model files remain intact if stored in `OLLAMA_MODELS`. [\[How to Run...ML Journey\]](https://mljourney.com/how-to-run-ollama-as-a-linux-service-with-systemd/)

# Recommended WSL2 + Ollama Setup

Based on your open tabs (Docker on WSL2, Claude Desktop Linux, Ollama service), a practical setup is:

```text
Windows 11
 ├─ Claude Desktop
 ├─ Browser
 └─ WSL2 Ubuntu
      ├─ Ollama (systemd service)
      ├─ Qwen / Llama models
      ├─ Docker
      └─ MCP servers
```

Benefits:

* Ollama auto-starts after reboot
* No need to manually run `ollama serve`
* Compatible with Claude Desktop MCP integrations
* Easier local AI development workflow
