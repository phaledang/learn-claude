## 2. Key Concepts

### 2.1 Overview

* **StreamableHTTP transport**
  * Enables MCP clients to connect to **remote MCP servers via HTTP**
  * Unlike STDIO transport → does **not require same machine**
  * Allows **public MCP servers** accessible over network [\[The Stream...transport\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296287)

***

### 2.2 Comparison with STDIO transport

* **STDIO transport**
  * Client + server must run on **same machine**
  * Full feature support (bidirectional interaction)
* **StreamableHTTP transport**
  * Supports **remote deployment**
  * Has **limitations due to HTTP protocol constraints** [\[The Stream...transport\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296287)

***

### 2.3 Key Configuration Settings

#### stateless\_http

* Controls **connection state management**
* When `true`:
  * No persistent state between requests
  * Communication becomes **strict request → response**

#### json\_response

* Controls **response format handling**
* When `true`:
  * Responses constrained to JSON format

#### Important behavior

* Default: both = **false**
* Some deployments may require setting them to **true**
* Enabling them can **break core MCP features** [\[The Stream...transport\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296287)

***

### 2.4 What breaks when restrictions are enabled

* Progress notifications → ❌ disappear
* Logging → ❌ stops working
* Server-initiated requests → ❌ fail

Reason:

* HTTP **cannot initiate communication from server → client** [\[The Stream...transport\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296287)

***

### 2.5 HTTP Communication Model (Root Cause)

* HTTP supports:
  * ✅ Client → Server request
  * ✅ Server → Client response

* HTTP does NOT support:
  * ❌ Server → Client request initiation

Impact:

* MCP features relying on **server push / bidirectional flow** break [\[The Stream...transport\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296287)

***

### 2.6 Affected MCP Message Types

#### Server-initiated requests

* Create Message requests
* List Roots requests

#### Notifications

* Progress notifications
* Logging notifications
* Initialized notifications
* Cancelled notifications

➡️ These rely on **server → client communication**, which HTTP cannot handle natively [\[The Stream...transport\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296287)

***

### 2.7 StreamableHTTP Trade-offs

#### Advantages

* Enables:
  * Remote hosting
  * Public MCP servers
  * Scalable deployment

#### Trade-offs

* Must sometimes:
  * Accept **reduced functionality**
  * Adapt to **HTTP limitations**

***

### 2.8 Design Considerations (Architecture Level)

When using StreamableHTTP:

* Choose transport based on:
  * Deployment needs (local vs remote)
  * Feature requirements

* If application depends on:
  * Real-time updates
  * Server-initiated workflows

➡️ Consider:

* Avoid restrictive configs
* Or rethink communication patterns [\[The Stream...transport\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296287)
