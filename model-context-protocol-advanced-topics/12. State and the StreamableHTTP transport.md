## 2. Key Concepts

### 2.1 Core Idea: State vs Stateless in MCP (Model Context Protocol)

* MCP servers can operate in **stateful** or **stateless HTTP mode**
* Controlled by flags:
  * `stateless_http=True`
  * `json_response=True`
* These flags **fundamentally change server behavior**, especially for scaling and communication [\[anthropic....ics/296285\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296285)

***

### 2.2 MCP Communication Model (Default / Stateful)

* MCP clients use **two connections**:
  * **GET (SSE)** → server → client communication
  * **POST** → client → server tool invocation
* Enables:
  * Tool execution with streaming
  * AI sampling (Claude)
  * Progress updates
  * Subscriptions / notifications [\[anthropic....ics/296285\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296285)

***

### 2.3 Scaling Problem (Why Stateless Exists)

* With **horizontal scaling + load balancer**:
  * GET and POST requests may hit **different server instances**
* Problem:
  * Server handling POST must coordinate with server handling SSE
* Result:
  * **Complex cross-instance coordination** [\[anthropic....ics/296285\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296285)

***

### 2.4 `stateless_http=True`

#### What it does

* Removes server-side session/state tracking

#### Behavior changes

* ❌ No session IDs (cannot track clients)
* ❌ No server → client requests (SSE disabled)
* ❌ No sampling (Claude cannot be used)
* ❌ No progress updates
* ❌ No subscriptions / notifications
* ✅ No client initialization required (no handshake) [\[anthropic....ics/296285\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296285)

#### Why it exists

* Eliminates coordination problem between instances
* Enables **clean horizontal scaling** behind load balancers [\[anthropic....ics/296285\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296285)

***

### 2.5 `json_response=True`

#### What it does

* Disables streaming for POST responses

#### Behavior changes

* ❌ No intermediate messages (progress, logs)
* ✅ Only final result returned as JSON [\[anthropic....ics/296285\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296285)

***

### 2.6 Trade-offs Summary

| Feature                 | Stateful (Default) | Stateless HTTP |
| ----------------------- | ------------------ | -------------- |
| Session tracking        | ✅ Yes              | ❌ No           |
| SSE (server → client)   | ✅ Yes              | ❌ No           |
| Sampling (Claude)       | ✅ Yes              | ❌ No           |
| Progress updates        | ✅ Yes              | ❌ No           |
| Subscriptions           | ✅ Yes              | ❌ No           |
| Horizontal scaling ease | ⚠️ Complex         | ✅ Simple       |
| Initialization required | ✅ Yes              | ❌ No           |

***

### 2.7 When to Use

#### Use `stateless_http=True` when:

* Need **horizontal scaling with load balancer**
* No need for:
  * AI sampling
  * Streaming/progress
  * subscriptions
* Want **simple request/response model** [\[anthropic....ics/296285\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296285)

#### Use `json_response=True` when:

* Want **non-streaming output**
* Integrating with systems expecting plain JSON [\[anthropic....ics/296285\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296285)

***

### 2.8 Development vs Production

* Test with **same transport as production**
* Stateful vs stateless differences are significant:
  * Bugs may only appear after switching transport [\[anthropic....ics/296285\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296285)

## 3. Q\&A

### Q1: Why does MCP need two connections?

* To separate:
  * Server-initiated messages (SSE)
  * Client tool calls (POST)

***

### Q2: Why is stateless HTTP important for scaling?

* Removes need to synchronize state across instances
* Avoids cross-server coordination complexity

***

### Q3: What is the biggest limitation of stateless mode?

* Cannot use **sampling (Claude)** and **SSE-based features**

***

### Q4: What problem does `json_response=True` solve?

* Simplifies integration by removing streaming behavior

***

### Q5: Can stateless mode still use AI models?

* No — sampling requires stateful coordination
