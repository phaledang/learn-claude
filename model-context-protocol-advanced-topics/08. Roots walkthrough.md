## 2. Key concepts

### 🔹 Roots (MCP concept)

* Defines **allowed file system scope** for MCP server access
* User explicitly controls **which directories/files AI can access**
* Acts as a **security boundary** to prevent unrestricted access [\[Roots walkthrough\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/295839)

***

### 🔹 1. Defining roots

* Roots are **provided by user via CLI arguments**
* Input = list of filesystem paths
* Example pattern:
  ```bash
  uv run main.py <root1> <root2> ...
  ```
* If no roots provided → program exits with usage message [\[Roots walkthrough\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/295839)

***

### 🔹 2. Passing roots to MCP client

* Root paths are:
  * Extracted from `sys.argv`
  * Passed into `MCPClient`
* MCP client uses them as **authorized access scope** [\[Roots walkthrough\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/295839)

***

### 🔹 3. Creating root objects

* Each path becomes a **root object**
* Represents:
  * A directory
  * A logical entry point for file access
* These objects are later used in MCP operations

***

### 🔹 4. Roots callback

* MCP server exposes a **callback to retrieve roots**
* Enables:
  * Dynamic querying of allowed paths
  * Standard interaction between client ↔ server

***

### 🔹 5. Using the roots

* Roots are used when:
  * Accessing files
  * Running tools
  * Querying data within allowed scope
* All operations must stay **within defined roots**

***

### 🔹 6. Accessing the roots

* System:
  * Looks up the correct root
  * Resolves path inside that root
* Prevents traversal outside allowed directories

***

### 🔹 7. Authorizing access

* Before any file operation:
  * Validate request is inside allowed roots
* Ensures:
  * No unauthorized file reading
  * Strong isolation between environments [\[Roots walkthrough\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/295839)

***

### 🔹 Code flow (simplified)

```python
root_paths = sys.argv[1:]

if not root_paths:
    print("Usage: uv run main.py <root1> [root2] ...")
    sys.exit(1)

# Pass to MCP client
client = MCPClient(roots=root_paths)
```

* Key idea: **runtime-defined security boundary**

***

### 🔹 Why Roots matter (critical insight)

* Security:
  * Prevents LLM/tools from accessing entire filesystem
* Principle:
  * **Least privilege access**
* Flexibility:
  * Different runs → different accessible scopes
* Alignment:
  * Enables safe enterprise/agent workflows

## 3. Q\&A

**Q1. What is a root in MCP?**

* A root is a user-defined filesystem path that MCP is allowed to access.

**Q2. How are roots provided?**

* Through CLI arguments when running the program.

**Q3. Why are roots important?**

* They enforce security boundaries and limit access scope.

**Q4. Who controls the roots?**

* The user (not the AI or server).

**Q5. What happens if no root is provided?**

* Program exits with usage instructions.
