* **Logging and progress notifications (MCP)**
  * Mechanism to provide **real-time feedback** during tool execution.
  * Improves **user experience** for long-running operations (e.g., research, data processing). [\[Log and pr...ifications\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296284)

* **Problem solved**
  * Without notifications: users see **no output until completion** → uncertainty (tool stalled vs running). [\[Log and pr...ifications\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296284)
  * With notifications: users see
    * progress updates
    * status messages
    * logs [\[Log and pr...ifications\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296284)

### MCP Server-side Implementation

#### Context Object

* Automatically injected into MCP tool functions
* Provides communication channel from **server → client during execution** [\[Log and pr...ifications\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296284)

#### Key APIs

* `context.info(message)`
  * Send **log messages**
* `context.report_progress(current, total)`
  * Send **progress updates** [\[Log and pr...ifications\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296284)

#### Example (Python MCP SDK)

```python
@mcp.tool(
    name="research",
    description="Research a given topic"
)
async def research(
    topic: str = Field(description="Topic to research"),
    *,
    context: Context
):
    await context.info("About to do research...")
    await context.report_progress(20, 100)

    sources = await do_research(topic)

    await context.info("Writing report...")
    await context.report_progress(70, 100)

    results = await generate_report(sources)
    return results
```

#### Execution Flow

* Emit log → emit progress → perform work → update → continue → return result

### MCP Client-side Implementation

#### Logging Callback

* Handles **log messages emitted by server**

```python
async def logging_callback(params):
    print(params.data)
```

#### Progress Callback

* Handles **progress updates**

```python
async def print_progress_callback(progress, total, message):
    if total is not None:
        percentage = (progress / total) * 100
        print(f"Progress: {percentage:.1f}%")
```

#### Attach Callbacks

```python
async with ClientSession(
    read,
    write,
    logging_callback=logging_callback
) as session:
    await session.initialize()

    await session.call_tool(
        name="add",
        arguments={"a": 1, "b": 3},
        progress_callback=print_progress_callback,
    )
```

#### Key Concept

* Server **emits notifications**
* Client **decides how to display them** [\[Log and pr...ifications\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296284)

## 3. Q\&A

### Q1. Why use logging and progress notifications?

* Provide visibility into long-running operations
* Prevent user uncertainty
* Improve trust and UX

### Q2. What enables notifications in MCP tools?

* The **Context object** injected into tool functions

### Q3. What are the two core notification methods?

* `context.info()` → logs
* `context.report_progress()` → progress

### Q4. Who decides how notifications are shown?

* The **client application**

### Q5. Are notifications required?

* No → optional UX enhancement
