## UV (Python) — Quick Introduction

### 1. What is **UV**

* **UV** is a **modern Python package + project manager** written in Rust [\[realpython.com\]](https://realpython.com/ref/tools/uv/), [\[datacamp.com\]](https://www.datacamp.com/tutorial/python-uv)
* Designed as an **all‑in‑one replacement** for multiple tools:
  * `pip` (install packages)
  * `virtualenv` (manage environments)
  * `pyenv` (manage Python versions)
  * `pip-tools`, `pipx`, `poetry` [\[docs.astral.sh\]](https://docs.astral.sh/uv/), [\[tutorials.technology\]](https://tutorials.technology/tutorials/uv-python-package-manager-2026.html)

👉 Think:

> **UV = Cargo (Rust) for Python**

***

### 2. Why UV (Key Value)

* ⚡ **Extremely fast**: \~10–100x faster than pip (Rust + parallel resolver) [\[docs.astral.sh\]](https://docs.astral.sh/uv/), [\[datacamp.com\]](https://www.datacamp.com/tutorial/python-uv)
* 📦 **Unified workflow**: one CLI tool for everything
* 🔁 **Reproducible builds**: built-in **lockfile (`uv.lock`)**
* 🧩 **Simpler dev experience**: no need to combine multiple tools
* 🌍 **Cross-platform consistency**

***

### 3. Core Capabilities

#### Dependency & Package Management

* Add/remove packages:
  ```bash
  uv add requests
  uv remove requests
  ```
* Automatically updates:
  * `pyproject.toml`
  * `uv.lock`

#### Virtual Environment Management

* Auto-created and managed (no manual activation) [\[pydevtools.com\]](https://pydevtools.com/handbook/explanation/whats-the-difference-between-pip-and-uv/)

#### Run Code / Scripts

```bash
uv run app.py
```

* Runs in an isolated environment automatically [\[docs.astral.sh\]](https://docs.astral.sh/uv/)

#### Python Version Management

```bash
uv python install 3.12
```

#### Tool Execution (like pipx)

```bash
uvx ruff check .
```

***

### 4. Project Structure (typical)

```text
project/
 ├── pyproject.toml     # project metadata & dependencies
 ├── uv.lock            # exact resolved versions (critical)
 ├── .venv/             # auto-managed virtual env
```

***

### 5. What is `uv.lock`

* Lockfile that stores:
  * exact versions of all dependencies (incl. transitive)
  * metadata for reproducible installs [\[pydevtools.com\]](https://pydevtools.com/handbook/how-to/how-to-use-a-uv-lockfile-for-reproducible-python-environments/)
* Guarantees:
  * same environment across machines / CI
* Usage:
  ```bash
  uv lock    # generate/update lockfile
  uv sync    # install exactly from lockfile
  ```

👉 Similar to:

* `package-lock.json` (Node)
* `Cargo.lock` (Rust)

***

### 6. UV vs pip (mental model)

| Feature             | pip              | uv                   |
| ------------------- | ---------------- | -------------------- |
| Install packages    | ✅                | ✅                    |
| Dependency locking  | manual / limited | ✅ built-in           |
| Virtual env         | external tools   | ✅ integrated         |
| Python version mgmt | external (pyenv) | ✅ integrated         |
| Performance         | slower           | ⚡ much faster        |
| Scope               | installer only   | full project manager |

👉 Summary:

> **pip = installer**  
> **uv = full Python development platform**

***

### 7. When to Use UV

* ✅ New Python projects (best default)

* ✅ CI/CD reproducibility required

* ✅ Need faster installs (large deps, AI stacks)

* ✅ Want to simplify dev tooling

* ⚠️ Consider alternatives if:
  * heavy non-Python ecosystem → Conda
  * strict compatibility with legacy pip workflows required

***

### 8. Simple Quick Start

```bash
# install
curl -LsSf https://astral.sh/uv/install.sh | sh

# new project
uv init my-app
cd my-app

# add dependency
uv add fastapi

# run project
uv run main.py
```
