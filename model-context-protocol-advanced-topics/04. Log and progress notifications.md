## 2. Key concepts

### Overview

* **Logging and progress notifications**
  * Improve **user experience** when working with MCP servers
  * Provide **real-time feedback** for long-running operations
  * Prevent confusion when no output is shown during execution [\[Log and pr...ifications\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296284)

***

### Problem (Without Notifications)

* Tool execution (e.g., research, data processing):
  * No visible output until completion
  * Users cannot determine:
    * If the tool is running
    * If it has stalled or failed [\[Log and pr...ifications\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296284)

***

### Solution (With Notifications)

* Real-time visibility:
  * Progress updates (e.g., progress bars)
  * Status messages
  * Execution logs [\[Log and pr...ifications\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296284)

***

### How it works (Server side – MCP Tool)

* MCP Python SDK provides **Context object** automatically
* Context enables communication during execution

#### Key methods

* `context.info()`
  * Send **log messages** to client
* `context.report_progress(current, total)`
  * Send **progress updates** [\[Log and pr...ifications\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296284)

#### Example

```python
@mcp.tool(
  name="research",
  description="Research a given topic"
)
async def research(
  topic: str = Field(description="Topic to research"),
  *,
  context: Context
):
  await context.info("About to do research...")
  await context.report_progress(20, 100)

  sources = await do_research(topic)

  await context.info("Writing report...")
  await context.report_progress(70, 100)

  results = await generate_report(sources)
  return results
```

***

### Client-side implementation

* Client must **handle notifications explicitly**

#### Logging callback

* Receives log messages from server

```python
async def logging_callback(params: LoggingMessageNotificationParams):
    print(params.data)
```

#### Progress callback

* Receives progress updates

```python
async def print_progress_callback(progress: float, total: float | None, message: str | None):
    if total is not None:
        percentage = (progress / total) * 100
        print(f"Progress: {progress}/{total} ({percentage:.1f}%)")
    else:
        print(f"Progress: {progress}")
```

#### Usage in client session

```python
async with ClientSession(
    read,
    write,
    logging_callback=logging_callback
) as session:
    await session.initialize()

    await session.call_tool(
        name="add",
        arguments={"a": 1, "b": 3},
        progress_callback=print_progress_callback,
    )
```

* Logging callback → configured at **session level**
* Progress callback → passed per **tool invocation** [\[Log and pr...ifications\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296284)

### Presentation strategies

* CLI applications
  * Print logs + progress to terminal

* Web applications
  * Use:
    * WebSockets
    * Server-Sent Events (SSE)
    * Polling

* Desktop applications
  * Update:
    * Progress bars
    * UI status indicators [\[Log and pr...ifications\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296284)

***

### Design considerations

* Notifications are **optional**
  * Can:
    * Use all
    * Use partial
    * Ignore completely

* Purpose:
  * Enhance **transparency**
  * Improve **user trust**
  * Not required for functional correctness [\[Log and pr...ifications\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296284)

## 3. Q\&A

**Q1. Why are logging and progress notifications important?**

* Provide real-time visibility during long-running operations to avoid user uncertainty. [\[Log and pr...ifications\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296284)

**Q2. What enables notifications in MCP tools?**

* The `Context` object passed to tool functions. [\[Log and pr...ifications\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296284)

**Q3. What are the two main APIs used?**

* `context.info()` → logging
* `context.report_progress()` → progress updates [\[Log and pr...ifications\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296284)

**Q4. Who controls how notifications are displayed?**

* The **client application**, not the MCP server. [\[Log and pr...ifications\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296284)

**Q5. Are notifications mandatory?**

* No, they are optional UX enhancements. [\[Log and pr...ifications\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296284)
