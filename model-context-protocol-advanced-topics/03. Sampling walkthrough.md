## 2. Key Concepts

### Overview

* Demonstrates **Sampling in Model Context Protocol (MCP)**
* Shows how a **server tool invokes an LLM** via a sampling request
* Flow:
  * Initiate sampling → send prompt → model generates → return result → server processes output [\[Sampling walkthrough\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/295172)

***

### Sampling Workflow

* **Initiating sampling**
  * Server creates a request via `ctx.session.create_message(...)`
* **Sampling callbacks**
  * Async execution allows tool to wait for model response
* **Message formats**
  * Uses structured types:
    * `SamplingMessage`
    * `TextContent`
* **Returning generated text**
  * Model response returned in `result.content`
* **Post-processing**
  * Server:
    * Uses generated text in workflow
    * Can trigger another sampling call
    * Or return result directly [\[Sampling walkthrough\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/295172)

***

### Code Structure (server.py)

#### MCP Server Initialization

```python
from mcp.server.fastmcp import FastMCP, Context
from mcp.types import SamplingMessage, TextContent

mcp = FastMCP(name="Demo Server")
```

* `FastMCP`: framework to build MCP server
* `Context`: runtime context for tool execution
* `mcp`: server instance

***

#### Tool Definition

```python
@mcp.tool()
async def summarize(text_to_summarize: str, ctx: Context):
```

* Defines a **tool exposed to MCP**
* Async function → supports long-running LLM calls

***

#### Prompt Construction

```python
prompt = f"""
Please summarize the following text:
{text_to_summarize}
"""
```

* Dynamic prompt created from input

***

#### Sampling Call

```python
result = await ctx.session.create_message(
    messages=[
        SamplingMessage(
            role="user",
            content=TextContent(type="text", text=prompt)
        )
    ],
    max_tokens=4000,
    system_prompt="You are a helpful research assistant.",
)
```

Key elements:

* `messages`: input to model
* `SamplingMessage`: structured message (role + content)
* `TextContent`: content payload
* `system_prompt`: controls model behavior
* `max_tokens`: response limit

***

#### Handling Response

```python
if result.content.type == "text":
    return result.content.text
else:
    raise ValueError("Sampling failed")
```

* Validate response type
* Return output or fail explicitly

***

#### Server Execution

```python
if __name__ == "__main__":
    mcp.run(transport="stdio")
```

* Starts MCP server
* Uses **STDIO transport** for communication

***

### Key Technical Insights

* Sampling is essentially:
  * **Server → LLM call inside tool execution**
* MCP abstracts:
  * Messaging format
  * Session management
* Strong typing:
  * `SamplingMessage`, `TextContent`
* Async pattern:
  * Enables scalable tool execution

## 3. Q\&A

**Q1. What is sampling in MCP?**

* A mechanism for a server tool to request text generation from an LLM using structured messages

**Q2. What API triggers model generation?**

* `ctx.session.create_message(...)`

**Q3. Why use async in tool functions?**

* To handle LLM calls without blocking execution

**Q4. What defines the model’s behavior?**

* `system_prompt`

**Q5. What happens after receiving the result?**

* Server can:
  * Use it in workflows
  * Trigger more sampling
  * Return output directly [\[Sampling walkthrough\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/295172)
