## 2. Key Concepts

### 🔹 Roots (Model Context Protocol – MCP)

* **Definition**
  * Mechanism to grant MCP servers access to **specific local files and folders** [\[Anthropic Courses\]](https://anthropic.skilljar.com/)
  * Acts as a **permission + context system**, not just access control [\[Anthropic Courses\]](https://anthropic.skilljar.com/)

***

### 🔹 Problem Roots Solve

* Claude (or MCP client) **cannot search entire filesystem** by default [\[Anthropic Courses\]](https://anthropic.skilljar.com/)
* Tool calls (e.g., convert video) only receive **filename, not full path** [\[Anthropic Courses\]](https://anthropic.skilljar.com/)
* Challenges:
  * File locations unknown
  * Complex directory structures
  * Requiring full paths = **poor UX** [\[Anthropic Courses\]](https://anthropic.skilljar.com/)

***

### 🔹 How Roots Work (Workflow)

* Standard flow with Roots:
  * Claude calls `list_roots` → discover accessible directories
  * Claude calls `read_dir` → search within allowed paths
  * Locate file → pass **full path to tool**
* Result:
  * Users can still use **simple commands** (e.g., `convert biking.mp4`) [\[Anthropic Courses\]](https://anthropic.skilljar.com/)

***

### 🔹 Security Model

* Roots define **strict boundaries**
  * Only approved directories are accessible
  * Any access outside → **error returned** [\[Anthropic Courses\]](https://anthropic.skilljar.com/)
* Example:
  * Allow: `Desktop/`
  * Deny: `Documents/`, `Downloads/`

***

### 🔹 Implementation Pattern

* MCP SDK **does NOT enforce roots automatically** [\[Anthropic Courses\]](https://anthropic.skilljar.com/)
* Required approach:
  * Implement validation logic manually

```python
def is_path_allowed(path, roots):
    for root in roots:
        if path.startswith(root):
            return True
    return False
```

* Apply:
  * Validate before any file operation
  * Enforce in all tools accessing filesystem [\[Anthropic Courses\]](https://anthropic.skilljar.com/)

***

### 🔹 Key Benefits

* **User Experience**
  * No need for full file paths
* **Performance**
  * Focused search within allowed dirs
* **Security**
  * Prevents access to sensitive areas
* **Flexibility**
  * Roots can be:
    * Provided via tools
    * Injected via prompts [\[Anthropic Courses\]](https://anthropic.skilljar.com/)

## 3. Q\&A

**Q1. What are Roots in MCP?**

* A mechanism to control and provide **contextual file access** for MCP servers.

**Q2. Why are Roots needed?**

* To solve the problem where AI tools cannot locate files without full paths while maintaining security.

**Q3. Does MCP SDK enforce Roots automatically?**

* No, developers must implement validation logic manually.

**Q4. How do Roots improve UX?**

* Users can use **simple file names** instead of full paths.

**Q5. What happens if access is outside roots?**

* The operation fails with an error due to permission restriction.
