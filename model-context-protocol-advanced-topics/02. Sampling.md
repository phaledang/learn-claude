## 2. Key Concepts

### 2.1 Sampling (Core Concept)

* **Definition**
  * Mechanism allowing an MCP server to access a language model (e.g., Claude) **through the client instead of directly** [\[Sampling\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296288)
* **Key idea**
  * Server delegates AI generation to client
  * Client already has credentials + connection

***

### 2.2 Problem Sampling Solves

* Without sampling:
  * Server must:
    * Manage API keys
    * Handle authentication
    * Implement model integration
    * Pay for token usage [\[Sampling\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296288)

* With sampling:
  * Server:
    * Generates prompt only
  * Client:
    * Calls model
    * Handles cost + execution

***

### 2.3 How Sampling Works (Flow)

* Step-by-step execution:
  1. Server performs its task (e.g., fetch data)
  2. Server creates prompt
  3. Server sends **sampling request** to client
  4. Client calls Claude with prompt
  5. Client returns generated result
  6. Server incorporates result into response [\[Sampling\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296288)

***

### 2.4 Benefits of Sampling

* **Reduce server complexity**
  * No AI integration required
* **Shift cost to client**
  * Prevents server-side token cost
* **No API key on server**
  * Avoids credential management
* **Safe for public servers**
  * Prevents abuse / cost explosion [\[Sampling\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296288)

***

### 2.5 Implementation

#### Server Side

* Use `create_message` to request generation:

```python
@mcp.tool()
async def summarize(text_to_summarize: str, ctx: Context):
    prompt = f"""
    Please summarize the following text:
    {text_to_summarize}
    """

    result = await ctx.session.create_message(
        messages=[
            SamplingMessage(
                role="user",
                content=TextContent(type="text", text=prompt)
            )
        ],
        max_tokens=4000,
        system_prompt="You are a helpful research assistant",
    )

    if result.content.type == "text":
        return result.content.text
    else:
        raise ValueError("Sampling failed")
```

#### Client Side

* Define callback to execute request:

```python
async def sampling_callback(context, params):
    text = await chat(params.messages)
    return CreateMessageResult(
        role="assistant",
        model=model,
        content=TextContent(type="text", text=text),
    )
```

* Register callback:

```python
async with ClientSession(
    read, write,
    sampling_callback=sampling_callback
) as session:
    await session.initialize()
```

***

### 2.6 When to Use Sampling

* Recommended for:
  * Public MCP servers
  * Multi-user environments
  * Cost-sensitive systems

* Main purpose:
  * **Move AI responsibility from server → client** [\[Sampling\]](https://anthropic.skilljar.com/model-context-protocol-advanced-topics/296288)

## 3. Q\&A

**Q1: What is sampling in MCP?**

* Delegation pattern where server requests client to call LLM

**Q2: Why not call the model directly from server?**

* Adds complexity, cost, and credential management

**Q3: Who pays for AI usage in sampling?**

* Client (not server)

**Q4: What is the key architectural shift?**

* AI execution moves from server → client

**Q5: What enables sampling technically?**

* `create_message` (server) + callback handler (client)
