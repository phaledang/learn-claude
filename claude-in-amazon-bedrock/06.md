# Module 1: Accessing Claude through AWS Bedrock

## Architecture Flow

```text
User
  ↓
Web Server
  ↓
Bedrock Client (Boto3)
  ↓
AWS Bedrock Service
  ↓
Claude Model
  ↓
Response
```

### Required Components

1. **Boto3 Client**
   * Connects to Bedrock Runtime
   * Configured with AWS Region

2. **Model ID / Inference Profile**
   * Identifies Claude model
   * Inference Profiles handle cross-region routing

3. **User Message**
   * Structured message object sent to Claude

### Important Distinction

| AWS Bedrock API   | Anthropic API           |
| ----------------- | ----------------------- |
| AWS Service       | Anthropic Service       |
| Uses Boto3 SDK    | Uses Anthropic SDK      |
| AWS Documentation | Anthropic Documentation |
| Hosted by AWS     | Hosted by Anthropic     |

Although both may run Claude models, the APIs and SDKs are different.

# Module 3: Multi-Turn Conversations

## Key Fact

Claude APIs are **stateless**.

They do not remember previous messages.

You must send:

```python
messages = [
    user,
    assistant,
    user,
    assistant
]
```

on every API request.

### Rules

✅ Alternate roles

```text
user → assistant → user → assistant
```

❌ Never:

```text
user → user
assistant → assistant
```

### Best Practice

Maintain conversation history in application code.

# Module 5: Temperature

## Definition

Temperature controls randomness.

Range:

```text
0.0 → deterministic
1.0 → creative
```

### Usage Guide

| Task             | Temperature |
| ---------------- | ----------- |
| Data Extraction  | 0.0         |
| Classification   | 0.0         |
| Coding           | 0.1-0.3     |
| Summaries        | 0.3         |
| Brainstorming    | 0.8         |
| Creative Writing | 1.0         |

### Principle

Lower temperature = consistency

Higher temperature = creativity

# Module 6: Streaming Responses

Without streaming:

```text
Request
Wait 10 seconds
Response
```

With streaming:

```text
Request
Chunk...
Chunk...
Chunk...
Complete
```

### Event Types

* Message Start
* Content Block Delta
* Content Block Stop
* Message Stop
* Metadata

### Event Processing

```python
for event in response["stream"]:
    if "content_block_delta" in event:
        chunk = event["content_block_delta"]["delta"]["text"]
```

Benefits:

* Better UX
* Faster perceived response times


* Cheaper
* Scalable

# Module 31: Agents

## Definition

An Agent is:

> A language model with tools operating in a loop until a goal is achieved.

Agent Loop:

```text
Inspect Environment
 ↓
Use Tool
 ↓
Get Feedback
 ↓
Repeat
```

Principles:

* Context is king
* Evals are essential
* Use focused tools
* Favor high-value, low-risk tasks

# Q\&A

### Q1. Why do Bedrock conversations require the full message history?

Because Claude APIs are stateless and do not remember previous interactions.

***

### Q2. What does temperature control?

The randomness of token selection.

***

### Q3. When should temperature be set near 0?

Data extraction, classification, and factual tasks.

***

### Q4. What is the purpose of a system prompt?

To assign Claude a role and influence behavior.

***

### Q5. What is RAG?

Retrieval Augmented Generation, a technique that retrieves relevant information before generation.

***

### Q6. Why are embeddings important?

They enable semantic search through vector similarity.

***

### Q7. What problem does MCP solve?

It reduces integration complexity by providing pre-built tools and schemas.

***

### Q8. What is Prompt Caching?

A feature that stores prompt processing work and reuses it to reduce cost and latency.

***

### Q9. What are the three MCP primitives?

Tools, Resources, and Prompts.

***

### Q10. What defines an AI Agent?

A language model repeatedly using tools until a goal is achieved.
