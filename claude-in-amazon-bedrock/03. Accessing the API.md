# Key Concepts

## 1. How AI Chat Applications Work

When a user interacts with an AI-powered application, there are multiple components involved behind the scenes. The user only sees a chat interface, but the request travels through several systems before a response is generated. [\[Accessing the API\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276716)

Example:

```text
User: Define quantum computing
```

The AI-generated answer is not produced directly in the browser. Instead, the message is sent through a backend service and AWS Bedrock before returning a response. [\[Accessing the API\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276716)

***

## 2. Request Flow with Amazon Bedrock

The page describes the typical request lifecycle when using Claude models through AWS Bedrock. [\[Accessing the API\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276716)

### Step 1: User Sends a Request

A user enters text into a chat interface and clicks **Send**. [\[Accessing the API\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276716)

```text
"Define quantum computing"
```

***

### Step 2: Web Application Receives the Request

The front-end sends the message to your backend server. [\[Accessing the API\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276716)

```text
Browser
   ↓
Application Server
```

The server is responsible for authentication, business logic, and communication with Bedrock.

***

### Step 3: Backend Calls AWS Bedrock

The backend uses a **Bedrock client** to send the request to AWS Bedrock. [\[Accessing the API\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276716)

Request typically contains:

* User message
* Claude model identifier
* Additional parameters (temperature, system prompts, etc.)

Example:

```json
{
  "modelId": "Claude Sonnet",
  "message": "Define quantum computing"
}
```

***

### Step 4: Claude Model Processes the Request

The selected model (for example Claude Haiku or Claude Sonnet) analyzes the prompt and generates a response. [\[Accessing the API\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276716)

```text
Input:
Define quantum computing

Output:
Quantum computing is a type of computing...
```

***

### Step 5: AWS Bedrock Returns the Response

AWS Bedrock sends the generated assistant message back to your application server. [\[Accessing the API\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276716)

```text
Claude Model
      ↓
AWS Bedrock
      ↓
Application Server
```

***

### Step 6: Response Delivered to User

The application server forwards the generated response to the user's browser, which displays it in the chat interface. [\[Accessing the API\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276716)

```text
Server
   ↓
Browser
   ↓
User sees answer
```

# Why This Architecture Matters

### Scalability

Your application can support many users while Bedrock manages model infrastructure.*(General architectural benefit; not explicitly stated in the lesson.)*

### Security

The backend acts as a controlled gateway between users and the model API.*(General architectural benefit.)*

### Flexibility

You can switch model versions (Haiku, Sonnet, etc.) simply by changing the model ID sent to Bedrock. [\[Accessing the API\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276716)

### Cost Control

Different models can balance speed, capability, and cost depending on your application's needs.*(General AWS design consideration.)*

# Q\&A

## Q1. What happens when a user submits a prompt?

**Answer:**  
The prompt is sent from the user interface to the application server, which then sends it to AWS Bedrock for processing. AWS Bedrock returns the generated response back through the server to the browser. [\[Accessing the API\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276716)

***

## Q2. What is AWS Bedrock's role?

**Answer:**  
AWS Bedrock provides managed access to foundation models such as Claude and returns generated responses to applications. [\[Accessing the API\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276716)

***

## Q3. What information is sent in the API request?

**Answer:**  
The request includes the user message and the selected model ID. [\[Accessing the API\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276716)

***

## Q4. Which Claude models are mentioned?

**Answer:**

* Claude Haiku
* Claude Sonnet [\[Accessing the API\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276716)

***

## Q5. Why does an application server sit between the user and Bedrock?

**Answer:**  
The server receives requests from users and makes API calls to AWS Bedrock on their behalf. [\[Accessing the API\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276716)

## Key Takeaway

The fundamental API workflow is:

```text
User
  → Web App
  → Application Server
  → AWS Bedrock
  → Claude Model
  → AWS Bedrock
  → Application Server
  → User
```
