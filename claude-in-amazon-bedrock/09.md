# Key Concepts

## What is Temperature?

* **Temperature** is a parameter that controls how **creative** or **deterministic** Claude's responses are. [\[Temperature\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276720)
* It accepts a value between **0.0 and 1.0**. [\[Temperature\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276720)
* Think of temperature as a **creativity dial**:
  * Lower temperature = predictable responses
  * Higher temperature = creative and varied responses [\[Temperature\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276720)

***

## How Claude Generates Text

Claude generates text through three main phases:

### 1. Tokenization

* Breaks input text into smaller pieces called **tokens**. [\[Temperature\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276720)

### 2. Prediction

* Calculates probabilities for possible next tokens. [\[Temperature\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276720)

### 3. Sampling

* Selects the next token based on the calculated probabilities. [\[Temperature\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276720)

Example:

| Possible Token | Probability |
| -------------- | ----------- |
| about          | 30%         |
| would          | 20%         |
| others         | remaining % |

Claude repeats this process until the response is complete. [\[Temperature\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276720)

***

## What Temperature Actually Changes

### Low Temperature (Near 0)

* Favors the highest-probability token.
* Produces stable and repeatable outputs.
* More deterministic behavior. [\[Temperature\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276720)

Example:

```text
temperature = 0.0
```

Claude will typically return nearly identical responses for the same prompt. [\[Temperature\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276720)

### High Temperature (Near 1)

* Gives lower-probability tokens a better chance.
* Produces diverse and creative outputs.
* Less predictable. [\[Temperature\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276720)

Example:

```text
temperature = 1.0
```

Claude may generate significantly different responses each time. [\[Temperature\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276720)

# Temperature Ranges and Recommended Usage

## Low Temperature (0.0–0.3)

Best for:

* Factual responses
* Coding assistance
* Data extraction
* Content moderation

Why?

* Accuracy and consistency are more important than creativity. [\[Temperature\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276720)

***

## Medium Temperature (0.4–0.7)

Best for:

* Summarization
* Educational content
* Problem solving
* Creative writing with constraints

Why?

* Balances accuracy and creativity. [\[Temperature\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276720)

***

## High Temperature (0.8–1.0)

Best for:

* Brainstorming
* Creative writing
* Marketing content
* Joke generation

Why?

* Encourages novel and diverse ideas. [\[Temperature\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276720)

# Using Temperature in Code

Example Python function:

```python
def chat(messages, system=None, temperature=1.0):
    params = {
        "modelId": model_id,
        "messages": messages,
        "inferenceConfig": {
            "temperature": temperature
        }
    }

    if system:
        params["system"] = [{"text": system}]

    response = client.converse(**params)

    return response["output"]["message"]["content"][0]["text"]
```

The temperature value is passed into:

```python
"inferenceConfig": {
    "temperature": temperature
}
```

This changes how Claude samples the next tokens. [\[Temperature\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276720)
