# Key Concepts

## 1. What Is Tool Use?

Tool use allows Claude to access information from external systems and services instead of relying only on its training data. This overcomes a major limitation of LLMs: lack of access to real-time information. [\[anthropic-...ock/276756\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276756)

Without tools, Claude cannot reliably provide:

* Current weather
* Recent news
* Live database information
* Enterprise system data
* Information generated after training completed [\[anthropic-...ock/276756\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276756)

### Example

User asks:

> "What's the weather in San Francisco?"

Without tools:

> "I don't have access to up-to-date weather information."

With tools:

> Claude can request weather data from an external API and provide a current answer. [\[anthropic-...ock/276756\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276756)

***

## 2. Why Tool Use Is Important

Tool use creates a bridge between Claude and external data sources. [\[anthropic-...ock/276756\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276756)

Benefits:

* Access real-time information
* Query databases
* Retrieve documents
* Call APIs
* Perform actions through external systems
* Improve answer accuracy and freshness [\[anthropic-...ock/276756\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276756)

***

## 3. How Tool Use Works

Tool use follows a multi-step interaction between:

* User
* Claude
* Your application/server
* External services/APIs [\[anthropic-...ock/276756\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276756)

### Tool Use Flow

```text
User Question
      |
      v
Claude receives question + tool definitions
      |
      v
Claude decides a tool is needed
      |
      v
Tool Request generated
      |
      v
Server executes tool
      |
      v
Tool result returned to Claude
      |
      v
Claude generates final answer
```

 [\[anthropic-...ock/276756\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276756)

## 4. Step-by-Step Workflow

### Step 1: Initial Request

You send:

* User question
* Tool definitions
* Instructions on how tools work

to Claude. [\[anthropic-...ock/276756\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276756)

***

### Step 2: Tool Request

Claude analyzes the request.

If additional information is needed, Claude generates a tool use request specifying:

* Which tool to call
* What parameters to provide [\[anthropic-...ock/276756\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276756)

***

### Step 3: Data Retrieval

Your server:

* Receives the tool request
* Executes code
* Calls APIs or data sources
* Gets the required information [\[anthropic-...ock/276756\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276756)

***

### Step 4: Tool Result

The retrieved data is sent back to Claude as a tool result. [\[anthropic-...ock/276756\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276756)

***

### Step 5: Final Response

Claude combines:

* User question
* Tool output

and produces the final answer. [\[anthropic-...ock/276756\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276756)

# Implementation Challenges

The course highlights that tool use often feels confusing because:

* Logical execution order differs from coding order.
* Developers implement pieces before they are actually executed. [\[anthropic-...ock/276756\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276756)

### Typical Development Tasks

You often need to:

1. Write the tool function
2. Create a JSON Schema
3. Handle ToolUse messages
4. Handle ToolResult messages
5. Include tool schemas in requests [\[anthropic-...ock/276756\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276756)

***

## Why It Feels Complex

Conceptually:

```text
Question
→ Tool Request
→ Tool Call
→ Tool Result
→ Final Answer
```

But implementation may require coding in this order:

```text
Tool Function
→ JSON Schema
→ Tool Result Handling
→ Tool Use Handling
→ Request Logic
```

The mismatch between logical flow and implementation flow often causes confusion for developers. [\[anthropic-...ock/276756\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276756)

# Q\&A

## Q1. What problem does tool use solve?

Tool use enables Claude to access external, current information rather than relying only on training data. [\[anthropic-...ock/276756\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276756)

***

## Q2. Why can't Claude answer today's weather without tools?

Because weather information changes continuously and is not contained in Claude's pre-trained knowledge. [\[anthropic-...ock/276756\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276756)

***

## Q3. Who executes the actual tool?

Your application/server executes the tool and retrieves the requested data. [\[anthropic-...ock/276756\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276756)

***

## Q4. What happens after Claude requests a tool?

The server runs the tool, retrieves information, and sends the result back to Claude. [\[anthropic-...ock/276756\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276756)

***

## Q5. Why is JSON Schema important?

It defines the structure and expected parameters of a tool so Claude knows how to call it correctly. [\[anthropic-...ock/276756\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276756)

***

## Q6. What is the final step in tool use?

Claude combines the original request and tool output to generate a final response. [\[anthropic-...ock/276756\]](https://anthropic-partners.skilljar.com/claude-in-amazon-bedrock/276756)
