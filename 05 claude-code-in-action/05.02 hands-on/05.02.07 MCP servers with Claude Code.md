## 1. Summary (with URL)

**Page:** MCP servers with Claude Code  
**URL:** <https://anthropic.skilljar.com/claude-code-in-action/303239>

### Core concepts

* **MCP (Model Context Protocol) servers**
  * Extend Claude Code with **external capabilities/tools**
  * Can run **locally or remotely**
  * Provide abilities Claude does not natively have [\[MCP server...laude Code\]](https://anthropic.skilljar.com/claude-code-in-action/303239)

* **Example: Playwright MCP server**
  * Enables **browser automation**
  * Allows Claude to:
    * Open web pages
    * Interact with UI
    * Analyze visual output [\[MCP server...laude Code\]](https://anthropic.skilljar.com/claude-code-in-action/303239)

* **Installation**
  ```bash
  claude mcp add playwright npx @playwright/mcp@latest
  ```
  * Registers server name = `playwright`
  * Defines how to start it locally [\[MCP server...laude Code\]](https://anthropic.skilljar.com/claude-code-in-action/303239)

* **Permission model**
  * By default: user approval required for each tool call
  * Can pre-approve via config:
  ```json
  {
    "permissions": {
      "allow": ["mcp__playwright"],
      "deny": []
    }
  }
  ```
  * Enables automatic execution without prompts [\[MCP server...laude Code\]](https://anthropic.skilljar.com/claude-code-in-action/303239)

* **Workflow example (UI improvement loop)**
  Claude can:
  * Open app (localhost:3000)
  * Generate component
  * Inspect UI result
  * Update prompt file
  * Re-test improved output [\[MCP server...laude Code\]](https://anthropic.skilljar.com/claude-code-in-action/303239)

* **Key advantage**
  * Claude evaluates **real output (visual + behavior)** instead of just code → higher-quality improvements [\[MCP server...laude Code\]](https://anthropic.skilljar.com/claude-code-in-action/303239)

* **Ecosystem**
  * MCP servers exist for:
    * Databases
    * APIs
    * File systems
    * Cloud services
    * Dev tools automation [\[MCP server...laude Code\]](https://anthropic.skilljar.com/claude-code-in-action/303239)

## 2. Q\&A

### Basic understanding

* **Q: What is an MCP server?**  
  A server that provides tools and context to extend Claude’s capabilities. [\[MCP server...laude Code\]](https://anthropic.skilljar.com/claude-code-in-action/303239)

* **Q: Where can MCP servers run?**  
  Locally or remotely. [\[MCP server...laude Code\]](https://anthropic.skilljar.com/claude-code-in-action/303239)

* **Q: Why use MCP?**  
  To enable Claude to interact with external systems (browser, DB, APIs).

***

### Playwright usage

* **Q: What does Playwright MCP enable?**  
  Browser control and UI interaction. [\[MCP server...laude Code\]](https://anthropic.skilljar.com/claude-code-in-action/303239)

* **Q: Why is browser access valuable?**  
  Allows Claude to analyze **actual UI output**, not just code. [\[MCP server...laude Code\]](https://anthropic.skilljar.com/claude-code-in-action/303239)

***

### Permissions

* **Q: Why configure permissions?**  
  To avoid repeated approval prompts. [\[MCP server...laude Code\]](https://anthropic.skilljar.com/claude-code-in-action/303239)

* **Q: What is `mcp__playwright`?**  
  The identifier used to allow Playwright tools. [\[MCP server...laude Code\]](https://anthropic.skilljar.com/claude-code-in-action/303239)

***

### Workflow usage

* **Q: What type of workflow is demonstrated?**  
  Iterative improvement loop:
  * Generate → Evaluate → Improve → Re-test

* **Q: What file can Claude update?**  
  Prompt file (e.g., `generation.tsx`). [\[MCP server...laude Code\]](https://anthropic.skilljar.com/claude-code-in-action/303239)

## 4. Key Takeaways

* MCP = **tool extension layer for Claude**
* Enables **real-world interaction (browser, APIs, systems)**
* Playwright MCP = **critical for UI-driven workflows**
* Configurable permissions improve developer experience
* Powerful pattern:
  ```
  Generate → Observe → Improve (closed-loop optimization)

* Transforms Claude from:
  ```
  Code generator → Autonomous development assistant
