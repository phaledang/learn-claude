## 1. Summary of page

**Source:** <https://anthropic.skilljar.com/claude-code-in-action/303236>

### Core concepts

* **Goal:** Efficiently implement changes in an existing codebase using Claude Code
* Focus areas:
  * Visual communication (screenshots)
  * Structured planning (Planning Mode)
  * Reasoning control (Effort level)

### Key techniques

* **Screenshots**
  * Paste via `Ctrl + V`
  * Provide precise UI/UX context for modifications
  * Helps Claude identify exact target area

* **Planning Mode**
  * Activate: `/plan` or `Shift + Tab`
  * Behavior:
    * Reads more files across project
    * Generates implementation plan
    * Requires approval before execution
  * Workflow:
    ```
    Explore → Plan → Review → Approve → Execute
    ```

* **Effort control**
  * Command: `/effort`
  * Levels:
    * `low` → faster, cheaper
    * `max` → deeper reasoning
  * Per-prompt boost:
    * keyword: `ultrathink`

### When to use what

| Scenario                         | Feature       |
| -------------------------------- | ------------- |
| Multi-file / architecture change | Planning Mode |
| Debugging / complex logic        | Effort level  |
| Both complexity types            | Combine both  |

## 2. Q\&A

### Q1. Why use screenshots with Claude Code?

* Provide exact visual context → reduces ambiguity in UI-related changes

### Q2. How to enable Planning Mode?

* `/plan` command
* or `Shift + Tab`

### Q3. What happens in Planning Mode?

* Claude:
  * scans codebase
  * builds detailed plan
  * waits for approval

### Q4. How to adjust reasoning depth?

* Use `/effort`
* Increase level for harder problems

### Q5. What is `ultrathink`?

* Temporary signal to increase reasoning for a single prompt

### Q6. Difference between Planning vs Effort?

| Planning                           | Effort                       |
| ---------------------------------- | ---------------------------- |
| Breadth (multi-file understanding) | Depth (reasoning complexity) |

## 4. Key takeaways

* **Precision**
  * Use screenshots to remove ambiguity

* **Control**
  * Planning Mode → structured execution
  * Effort → reasoning depth

* **Pattern mapping (important for architecture mindset)**
  ```
  UI change → screenshot
  System change → planning
  Logic complexity → effort tuning
  ```

* **Advanced usage**
  * Combine:
    ```
    /plan + high effort → architecture refactor
    ```

* **Cost awareness**
  * Planning + high effort = higher token usage
