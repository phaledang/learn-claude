## 1. Summary

**Source:** *Controlling context* ([Anthropic Academy page](https://anthropic.skilljar.com/claude-code-in-action/303237))

**Core topic:** Techniques to control and manage conversation context when working with Claude Code to ensure accuracy, focus, and productivity.

### Key concepts

* **Problem:**
  * Long conversations accumulate irrelevant or noisy context
  * Claude may:
    * Go off-track
    * Try to solve too many tasks at once
    * Repeat mistakes across iterations

* **Solution:** Use **context control techniques**
  * Interrupt execution
  * Rewind conversation state
  * Manage memory and history

### Techniques overview

| Technique                  | Purpose                                                  |
| -------------------------- | -------------------------------------------------------- |
| `Escape`                   | Stop current response and redirect focus                 |
| `Escape + /memory`         | Fix recurring mistakes by storing corrections            |
| `/rewind` or double Escape | Go back to earlier conversation state                    |
| `/compact`                 | Summarize conversation while keeping important knowledge |
| `/clear`                   | Start fresh with no context                              |

## 2. Q\&A (from page)

### Q1. Why is controlling context important?

* Prevents irrelevant or noisy history from affecting current tasks
* Keeps Claude focused on the intended objective
* Improves accuracy and efficiency

***

### Q2. What happens if Claude goes in the wrong direction?

* Use **Escape** to interrupt the response
* Redirect with more specific instruction

***

### Q3. How to fix repeated mistakes?

* Workflow:
  1. Press **Escape**
  2. Use `/memory` (or edit CLAUDE.md)
  3. Add correction
  4. Continue task

***

### Q4. When should `/rewind` be used?

* When conversation contains:
  * Debugging noise
  * Irrelevant exploration
* Allows returning to a clean earlier state

***

### Q5. Difference between `/compact` and `/clear`?

| Command    | Behavior                            |
| ---------- | ----------------------------------- |
| `/compact` | Keeps knowledge, compresses history |
| `/clear`   | Removes all context (fresh start)   |

***

### Q6. Can previous conversations be restored after `/clear`?

* Yes, using `/resume`

## 4. Key Takeaways

### Core principles

* Context = critical factor in AI output quality
* Control context actively → better results

### Practical usage patterns

* **Interrupt early** → avoid wasted tokens
* **Iterate in small tasks** → avoid over-complex responses
* **Store corrections** → improve long-term reliability
* **Clean context frequently** → avoid noise accumulation

### Decision guideline

| Scenario                | Best action        |
| ----------------------- | ------------------ |
| Claude going wrong      | `Escape`           |
| Repeated mistake        | `Escape + /memory` |
| Too much noise          | `/rewind`          |
| Long but useful context | `/compact`         |
| New unrelated task      | `/clear`           |
