## 1. Summary (Page: <https://anthropic.skilljar.com/claude-code-in-action/303240>)

**Topic: GitHub Integration in Claude Code**

* Claude Code provides **official GitHub integration via GitHub Actions**
* Enables Claude to act as an **automated collaborator inside repositories**

### Core capabilities

* Runs inside **GitHub Actions workflows**
* Supports:
  * **Issue / PR interaction** via `@claude`
  * **Automatic pull request review**

### Setup flow

* Run command:
  ```
  /install-github-app
  ```
* This process:
  * Installs Claude GitHub App
  * Adds API key
  * Generates a **PR with workflow files**
* After merging:
  * Workflows exist under `.github/workflows`

### Default workflows

| Workflow         | Behavior                            |
| ---------------- | ----------------------------------- |
| Mention Action   | Triggered by `@claude` in issues/PR |
| PR Review Action | Runs automatically on PR creation   |

**Mention Action flow**

* Analyze request
* Create task plan
* Execute using full repo access
* Reply in issue/PR

**PR Action flow**

* Review changes
* Analyze impact
* Post report

### Customization

#### 1. Environment setup

```yaml
- name: Project Setup
  run: |
    npm run setup
    npm run dev:daemon
```

#### 2. Custom instructions

* Provide runtime context
* Example:
  * App already running
  * Logs location
  * DB access
  * MCP tools usage

#### 3. MCP server integration

```json
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest", "--allowed-origins", "..."]
    }
  }
}
```

#### 4. Tool permissions (critical in CI)

```txt
allowed_tools: "Bash(npm:*), mcp__playwright__browser_click, ..."
```

* Must explicitly define **every tool**
* No shortcut like local environment

### Best practices

* Start with default workflows
* Gradually customize
* Add **custom instructions for context**
* Explicitly define tool permissions
* Test simple → complex workflows

## 2. Q\&A

**Q1. What is the main purpose of Claude GitHub integration?**

* Enable Claude to act inside GitHub workflows (issues, PRs, automation)

**Q2. How is the integration installed?**

* Run `/install-github-app` → auto setup + PR

**Q3. What are the two default workflows?**

* Mention Action
* Pull Request Review Action

**Q4. What happens when using `@claude`?**

* Claude:
  * Understands request
  * Builds plan
  * Executes tasks
  * Replies in thread

**Q5. What happens on PR creation?**

* Claude:
  * Reviews code
  * Analyzes impact
  * Posts report

**Q6. Why are tool permissions important?**

* In GitHub Actions, all tools must be explicitly allowed
* Especially critical when using MCP servers

**Q7. How do MCP servers enhance this setup?**

* Provide external capabilities (e.g., browser via Playwright)

**Q8. What is the role of custom instructions?**

* Provide runtime/project context to Claude

## 4. Key Takeaways

* Claude can operate **directly inside GitHub workflows**
* Two main automation patterns:
  * **On-demand interaction (`@claude`)**
  * **Automated PR review**
* Integration = **GitHub Actions + Claude runtime**
* MCP enables **tool extension (e.g., Playwright automation)**
* CI/CD requires:
  * Explicit **tool permissions**
  * Defined **environment setup**
* Best usage pattern:
  * Start simple → iterate → add MCP + instructions
