Source: Anthropic Skilljar – Claude Code in Action (Useful hooks)

  * Automatic triggers executed when Claude modifies code.
  * Purpose: **catch errors early + enforce consistency in large projects**.

* **Key Problem Addressed**
  * AI modifies part of code but misses dependent updates (e.g., function signature changes → call sites not updated).
  * Leads to:
    * Type errors
    * Duplicate logic
    * Inconsistent codebase

***

### Hook #1: TypeScript Type Checking Hook

* **Trigger**: Post-tool-use (after file modification)

* **Behavior**:
  * Runs `tsc --noEmit`
  * Captures type errors
  * Feeds errors back to Claude
  * Claude fixes impacted files

* **Generalization**:
  * Typed languages → compiler
  * Untyped languages → automated tests

***

### Hook #2: Query Duplication Prevention Hook

* **Problem**:
  * Claude creates **new DB queries instead of reusing existing ones**

* **Trigger**:
  * When files in `./queries` directory are modified

* **Workflow**:
  * Launch a **second Claude instance**
  * Review code changes
  * Detect duplicate queries
  * Send feedback to original Claude
  * Claude refactors to reuse existing query

***

### Implementation Notes

* Hook types:
  * `pre-tool-use`
  * `post-tool-use`

* Trade-offs:
  | Hook           | Cost | Benefit                    |
  | -------------- | ---- | -------------------------- |
  | Type checking  | Low  | Immediate error correction |
  | Query reviewer | High | Cleaner architecture       |

* Recommendation:
  * Apply heavy hooks **only on critical directories**

***

### Core Design Principles

* Integrate **compiler/linter feedback loop**
* Use **AI-to-AI review pattern**
* Enforce **code reuse + consistency**
* Balance:
  * Automation benefit
  * Performance/API cost [\[Useful hooks!\]](https://anthropic.skilljar.com/claude-code-in-action/312004)

## 2. Q\&A

**Q2. What issue does the TypeScript hook solve?**  
→ Missing updates across call sites when function signatures change.

**Q3. How does the TypeScript hook work?**  
→ Runs compiler → captures errors → sends feedback → triggers fix.

**Q4. What problem does the query duplication hook solve?**  
→ Prevents duplication of database query logic.

**Q5. Why use a second Claude instance?**  
→ To act as an independent reviewer for validation.

**Q6. When should heavy hooks be applied?**  
→ Only on high-value directories to control cost.
