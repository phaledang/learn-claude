## 1. Summary

**Page:** *Introducing hooks*  
**URL:** <https://anthropic.skilljar.com/claude-code-in-action/312000>

* Hooks = mechanism to **inject custom logic before/after tool execution in Claude Code**
* Enable **automation + control layer** around Claude tool usage
* Execution flow:
  1. User prompt → Claude
  2. Claude decides to use a tool
  3. Tool executes
  4. Result returned
* **Hooks intercept step 2–3** to run custom commands
* Two core types:
  * **PreToolUse** → executed *before* tool call
  * **PostToolUse** → executed *after* tool call [\[Introducing hooks\]](https://anthropic.skilljar.com/claude-code-in-action/312000)

**Value:**

* Extend Claude without modifying core logic
* Integrate with CI/CD, validation, or internal tooling

***

## 2. Key Concepts

### Hook Types

| Hook Type   | Timing           | Capabilities                    |
| ----------- | ---------------- | ------------------------------- |
| PreToolUse  | Before tool runs | Allow / Block / Modify behavior |
| PostToolUse | After tool runs  | Trigger follow-up actions       |

***

### Hook Capabilities

**PreToolUse**

* Inspect tool request
* Allow execution
* Block execution with error
* Enforce access control

**PostToolUse**

* Cannot block (already executed)
* Trigger automation
* Provide feedback to Claude

***

## 3. Configuration

### Supported Locations

| Scope                 | Location                      |
| --------------------- | ----------------------------- |
| Global                | `~/.claude/settings.json`     |
| Project (shared)      | `.claude/settings.json`       |
| Personal (local only) | `.claude/settings.local.json` |

***

### Example: PreToolUse

```json
{
  "PreToolUse": [
    {
      "matcher": "Read",
      "hooks": [
        {
          "type": "command",
          "command": "node /home/hooks/read_hook.js"
        }
      ]
    }
  ]
}
```

**Behavior:**

* Triggered before `Read` tool
* Can block or allow execution

***

### Example: PostToolUse

```json
{
  "PostToolUse": [
    {
      "matcher": "Write|Edit",
      "hooks": [
        {
          "type": "command",
          "command": "node /home/hooks/edit_hook.js"
        }
      ]
    }
  ]
}
```

**Behavior:**

* Runs after file changes
* Cannot cancel the operation

## 5. Q\&A

**Q1: What problem do hooks solve?**

* Add automation + control around Claude tool execution

**Q2: What is the difference between PreToolUse and PostToolUse?**

* Pre → control (can block)
* Post → enhancement (cannot block)

**Q3: Where are hooks defined?**

* Claude settings files (global/project/local)

**Q4: Can PostToolUse prevent a tool execution?**

* No, execution already completed

**Q5: What does "matcher" do?**

* Filters which tools the hook applies to
