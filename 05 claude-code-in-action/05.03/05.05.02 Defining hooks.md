## 1. Summary

**Page:** Defining hooks – Claude Code in Action  
URL: <https://anthropic.skilljar.com/claude-code-in-action/312002>

* Hooks = mechanism to **intercept and control tool calls** in Claude Code
* Enable **fine-grained governance** over execution (before / after tool usage)
* Two types:
  * **PreToolUse** → executed *before* tool call → can **block execution**
  * **PostToolUse** → executed *after* tool call → used for **post-processing / logging**

### Hook lifecycle (core flow)

1. Select hook type (Pre or Post)
2. Define **target tools** to monitor
3. Implement **command handler**
   * Receives tool call data via **stdin (JSON)**
4. Return decision via **exit code**

### Tool-call payload (input to hook)

```json
{
  "session_id": "uuid",
  "transcript_path": "/path",
  "hook_event_name": "PreToolUse",
  "tool_name": "Read",
  "tool_input": {
    "file_path": "/code/queries/.env"
  }
}
```

### Control mechanism (critical design)

| Exit Code | Behavior                          |
| --------- | --------------------------------- |
| 0         | Allow execution                   |
| 2         | Block execution (PreToolUse only) |

* Error output (stderr) → passed back to Claude as explanation

### Key capability

* Central control over:
  * Allowed tools
  * Input parameters (e.g., file paths)
  * Security policies
* Example:
  * Block access to sensitive files (`.env`)
  * Monitor multiple tools (e.g., `Read`, `Grep`)

## 2. Q\&A

**Q1: What is the main purpose of hooks?**  
A: Intercept and control tool calls before or after execution.

**Q2: Difference between PreToolUse and PostToolUse?**

* PreToolUse → prevent execution
* PostToolUse → act after execution

**Q3: How does a hook receive data?**  
A: JSON via standard input (stdin).

**Q4: How does a hook communicate decision back?**  
A: Using exit codes (0 = allow, 2 = block).

**Q5: Can PostToolUse block execution?**  
A: No (runs after execution).

**Q6: What triggers a hook?**  
A: Specific tool calls defined in configuration.

**Q7: Why monitor multiple tools (Read + Grep)?**  
A: Different tools can access same sensitive data paths.

## 4. Key Takeaways

* Hooks = **policy enforcement layer** for Claude Code
* **PreToolUse = control gate (allow/block)**
* **PostToolUse = observation & processing layer**
* Communication pattern:
  ```
  Claude → Hook (JSON stdin) → Exit code → Claude
  ```
* Enables:
  * Security (block secrets access)
  * Governance (restrict tools)
  * Observability (logging / auditing)
* Must explicitly define:
  * Target tools
  * Evaluation logic
* Works seamlessly with MCP ecosystem (tools can expand dynamically)
