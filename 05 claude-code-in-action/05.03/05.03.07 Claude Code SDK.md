### The Claude Code SDK

* The **Agent SDK** allows running Claude Code **programmatically** inside applications and scripts. [\[The Claude Code SDK\]](https://anthropic.skilljar.com/claude-code-in-action/312001)
* Supported languages:
  * **TypeScript (Node.js)**
  * **Python** [\[The Claude Code SDK\]](https://anthropic.skilljar.com/claude-code-in-action/312001)
* Provides the same **agent loop as CLI**, including:
  * File reading
  * File editing
  * Tool execution
  * Message streaming (events) [\[The Claude Code SDK\]](https://anthropic.skilljar.com/claude-code-in-action/312001)

### Key Setup Flow

* Install SDK package:
  ```bash
  mkdir sdk-demo
  cd sdk-demo
  npm init -y
  npm install @anthropic-ai/claude-agent-sdk
  ```
* Important distinction:
  * `@anthropic-ai/claude-agent-sdk` → SDK (importable)
  * `@anthropic-ai/claude-code` → CLI (not importable) [\[The Claude Code SDK\]](https://anthropic.skilljar.com/claude-code-in-action/312001)

### Minimal Example

```javascript
import { query } from "@anthropic-ai/claude-agent-sdk";

const prompt = "List the files in the current directory";

for await (const message of query({ prompt })) {
  console.log(JSON.stringify(message, null, 2));
}
```

* Executes an agent loop and streams:
  * Tool calls
  * Tool results
  * Claude responses [\[The Claude Code SDK\]](https://anthropic.skilljar.com/claude-code-in-action/312001)

### Tool Control

* Default: full tool access
* Restrict tools via:

```javascript
options: {
  allowedTools: ["Read", "Glob"]
}
```

* Equivalent to CLI flag `--allowedTools` [\[The Claude Code SDK\]](https://anthropic.skilljar.com/claude-code-in-action/312001)

### Extended Capabilities

SDK supports:

* Custom system prompts
* MCP servers
* Hooks
* Subagents
* Session resumption [\[The Claude Code SDK\]](https://anthropic.skilljar.com/claude-code-in-action/312001)

## 2. Q\&A from the Page

**Q1. What is the main purpose of the Claude Code SDK?**

* Run Claude Code programmatically in apps/scripts using an agent loop.

**Q2. What languages are supported?**

* TypeScript and Python.

**Q3. What is the core API used in examples?**

* `query()` function to run prompts and stream results.

**Q4. What kind of output does the SDK return?**

* A stream of JSON messages representing:
  * conversation events
  * tool calls
  * tool results

**Q5. How to restrict tool usage?**

* Use `allowedTools` in query options.

**Q6. Difference between SDK and CLI package?**

* SDK: importable library
* CLI: standalone tool, not importable

**Q7. What advanced features does SDK support?**

* MCP, hooks, subagents, system prompts, session reuse

## 4. Key Takeaways

* Claude Code SDK = **programmatic version of CLI agent loop**
* Core primitive = **streaming query() async iterator**
* Key architecture:
  * Prompt → Agent loop → Tools → Stream events
* Tool governance:
  * Default: full access
  * Controlled: `allowedTools`
* SDK enables:
  * Embedding agent into backend/services
  * Building custom workflows (instead of CLI)
* Strong extensibility:
  * MCP integration
  * Hooks for lifecycle customization
  * Subagents for decomposition
