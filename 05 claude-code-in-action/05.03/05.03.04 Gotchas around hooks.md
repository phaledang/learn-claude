## 1. Summary of page content

**Source:** [Gotchas around hooks](https://anthropic.skilljar.com/claude-code-in-action/312423)

### Key concepts

* **Hook configuration issue**
  * Running `npm run setup` generates **multiple `settings.json` files** in `.claude`.

* **Security recommendations for hooks**
  * Claude Code documentation recommends using **absolute paths (not relative paths)** for hook scripts.
  * Reason:
    * Prevents **path interception attacks**
    * Prevents **binary planting attacks** [\[Gotchas around hooks\]](https://anthropic.skilljar.com/claude-code-in-action/312423)

* **Problem introduced by absolute paths**
  * Absolute paths differ across machines:
    * Example:
      * `/Users/A/project/...`
      * `C:\workspace\project\...`
  * Result:
    * Difficult to **share configuration files** across developers [\[Gotchas around hooks\]](https://anthropic.skilljar.com/claude-code-in-action/312423)

* **Solution implemented in project**
  * Use a template file:
    * `settings.example.json`
  * Introduce placeholder:
    * `$PWD` → represents project root path
  * Setup process:
    * `npm run setup` triggers `init-claude.js`
    * Script performs:
      * Replace `$PWD` with actual absolute path
      * Copy `settings.example.json`
      * Output `settings.local.json` [\[Gotchas around hooks\]](https://anthropic.skilljar.com/claude-code-in-action/312423)

### Architecture pattern

```
settings.example.json   (shareable template)
        ↓
npm run setup
        ↓
init-claude.js
        ↓
replace $PWD → absolute path
        ↓
settings.local.json     (machine-specific)
```

## 2. Q\&A from page content

### Q1. Why are absolute paths recommended for hooks?

* To prevent:
  * Path interception attacks
  * Binary planting attacks [\[Gotchas around hooks\]](https://anthropic.skilljar.com/claude-code-in-action/312423)

### Q2. What problem do absolute paths introduce?

* Cannot reuse/share configurations across different machines due to path differences [\[Gotchas around hooks\]](https://anthropic.skilljar.com/claude-code-in-action/312423)

### Q3. How does the project solve path portability?

* Uses:
  * `settings.example.json`
  * `$PWD` placeholder
  * Setup script (`init-claude.js`) to resolve paths [\[Gotchas around hooks\]](https://anthropic.skilljar.com/claude-code-in-action/312423)

### Q4. What happens during `npm run setup`?

* Installs dependencies
* Executes script that:
  * Rewrites placeholders
  * Generates local config (`settings.local.json`) [\[Gotchas around hooks\]](https://anthropic.skilljar.com/claude-code-in-action/312423)

### Q5. Why are there multiple settings files?

* Separation of concerns:
  * Template (shareable)
  * Local (machine-specific)

## 4. Key takeaways

### Design principles

* **Security-first configuration**
  * Prefer absolute paths for execution integrity

* **Environment portability pattern**
  ```
  Template + Placeholder → Setup Script → Localized Config
  ```

### Best practices

* Never directly share machine-specific configs
* Use:
  * Template config (`*.example.json`)
  * Setup automation for path resolution

### Reusable pattern (generalized)

| Problem                               | Solution                          |
| ------------------------------------- | --------------------------------- |
| Environment-specific config           | Use placeholders                  |
| Security requirement (absolute paths) | Resolve dynamically               |
| Team collaboration                    | Separate template vs local config |
