## 1) Summary

**Page:** [Another useful hook](https://anthropic.skilljar.com/claude-code-in-action/312427)

### Core concepts

* Claude Code provides **multiple hook types** beyond `PreToolUse` and `PostToolUse`. [\[anthropic....ion/312427\]](https://anthropic.skilljar.com/claude-code-in-action/312427)
* Hooks trigger at specific lifecycle events:

| Hook             | Trigger                                 |                                                             |
| ---------------- | --------------------------------------- | ----------------------------------------------------------- |
| Notification     | When Claude needs permission or is idle |                                                             |
| Stop             | When Claude finishes responding         |                                                             |
| SubagentStop     | When a sub-agent task completes         |                                                             |
| PreCompact       | Before context compaction               |                                                             |
| UserPromptSubmit | Before prompt is processed              |                                                             |
| SessionStart     | When session starts/resumes             |                                                             |
| SessionEnd       | When session ends                       |  [\[anthropic....ion/312427\]](https://anthropic.skilljar.com/claude-code-in-action/312427) |

***

### Key challenge

* **Hook input (`stdin`) structure varies by:**
  * Hook type (e.g., `PostToolUse`, `Stop`)
  * Tool being used (for tool hooks) [\[anthropic....ion/312427\]](https://anthropic.skilljar.com/claude-code-in-action/312427)

* Example differences:
  * `PostToolUse`: includes `tool_input`,`tool_response`
  * `Stop`: minimal metadata only [\[anthropic....ion/312427\]](https://anthropic.skilljar.com/claude-code-in-action/312427)

***

### Practical solution pattern

* Use a **debug/helper hook** to inspect runtime input:

```json
"PostToolUse": [
  {
    "matcher": "*",
    "hooks": [
      {
        "type": "command",
        "command": "jq . > post-log.json"
      }
    ]
  }
]
```

* Purpose:
  * Log raw hook input
  * Understand actual schema dynamically [\[anthropic....ion/312427\]](https://anthropic.skilljar.com/claude-code-in-action/312427)

## 2) Q\&A

### Q1: Why are hooks difficult to implement?

* Because **input schema is not fixed** and varies depending on hook + tool. [\[anthropic....ion/312427\]](https://anthropic.skilljar.com/claude-code-in-action/312427)

### Q2: What is the role of `PostToolUse`?

* Executes after a tool runs and includes both **input and output of the tool**. [\[anthropic....ion/312427\]](https://anthropic.skilljar.com/claude-code-in-action/312427)

### Q3: What does the `Stop` hook provide?

* Minimal data indicating that the session has finished. [\[anthropic....ion/312427\]](https://anthropic.skilljar.com/claude-code-in-action/312427)

### Q4: How to debug hook input structure?

* Use a helper hook that writes input to a file (e.g., via `jq`). [\[anthropic....ion/312427\]](https://anthropic.skilljar.com/claude-code-in-action/312427)

### Q5: What is `UserPromptSubmit` used for?

* Intercept user input **before Claude processes it**. [\[anthropic....ion/312427\]](https://anthropic.skilljar.com/claude-code-in-action/312427)

A. PreCompact  
B. Stop  
C. UserPromptSubmit  
D. SubagentStop

✅ Answer: C

***

**4. What is the purpose of using `jq . > post-log.json`?**  
A. Modify hook input  
B. Validate schema  
C. Log raw input for inspection  
D. Execute tool

✅ Answer: C

## 4) Key Takeaways

* Hooks are **event-driven extension points** across the Claude execution lifecycle
* **Input schema is dynamic**, not static → requires runtime inspection
* Debugging strategy:
  * Always log input using a helper hook
* Advanced hooks enable:
  * Session lifecycle control
  * Prompt interception
  * Tool execution tracking
