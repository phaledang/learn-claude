## 1. Summary

**Page:** Prompts in the client  
**URL:** <https://anthropic.skilljar.com/introduction-to-model-context-protocol/296692>

### Core concept

* MCP **Prompt feature** = reusable, parameterized instruction templates exposed from MCP server → consumed by MCP client.
* Client responsibilities:
  * **List prompts**
  * **Retrieve prompt with variables**
  * **Send fully constructed prompt to Claude**

### Execution flow

1. Server defines prompts using `@mcp.prompt`
2. Client:
   * calls `list_prompts()` → discover available prompts
   * calls `get_prompt(prompt_name, args)` → inject variables
3. Prompt function returns **message blocks (user + assistant format)**
4. Client sends final messages → Claude
5. Claude executes with tools/resources if needed

***

### Code pattern (client side)

```python
async def list_prompts(self) -> list[types.Prompt]:
    result = await self.session().list_prompts()
    return result.prompts
```

```python
async def get_prompt(self, prompt_name, args: dict[str, str]):
    result = await self.session().get_prompt(prompt_name, args)
    return result.messages
```

***

### Variable interpolation

* Client passes args → becomes keyword arguments in prompt function
* Example:

```json
{
  "doc_id": "plan.md"
}
```

* Injected into prompt template → dynamic prompt generation

***

### CLI interaction model

* `/` → shows available prompts
* Selecting prompt:
  * asks for inputs (e.g. document)
  * builds final prompt
  * sends to Claude with tool access

***

### Key characteristics

* Prompts = **pre-tested, reusable instruction bundles**
* Parameterized → flexible reuse
* Returned as **structured message blocks**
* Enables **consistent AI behavior across workflows**

## 2. Q\&A

### Q1: What is the role of prompts in MCP?

* Provide reusable, structured instruction templates for AI execution

### Q2: What does `list_prompts()` do?

* Retrieves all available prompts from MCP server

### Q3: What does `get_prompt()` return?

* Fully constructed prompt messages after variable interpolation

### Q4: How are variables passed to prompts?

* Through `args` (dictionary) → converted to keyword arguments

### Q5: Where are prompts defined?

* On the MCP server using `@mcp.prompt`

### Q6: What does a prompt function return?

* A list of user/assistant messages ready for the model

### Q7: Why use prompts instead of raw prompting?

* Ensures consistency, reuse, and quality across workflows

## 4. Key Takeaways

* MCP Prompt = **reusable prompt abstraction layer**
