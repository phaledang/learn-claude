## 1. Summary (with URL)

**Page:** *Defining tools with MCP*  
**URL:** <https://anthropic.skilljar.com/introduction-to-model-context-protocol/296697>

### Core concepts

* MCP tools are defined **inside an MCP server** using the **official Python SDK**
* The SDK removes the need to manually write JSON schemas
* Tools are defined using **Python decorators + type hints**
* Example scenario:
  * A document management server
  * Documents stored in an **in-memory dictionary**
  * Two tools:
    * `read_doc_contents`
    * `edit_document`

### Key implementation pattern

```python
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("DocumentMCP", log_level="ERROR")

docs = {
  "plan.md": "The plan outlines the steps...",
}
```

### Tool definition pattern

* Use decorator `@mcp.tool(...)`
* Use **Pydantic Field** for parameter descriptions
* SDK auto-generates schema

Example:

```python
@mcp.tool(
  name="read_doc_contents",
  description="Read the contents of a document"
)
def read_document(doc_id: str = Field(...)):
    return docs[doc_id]
```

## 2. Q\&A

### Q1: What is the purpose of MCP Python SDK?

* Simplifies MCP server creation
* Automatically generates tool schemas
* Avoids manual JSON schema writing

### Q2: How are tools defined in MCP?

* Using Python decorators: `@mcp.tool`
* Function signature defines parameters
* Type hints + Field descriptions → schema

### Q3: What is the structure of tool parameters?

* Defined as function arguments
* Annotated with type hints
* Described using `Field(...)`

### Q4: What does the SDK handle automatically?

* JSON schema generation
* Tool registration
* Parameter validation

### Q5: What happens if a document is not found?

* Exception is raised:

```python
raise ValueError("Doc not found")
```

### Q6: What are the example tools?

* Read document content
* Edit document using string replacement

## 4. Key Takeaways

### MCP Tool Design Pattern

* **Function = Tool**
* **Decorator = Tool registration**
* **Type hints = Schema definition**

### Benefits

* No manual schema
* Clean Pythonic design
* Strong validation via Pydantic
* Clear contract between Claude ↔ tools

### Architecture Mapping (important for SA role)

| MCP Concept | Equivalent       |
| ----------- | ---------------- |
| Tool        | Function / API   |
| Decorator   | Tool registry    |
| Field       | Parameter schema |
| MCP Server  | Tool host        |
| Docs dict   | Data source      |
