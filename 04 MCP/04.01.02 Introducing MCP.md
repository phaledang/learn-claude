## 1. Summary

**Page:** *Introducing MCP*  
**URL:** <https://anthropic.skilljar.com/introduction-to-model-context-protocol/296689>

### Core concept

* **Model Context Protocol (MCP)** = communication layer between Claude and external systems
* Purpose:
  * Provide **tools + context + resources**
  * **Remove need for custom integration code**
* Architecture:
  ```
  MCP Client (your app/server)
        ↓
  MCP Server(s)
        ↓
  External Systems (GitHub, AWS, etc.)
  ```

### Key ideas

* MCP shifts responsibility:
  * ❌ Without MCP → developer builds all API integrations + tool schemas
  * ✅ With MCP → MCP server handles tool definitions + execution
* MCP servers expose:
  * Tools (functions like get\_repos)
  * Resources (data)
  * Prompts (predefined instructions)

### Problem solved

* Example: GitHub integration
  * Without MCP:
    * Must implement APIs for repos, PRs, issues, etc.
    * High complexity + maintenance cost
  * With MCP:
    * Use existing **GitHub MCP server**
    * Already exposes standardized tools

### Key distinction

| Concept           | Responsibility                             |
| ----------------- | ------------------------------------------ |
| MCP               | Provides tools + schemas + execution layer |
| Tool use (Claude) | Calls the tools at runtime                 |

## 2. Q\&A

### Q1: What is MCP?

* A protocol that allows Claude to access external tools, data, and prompts through standardized servers.

### Q2: What is an MCP Server?

* A server that exposes:
  * Tools
  * Resources
  * Prompts  
    as a unified interface for external systems.

### Q3: What is an MCP Client?

* The application (your backend) that connects to MCP servers.

### Q4: What problem does MCP solve?

* Eliminates the need to manually:
  * Write API integrations
  * Define tool schemas
  * Maintain external service logic

### Q5: How is MCP different from calling APIs directly?

* APIs:
  * Developer defines everything
* MCP:
  * Tools are prebuilt and standardized

### Q6: Who builds MCP servers?

* Anyone:
  * Service providers (AWS, GitHub)
  * Third-party developers

### Q7: Is MCP the same as tool use?

* No:
  * MCP → defines tools
  * Tool use → Claude executes tools

## 4. Key Takeaways

* MCP = **standardized integration layer for AI systems**
* Moves **integration complexity out of your app**
* Enables **plug-and-play tool ecosystem**
* Architecture pattern:
  ```
  App → MCP Client → MCP Server → External APIs
  ```
* Distinction:
  * MCP = infrastructure
  * Tool use = execution mechanism
* Strategic value:
  * Reduced engineering effort
  * Faster integration
  * Reusable ecosystem
