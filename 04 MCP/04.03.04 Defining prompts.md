## 1. Summary

**Page:** [Defining prompts – Anthropic MCP course](https://anthropic.skilljar.com/introduction-to-model-context-protocol/296698)

### Core concept: MCP Prompts

* MCP prompts = **pre-built, reusable instruction templates**
* Defined **on MCP server**, used by **clients**
* Purpose: replace ad-hoc user prompts with **optimized, tested instructions**

### Why prompts matter

* Users *can* write prompts → but:
  * inconsistent
  * missing edge cases
* MCP prompts provide:
  * **higher quality output**
  * **standardized behavior**
  * **embedded prompt engineering best practices**

### Key idea

> Prompt engineering effort is centralized at the MCP server → reused by all clients

***

### Example: `/format` command

**Workflow**

1. User types `/format doc_id`
2. Client calls MCP prompt `format`
3. Claude executes pre-built prompt
4. Document → converted to structured Markdown

***

### Implementation pattern

```python
@mcp.prompt(
    name="format",
    description="Rewrites the contents of the document in Markdown format."
)
def format_document(
    doc_id: str = Field(description="Id of the document to format")
) -> list[base.Message]:
    
    prompt = f"""
    Your goal is to reformat a document to be written with markdown syntax.

    The id of the document:
    <document_id>{doc_id}</document_id>

    Add headers, bullet points, tables as needed.
    Use 'edit_document' tool to update the document.
    """

    return [base.UserMessage(prompt)]
```

### Important mechanics

* Prompt function returns **list of messages**
* Messages sent directly to Claude
* Supports:
  * multi-turn prompt structure
  * user + assistant messages

***

### Testing approach

* Use **MCP Inspector**
* Validate:
  * variable interpolation
  * final prompt structure
  * message sequence

***

### Benefits

* **Consistency** → same result across users
* **Expertise reuse** → encode domain knowledge
* **Reusability** → shared across clients
* **Maintainability** → update once → affect all apps

## 2. Q\&A

### Q1. What is an MCP prompt?

A reusable, server-defined instruction template used by clients instead of writing prompts manually.

### Q2. Where are prompts defined?

On the **MCP server**.

### Q3. What does a prompt function return?

A **list of messages** sent to Claude.

### Q4. Why use prompts instead of letting users write instructions?

To ensure:

* consistent output
* better quality
* handling of edge cases

### Q5. What is the role of MCP Inspector?

To test and verify prompt behavior before deployment.

### Q6. How is a prompt invoked?

Through a command (e.g., `/format`) or client-triggered action.

## 4. Key Takeaways

* MCP prompts = **prompt-as-a-service layer**
* Centralized prompt engineering → improves:
  * reliability
  * maintainability
* Pattern:
  ```
  User → Client → MCP Prompt → Claude
  ```
* Prompts behave similarly to:
  * tools (function-based)
  * resources (data-based)
* Best suited for:
  * domain-specific workflows
  * repeatable tasks (format, summarize, analyze)
