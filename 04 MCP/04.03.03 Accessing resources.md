### Core Concepts

* **Resources in MCP**
  * Mechanism to **expose data directly to prompts** (no tool call required)
  * Used for **efficient context injection** into AI models [\[Accessing resources\]](https://anthropic.skilljar.com/introduction-to-model-context-protocol/296695)

* **Execution Flow**
  * User types: `@resource_name`
  * Client detects → sends **ReadResourceRequest**
  * Server returns **ReadResourceResult** with content [\[Accessing resources\]](https://anthropic.skilljar.com/introduction-to-model-context-protocol/296695)

* **Key Benefit**
  * Avoids tool round-trip → **faster + simpler context usage**
  * Content becomes part of **initial prompt context** [\[Accessing resources\]](https://anthropic.skilljar.com/introduction-to-model-context-protocol/296695)

***

### Implementation Pattern

```python
import json
from pydantic import AnyUrl

async def read_resource(self, uri: str) -> Any:
    result = await self.session().read_resource(AnyUrl(uri))
    resource = result.contents[0]

    if isinstance(resource, types.TextResourceContents):
        if resource.mimeType == "application/json":
            return json.loads(resource.text)
        return resource.text
```

#### Key Points

* Call: `session().read_resource(uri)`
* Response:
  * `contents[]` list (usually take first item)
  * Contains:
    * content (text/data)
    * MIME type
    * metadata
* MIME-based handling:
  * `application/json` → parse JSON
  * otherwise → raw text [\[Accessing resources\]](https://anthropic.skilljar.com/introduction-to-model-context-protocol/296695)

## 2. Q\&A

**Q1. What is a resource in MCP?**

* A way for servers to expose data that can be directly included in prompts instead of accessed via tools.

**Q2. How does resource access flow work?**

* `@resource` → client detects → ReadResourceRequest → server returns content.

**Q3. Why use resources instead of tools?**

* More efficient → avoids additional tool calls → better latency and simplicity.

**Q4. What does ReadResourceResult contain?**

* contents list (data), MIME type, metadata.

**Q5. How is JSON handled?**

* If MIME type is `application/json` → parse using `json.loads`.

**Q6. What UX feature supports resource usage?**

* Autocomplete when typing `@`.

## 4. Key Takeaways

* **Resources = context injection layer**
  * alternative to tools

* **Flow pattern**
  ```
  User (@resource) 
      → Client 
          → MCP Server (ReadResource) 
              → Content → Prompt
  ```

* **Design principle**
  * Use **resources for static/known data**
  * Use **tools for dynamic actions**

* **Implementation**
  * Must implement `read_resource`
  * Handle **MIME type correctly**

* **UX advantage**
  * Autocomplete + inline context
  * no extra LLM-tool loop
