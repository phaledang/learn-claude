## 1. Summary

**Page:** [The server inspector](https://anthropic.skilljar.com/introduction-to-model-context-protocol/296693)

### Purpose

* MCP development requires **testing tools without a full client/app integration**
* The **MCP Inspector (built into Python SDK)** provides:
  * Browser-based UI
  * Real-time debugging
  * Tool execution testing

### Key Concept

* Inspector = **local dev environment for MCP server**
* Eliminates need for:
  * Custom test scripts
  * Full client integration

***

### How to Start Inspector

```bash
mcp dev mcp_server.py
```

* Starts local server
* Provides URL (e.g., `http://127.0.0.1:6274`)
* Open in browser → Inspector UI

***

### UI Components

* **Connect button**
  * Initializes server
  * Status: `Disconnected → Connected`
* **Navigation tabs**
  * Tools
  * Resources
  * Prompts
* **Tool testing panel**
  * Input fields
  * Output results
  * Execution status

***

### Tool Testing Flow

Example: test `read_doc_contents`

1. Go to **Tools**
2. Click **List Tools**
3. Select tool (`read_doc_contents`)
4. Provide input:
   ```json
   { "doc_id": "deposition.md" }
   ```
5. Click **Run Tool**
6. Check:
   * Success status
   * Returned data

***

### Advanced Testing

* Supports **multi-step workflows**
  * Example:
    * Run edit tool → modify doc
    * Run read tool → verify changes
* State persists across calls
  * Enables testing **tool interaction logic**

***

### Development Benefits

* Fast iteration
* Real-time debugging
* Validate:
  * Tool logic
  * Edge cases
  * State handling
* Replaces:
  * External test harness
  * Manual integration testing
