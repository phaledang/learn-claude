## 2. Structured Content

### What Projects Are
- Persistent workspace  
- Contains knowledge, chats, instructions  
- Used for specific workstreams  

### When to Use
- Ongoing tasks  
- Reusable materials  
- Consistent response format  
- Collaboration needs  

### Creating Project
- Setup → name, description, visibility  
- Add instructions → tone, style, rules  
- Add knowledge → documents  

### Knowledge Base
- Supports multiple file types  
- Uses RAG for large data  

### Collaboration
- Can view  
- Can edit  
- Owner  

### Use Cases
- Product launch  
- Research  
- Client hub  
- Event planning  
- Job description  

### Best Practices
- Start focused  
- Keep data updated  
- Clear instructions  
- Use descriptive filenames
