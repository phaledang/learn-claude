## 2. Key Concepts

### 2.1 What are Subagents

* Specialized assistants that Claude Code delegates tasks to [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)
* Operate in **separate conversation context windows** [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)
* Workflow:
  * Receive:
    * Custom system prompt (role + behavior)
    * Task description from parent agent [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)
  * Execute independently:
    * Read files, run searches, call tools [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)
  * Return:
    * **Only a concise summary** to main thread [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)
* Intermediate steps are **isolated and discarded** [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)

***

### 2.2 Why Subagents Matter (Context Management Problem)

* Main conversation has a **finite context window** [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)
* Without isolation:
  * Every:
    * Tool call
    * File read
    * Search result  
      → accumulates in context [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)
* When full:
  * Model may **lose earlier information** [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)

✅ Subagent solution:

* Run tasks in **separate context window** [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)
* Return **summary only**
* Keep main context **clean and efficient**

⚠️ Trade-off:

* Lose visibility into:
  * intermediate steps
  * reasoning trail [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)

***

### 2.3 Execution Model (Parent–Subagent Pattern)

**Parent Agent**

* Receives user request
* Creates task description
* Delegates work

**Subagent**

* Executes task independently
* Uses tools and data
* Returns summarized result

**Lifecycle**

1. Spawn subagent
2. Execute task
3. Return summary
4. Context discarded [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)

***

### 2.4 Practical Example

**Use case: find service handling refunds in a codebase**

Without subagent:

* Reads \~15 files
* Runs multiple searches
* Traces functions
* → All stored in main context

With subagent:

* Subagent handles exploration
* Main thread receives:
  * ✅ Final answer only
* Context remains minimal [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)

## 3. Q\&A

**Q1. What is a subagent?**  
A specialized assistant delegated by the main agent to perform a specific task in its own context window. [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)

**Q2. What does a subagent receive?**

* A system prompt defining role
* A task description from the parent agent [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)

**Q3. What is returned to the main conversation?**  
Only a **concise summary**, not intermediate steps. [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)

**Q4. Why use subagents?**  
To avoid filling the main context window and maintain performance. [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)

**Q5. What is the downside of subagents?**  
Lack of visibility into how the result was produced. [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)
