# 2. Key Concepts

## Overview

The lesson focuses on patterns that make subagents effective. Poorly configured subagents may wander, run too long, or return unusable output. Four key practices improve reliability: descriptions, output formats, obstacle reporting, and tool limitations. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

***

## How Subagent Config Data Gets Used

### Main Agent Uses Name and Description

* Every available subagent's name and description are included in the main agent's system prompt. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)
* The main agent uses them to determine:
  * When to launch a subagent.
  * Which subagent should handle a task. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

### Description Influences Delegation

* The description not only controls triggering.
* It also guides how the main agent writes the input prompt for the subagent. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)
* Better descriptions produce better delegated tasks. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

**Key Insight**

```text
Description
  ├─ Controls WHEN a subagent runs
  └─ Influences WHAT instructions the subagent receives
```

## Writing Descriptions That Shape Input Prompts

### Problem With Generic Descriptions

Generic description:

```text
Use get diff to find the current changes.
```

Result:

* Instructions are vague.
* Subagent must determine important files itself. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

### Improved Description

Add explicit guidance:

```text
You must tell the agent precisely which files you want it to review.
```

Result:

* Main agent generates more detailed prompts.
* Specific file lists are provided automatically. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

### Another Example

For a web-search subagent:

```text
Return sources that can be cited.
```

Result:

* Main agent includes citation requirements when delegating. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

**Best Practice**

* Treat descriptions as instructions for both:
  * Triggering behavior.
  * Delegation quality. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

## Defining an Output Format

### Most Important Improvement

The page identifies structured output as the single most important enhancement for subagents. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

### Benefits

#### 1. Natural Stopping Point

A subagent knows its work is complete when all required sections are filled. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

#### 2. Prevents Excessive Runtime

Without a defined output:

* Subagents struggle to decide when enough work is done.
* Research often continues longer than necessary. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

### Example Code Review Output Structure

1. Summary
2. Critical Issues
3. Major Issues
4. Minor Issues
5. Recommendations
6. Approval Status [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

### Why This Works

The format becomes a checklist.

```text
Fill every section
      ↓
Work completed
      ↓
Return results
```

 [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

## Reporting Obstacles

### Problem

Subagents often discover useful information during execution:

* Environment quirks.
* Dependency fixes.
* Required command flags.
* Workarounds. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

If these findings are not returned:

* The main thread must rediscover them.
* Time and tokens are wasted. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

### Recommended Section

```text
Obstacles Encountered
```

Include:

* Setup issues.
* Environment quirks.
* Workarounds.
* Commands needing special flags.
* Dependency/import problems. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

### Benefit

Knowledge discovered during execution becomes reusable by the primary workflow. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

## Limiting Tool Access

### Principle

Do not give every tool to every subagent. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

### Benefits

* Prevents unintended side effects.
* Makes roles clearer.
* Reduces risk. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

### Recommended Tool Allocation

#### Research / Read-Only Subagent

Tools:

* Glob
* Grep
* Read

Characteristics:

* Cannot modify files. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

#### Code Reviewer

Tools:

* Bash

Purpose:

* Run git diff.
* Inspect changes.

Does not need:

* Edit
* Write [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

#### Styling / Code Modification Agent

Tools:

* Edit
* Write

Purpose:

* Modify source code. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

## Putting It All Together

### Four Characteristics of Effective Subagents

| Characteristic        | Purpose                                               |
| --------------------- | ----------------------------------------------------- |
| Specific descriptions | Control triggering and delegation quality             |
| Structured output     | Define completion criteria and improve result quality |
| Obstacle reporting    | Preserve discoveries and workarounds                  |
| Limited tool access   | Reduce risk and clarify responsibilities              |

 [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

### Core Takeaway

A well-designed subagent is:

* Focused.
* Predictable.
* Efficient.
* Able to stop at the right time.
* Able to communicate its findings clearly. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

# 3. Q\&A

### Q1. Why are subagent descriptions important?

**Answer:** They determine both when a subagent is launched and how the main agent constructs the task prompt. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

### Q2. What is the most impactful subagent improvement?

**Answer:** Defining a structured output format in the system prompt. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

### Q3. Why use structured outputs?

**Answer:** They create natural stopping conditions and prevent subagents from running longer than necessary. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

### Q4. What should an obstacle report contain?

**Answer:** Setup issues, workarounds, environment quirks, special command flags, and dependency problems. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

### Q5. Why limit tool access?

**Answer:** To prevent unintended side effects and make the subagent's role more clearly defined. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

### Q6. Which tools should a research subagent typically receive?

**Answer:** Glob, Grep, and Read. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

### Q7. Should code reviewers receive Edit and Write permissions?

**Answer:** No. They typically need Bash access but not code modification permissions. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)

### Q8. When should Edit and Write permissions be granted?

**Answer:** When the subagent's responsibility is to make code changes. [\[Designing...subagents\]](https://anthropic.skilljar.com/introduction-to-subagents/450700)
