## 2. Key Concepts

### 2.1 Definition

* **Subagents** = specialized assistants delegated by a main agent (Claude Code). [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)
* Each subagent:
  * Runs in its **own independent context window** [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)
  * Executes tasks (file reads, searches, tool calls) [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)
  * Returns only a **summary** to the main conversation [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)
* Intermediate steps are **isolated and discarded** after completion [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)

***

### 2.2 Why Subagents Matter (Context Management)

* Main agent context window is **finite** → accumulates:
  * Messages
  * Tool calls
  * File reads [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)
* Large workflows can:
  * Fill context
  * Cause loss of earlier information [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)
* Subagents mitigate this by:
  * Spinning up **separate context windows** [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)
  * Keeping main conversation **clean and focused** [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)

***

### 2.3 How Subagents Work

#### Inputs

* Receive:
  * **System prompt** (defines role/behavior) [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)
  * **Task description** from parent agent [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)

#### Execution

* Perform:
  * File reading
  * Searching
  * Code editing
* All actions remain inside **isolated context** [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)

#### Output

* Return:
  * Only a **concise summary**
* Entire working context:
  * **Discarded after completion** [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)

***

### 2.4 Trade-off

* ✅ Benefits:
  * Clean main context
  * Reduced noise
  * Better scalability for long sessions
* ❌ Limitation:
  * No visibility into intermediate reasoning steps [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)

***

### 2.5 Practical Example

* Scenario: Find which service handles refunds in a codebase
* Without subagent:
  * Reads many files
  * Runs multiple searches
  * Context becomes cluttered
* With subagent:
  * Subagent performs exploration
  * Main agent receives **only final answer**
  * Context stays minimal [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)

***

### 2.6 Built-in Subagents

* **General-purpose subagent**
  * Multi-step execution + action [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)
* **Explore**
  * Codebase search/navigation [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)
* **Plan**
  * Research & analysis before planning [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)

***

### 2.7 Custom Subagents

* Can define:
  * Custom system prompts
  * Tool access
* Example roles:
  * Code reviewer
  * Test generator
  * Documentation writer [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)

***

### 2.8 Key Benefits Summary

* Task decomposition → focused execution
* Context isolation → no pollution of main thread
* Output abstraction → only relevant summary returned [\[anthropic....nts/450699\]](https://anthropic.skilljar.com/introduction-to-subagents/450699)

## 3. Q\&A

**Q1. What is a subagent?**

* A delegated, specialized assistant running in its own isolated context that returns only a summary.

**Q2. Why are subagents needed?**

* To prevent context window overflow and maintain efficiency in long or complex workflows.

**Q3. What does a subagent receive to start work?**

* A system prompt (role definition) and a task description.

**Q4. What happens to intermediate steps?**

* They remain isolated and are discarded after completion.

**Q5. What is the main trade-off?**

* Loss of visibility into the reasoning process.

**Q6. Can subagents be customized?**

* Yes, with custom prompts and tool configurations.
