## Source

## 1. Core Concept

* A **subagent = specialized AI worker** with:
  * Its own **context window**
  * Its own **system prompt**
  * Its own **tool access**
* Main agent delegates tasks → subagent executes → returns **summary only**
* Purpose:
  * Avoid context overload
  * Enable specialization (reviewer, tester, doc writer, etc.) [\[anthropic....illjar.com\]](https://anthropic.skilljar.com/introduction-to-subagents), [\[code.claude.com\]](https://code.claude.com/docs/en/sub-agents)

***

# 2. How to Create a Subagent (Step-by-step)

### ✅ Method (Recommended)

* Use `/agents` slash command → UI opens
* Select **Create new agent**
* Claude can auto-generate config from description [\[github.com\]](https://github.com/ai-native-engineer/anthropic-mirror/blob/main/anthropic.skilljar.com/introduction-to-subagents/02-creating-a-subagent.md)

### ✅ Scope selection

* **Project-level**
  * Only available in current project
* **User-level**
  * Shared across all projects

### ✅ Creation approaches

* Manual config (advanced)
* **Claude-generated config (recommended)**
  * Input: "what agent should do"
  * Output: name + description + system prompt

***

# 3. Tool Configuration (Critical Design Decision)

### Tool categories

* Read-only tools
* Edit tools
* Execution tools
* MCP tools
* Other tools [\[github.com\]](https://github.com/ai-native-engineer/anthropic-mirror/blob/main/anthropic.skilljar.com/introduction-to-subagents/02-creating-a-subagent.md)

### Design principle

* **Least-privilege**
  * Only enable what agent needs

### Example

| Use case         | Tools                       |
| ---------------- | --------------------------- |
| Code reviewer    | ✅ Read, ✅ Execution, ❌ Edit |
| Code modifier    | ✅ Read, ✅ Edit              |
| Debug/test agent | ✅ Execution                 |

***

# 4. Model Selection Strategy

| Model       | Use case                                                                             |
| ----------- | ------------------------------------------------------------------------------------ |
| **Haiku**   | Fast, lightweight tasks                                                              |
| **Sonnet**  | Balanced (default choice)                                                            |
| **Opus**    | Deep analysis, complex reasoning                                                     |
| **Inherit** | Same as main conversation [\[github.com\]](https://github.com/ai-native-engineer/anthropic-mirror/blob/main/anthropic.skilljar.com/introduction-to-subagents/02-creating-a-subagent.md) |

✅ Architect rule:

* Use **Haiku for scale**
* Use **Opus for correctness-critical tasks**

***

# 5. Config File Structure (Critical for Implementation)

### Location

```
.claude/agents/<agent-name>.md
```

***

## YAML Frontmatter (Config)

```yaml
---
name: code-quality-reviewer
description: Use this agent when reviewing modified code...
tools: Bash, Glob, Grep, Read, WebSearch
model: sonnet
color: purple
---
```

***

## Markdown Body = System Prompt

* Defines:
  * Role
  * Behavior
  * Output structure
* This is the **most important part**

***

# 6. Field Breakdown (Must master)

| Field           | Purpose                           |
| --------------- | --------------------------------- |
| **name**        | Unique identifier (`@agent-name`) |
| **description** | When Claude should use it         |
| **tools**       | Allowed capabilities              |
| **model**       | Which Claude model                |
| **color**       | UI identification                 |

✅ Key rule:

* `name + description` → **mandatory drivers of delegation** [\[anthropicc...ations.com\]](https://www.anthropiccertifications.com/courses/introduction-to-subagents/the-subagent-config-file)

***

# 7. System Prompt Design (Most Important)

### What it controls

* What to analyze
* How to reason
* How to output results

### Best practices

* Be explicit:
  * Scope
  * Constraints
  * Output format
* Example:
  * "Check security vulnerabilities"
  * "Return issues as bullet list with severity"

✅ Insight:

> Poor system prompt = irrelevant results  
> Strong system prompt = specialized expert behavior

***

# 8. Automatic Delegation (Advanced)

### Enable auto usage

Add in description:

```
"Proactively suggest running this agent..."
```

### Improve accuracy

* Add:
  * Trigger scenarios
  * Example conversations

✅ Key idea:

* Claude decides delegation based on **description field**

***

# 9. Testing & Iteration

### How to test

* Perform task (e.g., code change)
* Ask Claude to review

### If not triggered:

* Fix **description**
  * Add:
    * clearer intent
    * explicit triggers

✅ Iteration loop:

1. Run
2. Observe
3. Refine description

***

# 10. Mental Model (Architecture View)

```
Main Agent (Orchestrator)
        ↓
  Subagent (Specialist)
        ↓
 Works in isolated context
        ↓
Returns summarized result
```

✅ Benefits:

* Context isolation
* Parallel execution
* Reusability
* Tool safety control [\[code.claude.com\]](https://code.claude.com/docs/en/agent-sdk/subagents)

***

# 11. Design Patterns (Architect-level)

### ✅ Pattern 1 — Reviewer Agent

* Tools: Read + Execution
* Model: Sonnet
* Focus: quality + security

### ✅ Pattern 2 — Test Agent

* Tools: Execution (bash)
* Model: Sonnet/Opus
* Focus: validation

### ✅ Pattern 3 — Research Agent

* Tools: WebSearch + Read
* Model: Haiku (cost-efficient)

***

# 12. Common Mistakes

* ❌ Too many tools → unsafe / noisy
* ❌ Weak description → no delegation
* ❌ Generic system prompt → poor specialization
* ❌ Wrong model choice → cost/performance issues

***

* Critical components:
  * **description → drives delegation**
  * **system prompt → defines intelligence**
  * **tools → define capability boundaries**
* Optimize for:
  * specialization
