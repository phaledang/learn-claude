## Summary

This lesson explains how connectors allow Claude to integrate with external tools and data sources. Instead of working in isolation, Claude becomes an informed collaborator that can access real data and perform actions on behalf of the user.

Connectors are powered by the Model Context Protocol (MCP), which acts as a universal integration standard across applications.

---

## Key Concepts

### What are Connectors?
- Enable Claude to access tools, files, and data
- Allow both reading and performing actions
- Reduce manual copy-paste workflows

### Model Context Protocol (MCP)
- Universal connector standard (like USB-C for AI)
- Provides consistent interface across tools
- Enables scalable integrations

---

## Types of Connectors

### 1. Web Connectors
- Connect to cloud services
- Examples: Google Drive, Slack, Notion, Asana

### 2. Desktop Extensions
- Run locally via Claude Desktop app
- Examples:
  - Local file system access
  - Browser automation
  - Native app integrations (e.g., Figma)

---

## Setting Up a Connector

1. Find connector (directory or "+" menu)
2. Click connect
3. Authenticate account
4. Grant permissions
5. Test connection

## Security & Permissions

- Permission-based access
- Claude only sees data user can access
- Connectors can be revoked anytime
- Only install trusted connectors
