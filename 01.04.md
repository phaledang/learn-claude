# Claude Desktop App: Chat, Cowork, Code

## 1. Summary
Source: https://anthropic.skilljar.com/claude-101/440908

### Chat mode:
- Best for quick interactions like brainstorming, drafting, or Q&A  
- Supports quick entry overlay, screenshots, dictation, and local connectors  
- Focused on interactive, back-and-forth work  
- Similar to claude.ai with desktop-native enhancements  
- Ideal when speed and iteration matter  

---

### Cowork mode:
- Best for complex, multi-step work (research, analysis, document creation)  
- Executes tasks autonomously based on a defined goal  
- Works across multiple sources (files, tools, web, connectors)  
- Supports:
  - Subagents for parallel task execution  
  - Scheduled tasks for recurring workflows  
  - Folder access for reading/writing local files  
  - Browser interaction (navigate and extract data from websites)  
  - Computer use (click, type, open apps) when no connector is available  
  - Plugins to extend capabilities (e.g., enterprise data, financial data)  
- Builds a plan, shows progress, and produces complete deliverables  
- Prioritizes execution paths:
  1. Connectors  
  2. Browser (Chrome)  
  3. Screen/computer interaction  
- Runs in a protected environment limited to shared folders  
- Ideal for deep work and end-to-end execution  

---

### Code mode:
- Best for building, testing, and deploying software  
- Works directly with your codebase (local or GitHub remote)  
- Supports:
  - Reading and modifying files  
  - Terminal execution  
  - Git version control  
  - Visual diffs for code changes  
- Execution environments:
  - Local → runs on your machine  
  - Remote → runs in cloud (e.g., GitHub), supports long-running tasks  
- Interaction modes:
  - Ask → requires approval before changes  
  - Code → applies file changes automatically, asks before running commands  
  - Plan → outlines approach before execution  
- Ideal for full development workflows  

---

### Key positioning:
- Chat = interaction  
- Cowork = execution (agentic workflows)  
- Code = development

## 2. Q&A

- Q: What are the three modes?
  - A: Chat, Cowork, Code  

- Q: When should you use Chat?
  - A: Quick questions, brainstorming, drafting, interactive problem-solving  

- Q: What makes Cowork different from Chat?
  - A: Executes multi-step tasks autonomously and produces full deliverables  

- Q: What are subagents?
  - A: Parallel workers handling subtasks  

- Q: What are scheduled tasks?
  - A: Automation for recurring workflows  

- Q: What is the difference between local and remote in Code?
  - A:
    - Local → runs on your machine  
    - Remote → runs in cloud  

- Q: What are Code interaction modes?
  - A: Ask, Code, Plan  

- Q: Which mode is best for multi-source research?
  - A: Cowork  

- Q: What is Cowork execution priority order?
  - A:
    1. Connectors  
    2. Browser  
    3. Screen/computer interaction

- Claude desktop app has 3 modes optimized for different workloads  
- Chat → fast, interactive, iterative  
- Cowork → autonomous, multi-step execution
