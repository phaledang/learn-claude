## 4Ds of AI Fluency 
***

### 1. **Delegation — Decide the Human–AI boundary**

* Definition
  * Decide **whether, when, and how to use AI** for a task [\[www-cdn.an...hropic.com\]](https://www-cdn.anthropic.com/b383cf6baddbfc72fdf8b0ed533a518e2872d531.pdf)

* Core idea
  * Split work between:
    * **AI → repetitive, scalable, data-heavy tasks**
    * **Human → judgment, ethics, strategy**

* Key activities
  * Define goals before using AI
  * Understand AI capabilities & limitations
  * Assign tasks strategically

* Example
  * ✅ AI: summarize logs, generate drafts
  * ✅ Human: approve architecture, validate business logic

***

### 2. **Description — Tell AI exactly what to do**

* Definition
  * Clearly describe goals so AI produces useful outputs [\[www-cdn.an...hropic.com\]](https://www-cdn.anthropic.com/b383cf6baddbfc72fdf8b0ed533a518e2872d531.pdf)

* Core idea
  * **Output quality = input clarity**

* Key components
  * Context → background, domain
  * Task → what to do
  * Format → expected structure
  * Constraints → rules, tone, limits

* Example
  * Weak: “Write API docs”
  * Strong:
    * “Write REST API documentation, markdown format, include request/response examples, for internal dev team”

***

### 3. **Discernment — Evaluate AI output critically**

* Definition
  * Assess usefulness, accuracy, and quality of AI output [\[www-cdn.an...hropic.com\]](https://www-cdn.anthropic.com/b383cf6baddbfc72fdf8b0ed533a518e2872d531.pdf)

* Core idea
  * AI outputs are **probabilistic, not guaranteed truth**

* What to check
  * Accuracy (facts, logic)
  * Relevance (fit to requirement)
  * Completeness (missing parts)
  * Risks (bias, hallucination)

* Typical actions
  * Cross-check important data
  * Review reasoning
  * Treat AI output as **draft, not final**

***

### 4. **Diligence — Use AI responsibly**

* Definition
  * Take responsibility for how AI is used and its outputs [\[www-cdn.an...hropic.com\]](https://www-cdn.anthropic.com/b383cf6baddbfc72fdf8b0ed533a518e2872d531.pdf)

* Core idea
  * **You own the outcome, not the AI**

* Key practices
  * Verify outputs before usage
  * Maintain transparency (AI-assisted work)
  * Ensure compliance (security, data, ethics)

* Example
  * Do not:
    * Ship AI-generated code without review
  * Do:
    * Validate, test, and approve before production

***

## 🔁 Relationship (how 4Ds work together)

```
Delegation   → decide what AI does
Description  → instruct AI clearly
Discernment  → evaluate results
Diligence    → take responsibility for outcome
```

* They form a **closed loop for AI collaboration**
* Missing any step → failure mode:
  * Bad delegation → wrong problem solved
  * Poor description → low-quality output
  * Weak discernment → wrong decisions
  * No diligence → risk in production

***

## Quick mental model (for Solution Architect mindset)

* Delegation → **Architecture decision**
* Description → **Interface / contract definition**
* Discernment → **Validation / testing**
* Diligence → **Governance / compliance**
