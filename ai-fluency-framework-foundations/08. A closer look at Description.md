## 2. Key Concepts

### ✅ Definition

* **Description (AI Fluency competency)**
  * Communicating intentions clearly to AI systems
  * Not just writing prompts → **creating a collaborative environment**
  * AI acts as an **interactive partner**, not a tool or database [\[A closer l...escription\]](https://anthropic.skilljar.com/ai-fluency-framework-foundations/291891)

***

### ✅ Core Principle

* AI **cannot infer intent automatically**
* Quality of output depends on:
  * Clarity of requirements
  * Structure of communication
  * Defined expectations for interaction [\[A closer l...escription\]](https://anthropic.skilljar.com/ai-fluency-framework-foundations/291891)

***

### ✅ 3 Types of Description

#### 1. Product Description (WHAT)

* Define expected output clearly
* Include:
  * Format
  * Audience
  * Style
  * Content scope

➡ Example

* “Create a technical README.md for Azure architecture, targeted to Solution Architects, using bullet format”

***

#### 2. Process Description (HOW)

* Guide the AI on **approach / reasoning**
* Control:
  * Steps
  * Methods
  * Constraints

➡ Example

* “Break down solution into architecture layers, then explain each component step-by-step”

***

#### 3. Performance Description (BEHAVIOR)

* Define how AI should act during interaction
* Control:
  * Tone (formal, technical)
  * Depth (concise vs detailed)
  * Interaction style (challenge vs support)

➡ Example

* “Respond in concise bullet points, avoid narrative explanation, keep it enterprise-ready”

***

### ✅ Key Takeaways

* Description = **structured communication layer**
* Covers:
  * Output definition (Product)
  * Execution guidance (Process)
  * Interaction style (Performance)
* Proper Description:
  * Reduces iteration cycles
  * Improves response accuracy
  * Enables AI collaboration

***

### ✅ Operational Insight (Important for Architect role)

* Treat AI like a **distributed system with strict interface contracts**
* Description = **API contract between human and AI**
  * Product → output schema
  * Process → execution workflow
  * Performance → runtime behavior

➡ Poor Description → undefined contract → inconsistent outputs

## 3. Q\&A

### Q1. What is the purpose of Description?

* Enable clear communication with AI to produce accurate and useful outputs

### Q2. Why is Description important?

* AI does not infer intent → requires explicit instruction

### Q3. What are the 3 components?

* Product (what)
* Process (how)
* Performance (behavior)

### Q4. What happens if Description is weak?

* Ambiguous output
* More iterations
* Misaligned expectations
